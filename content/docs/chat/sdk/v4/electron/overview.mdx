This documentation set describes Electron SDK v3.8.3-patch.10. It is organized by OpenIM SDK capabilities: initialization, login, event handlers, users, friends, blacklist, groups, conversations, messages, push, logs, files, and common models.

Each exported method has its own page with the platform method signature, parameters, return value, example shape, related types, and related pages.

## Integration Model

| Model            | Description                                                                                           |
| ---------------- | ----------------------------------------------------------------------------------------------------- |
| Initialization   | Configure API address, WebSocket address, local data directory, logging, and the connection listener. |
| Login state      | Register listeners before login when possible, then call the login API with a backend-issued token.   |
| Async result     | Use the platform result model: Future, Promise, or callback depending on the SDK.                     |
| Message creation | Create local message objects first, then pass them to the send APIs.                                  |
| operationID      | Generate a traceable operationID for each business operation.                                         |

## Currently Unavailable Capabilities

Electron SDK v3.8.3-patch.10 does not expose standalone methods for public group variants, polls, scheduled messages, message threads, translation, independent reports, group metadata, or metacounters. Add these pages only after the platform SDK exposes corresponding methods.
