## 加入群组

拿到可加入的 `groupID` 后调用 `joinGroup()`。固定 Flutter package 的 barrel 没有导出内部 `JoinSource`，公开方法实际接收 `int`：`2` 表示邀请、`3` 表示搜索、`4` 表示二维码。应用可以为这些固定值声明自己的具名常量。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `String` | 必填 | 要加入的群组 ID。 |
| `reason` | `String?` | 可选 | 申请说明；没有说明时可以省略。 |
| `joinSource` | `int` | 可选 | 加入途径：邀请 `2`、搜索 `3`、二维码 `4`；默认值为搜索 `3`。 |
| `ex` | `String?` | 可选 | 申请加入时携带的业务扩展字符串。 |

```dart
const joinSourceSearch = 3;

await OpenIM.iMManager.groupManager.joinGroup(
  groupID: groupID,
  reason: '申请加入项目讨论组',
  joinSource: joinSourceSearch,
);
```

`joinSource` 只记录来源，不决定是否审核。群组允许直接加入时，Future 成功表示已加入；需要审核时只表示申请已提交，不能立即写入成员记录。

## 退出群组

```dart
await OpenIM.iMManager.groupManager.quitGroup(groupID: groupID);
```

普通成员和管理员可退出；群主必须先转让群主身份。退出不等于删除历史会话。

## 解散群组

```dart
await OpenIM.iMManager.groupManager.dismissGroup(groupID: groupID);
```

只有群主可以解散。解散影响所有成员，不同于隐藏或删除当前用户的会话。

## 调用后的状态变化

| API | Future 成功后 | 事件处理 |
| --- | --- | --- |
| `joinGroup()` | 直接加入时表示已经加入；需要审核时只表示申请已提交。 | 只有当前账号实际加入后，才通过 `onJoinedGroupAdded` 写入已加入群组列表。 |
| `quitGroup()` | 当前账号已经退出目标群组。 | 通过 `onJoinedGroupDeleted` 按 `groupID` 移除群组。 |
| `dismissGroup()` | 群主已经解散目标群组。 | 通过 `onGroupDismissed` 将群组标记为已解散并关闭成员管理入口。 |

不要在 `joinGroup()` 的 Future 成功分支中伪造 `GroupInfo` 或 `GroupMembersInfo`。完整事件处理方式见[获取和搜索群组](/sdk/flutter/group/retrieving-groups/retrieve-and-search-groups)。历史会话是否保留属于会话数据，不应由群组事件处理器直接删除；需要校准当前账号是否仍在群内时，重新调用 `isJoinedGroup(groupID: groupID)`。
