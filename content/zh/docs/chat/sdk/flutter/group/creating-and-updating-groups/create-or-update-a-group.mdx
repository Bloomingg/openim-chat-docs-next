## 创建群组

`createGroup()` 接收 `GroupInfo` 以及初始普通成员、管理员和群主配置。普通成员与管理员列表不能重复，群主也不要放入这两个列表。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupInfo` | `GroupInfo` | 必填 | 初始群资料；`groupType` 使用公开的 `GroupType` 常量。 |
| `memberUserIDs` | `List<String>` | 可选 | 创建后作为普通成员加入群组的用户 ID。 |
| `adminUserIDs` | `List<String>` | 可选 | 创建后作为管理员加入群组的用户 ID。 |
| `ownerUserID` | `String?` | 可选 | 群主用户 ID；省略时由服务端按当前用户处理。 |

```dart
final group = await OpenIM.iMManager.groupManager.createGroup(
  groupInfo: GroupInfo(
    groupID: '',
    groupName: '项目讨论组',
    groupType: GroupType.work,
    faceURL: 'https://example.com/group.png',
  ),
  memberUserIDs: ['user-003', 'user-004'],
  adminUserIDs: ['user-002'],
  ownerUserID: currentUserID,
);
```

`groupID` 传空字符串时由 OpenIMServer 生成；自定义时必须全局唯一。Future 成功后返回新建群组的 `GroupInfo`。

## 更新群资料

`setGroupInfo()` 接收包含 `groupID` 的 `GroupInfo`。只设置本次需要修改的字段，避免旧快照覆盖刚同步的新状态。

固定 Flutter package 的 barrel 没有导出内部 `AllowType`。`applyMemberFriend` 和 `lookMemberInfo` 是公开模型上的 `int?` 字段，固定值 `0` 表示允许、`1` 表示不允许。应用可以在自己的业务层声明具名常量：

```dart
const allowed = 0;

await OpenIM.iMManager.groupManager.setGroupInfo(
  GroupInfo(
    groupID: groupID,
    groupName: '新的群名称',
    notification: '本周五发布',
    introduction: '项目协作群',
    needVerification: GroupVerification.allNeedVerification,
    applyMemberFriend: allowed,
    lookMemberInfo: allowed,
    ex: jsonEncode({'projectID': 'project-42'}),
  ),
);
```

### 参数说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `groupID` | `String` | 要更新的群组 ID。 |
| `groupName` | `String?` | 群名称。 |
| `notification` | `String?` | 群公告。 |
| `introduction` | `String?` | 群介绍。 |
| `needVerification` | `int?` | 入群验证策略，使用公开的 `GroupVerification` 常量。 |
| `applyMemberFriend` | `int?` | 是否允许群成员互相申请好友：`0` 允许，`1` 不允许。 |
| `lookMemberInfo` | `int?` | 是否允许查看群成员资料：`0` 允许，`1` 不允许。 |
| `ex` | `String?` | 完整替换的业务扩展字符串。 |

`ex` 是完整字符串，不会自动合并 JSON 字段。复杂查询、独立权限控制和审计数据应由业务后端保存。

## 群权限和验证设置

客户端应先读取最新 `GroupInfo` 和当前用户的群角色，再根据 `applyMemberFriend`、`lookMemberInfo`、`needVerification` 决定是否显示对应入口，以及提交后是直接执行还是进入审核。SDK 只提供字段值，不会自动替应用隐藏按钮或组织流程；最终权限仍由 OpenIMServer 校验。

## 调用后的状态变化

`createGroup()` Future 成功后，可以使用返回的 `GroupInfo` 打开群页面；`onJoinedGroupAdded` 用于合并当前账号的群列表增量。`setGroupInfo()` Future 成功只表示服务端完成设置请求，后续 `onGroupInfoChanged` 按 `groupID` 合并最终群资料。完整 listener 见[获取和搜索群组](/sdk/flutter/group/retrieving-groups/retrieve-and-search-groups)。需要立即确认最终值时，再调用 `getGroupsInfo(groupIDList: [groupID])`；重新查询失败不应被误报为设置失败。
