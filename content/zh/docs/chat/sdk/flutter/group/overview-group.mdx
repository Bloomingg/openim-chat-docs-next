OpenIMSDK 的多人聊天使用群组承载共享资料、成员关系和权限。`groupID` 是群组稳定标识，发送群消息、读取群资料、管理成员和处理入群申请都使用它。当前用户聊天列表中的群会话仍由 `conversationID` 管理，两者不能混用。

## 核心数据类型

| 类型 | 用途 |
| --- | --- |
| `GroupInfo` | 群名称、头像、公告、群主、验证配置、成员数和扩展字段。 |
| `GroupMembersInfo` | 用户在指定群内的昵称、角色、禁言和入群信息。 |
| `GroupApplicationInfo` | 入群申请及处理状态。 |

群角色使用 `GroupRoleLevel`：普通成员为 `20`，管理员为 `60`，群主为 `100`。前端可据此控制入口，但最终权限由 OpenIMServer 校验。

## 状态更新

群组事件由对应数据页面统一监听：群资料、当前用户加入或退出以及群组解散见[获取和搜索群组](/sdk/flutter/group/retrieving-groups/retrieve-and-search-groups)；成员新增、删除和资料变化见[获取群成员](/sdk/flutter/group/retrieving-group-members/retrieve-group-members)；入群申请变化见[处理入群申请](/sdk/flutter/group/managing-group-applications/manage-group-applications)。

概览页只说明事件归属，不设置 listener。群组写操作的 Future 结果、listener 事件增量和重新查询校准应分别处理；重新登录后的群组变化由事件同步，必要时重新查询群资料、成员或申请列表校准页面状态。

OpenIM Flutter SDK 的群组 API 只处理 OpenIMSDK 群资料、成员关系和权限。公开发现、复杂目录和大规模群聊的容量治理属于业务后端与 OpenIMServer 配置，不应扩写成不存在的客户端群组类型。
