入群申请由群主或管理员处理，与成员邀请不同：申请人主动提交请求，邀请则由群内用户发起。

## 查询申请

管理端使用 `getGroupApplicationListAsRecipient()` 读取收到的申请，申请人使用 `getGroupApplicationListAsApplicant()` 读取自己发出的申请。两个操作的请求对象字段相同，但查询方向和权限不同。

### 查询收到的申请

#### 参数说明

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupIDs` | `List<String>` | 可选 | 要筛选的群组 ID；空列表表示不按群组限制。 |
| `handleResults` | `List<int>` | 可选 | 处理状态：未处理 `0`、同意 `1`、拒绝 `-1`。 |
| `offset` | `int` | 必填 | 起始偏移量，第一页传 `0`。 |
| `count` | `int` | 必填 | 本次读取数量；`offset > 0` 时必须大于 `0`。 |

```dart
final pendingApplications = await OpenIM.iMManager.groupManager
    .getGroupApplicationListAsRecipient(
  req: GetGroupApplicationListAsRecipientReq(
    groupIDs: [groupID],
    handleResults: [0],
    offset: 0,
    count: 20,
  ),
);
```

Future 成功返回当前用户有权处理的 `List<GroupApplicationInfo>` 快照。

### 查询发出的申请

该操作的 `groupIDs`、`handleResults`、`offset` 和 `count` 分别传给 `GetGroupApplicationListAsApplicantReq`，字段含义相同，但结果只包含当前用户发出的申请：

#### 参数说明

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupIDs` | `List<String>` | 可选 | 要筛选的群组 ID；空列表表示不按群组限制。 |
| `handleResults` | `List<int>` | 可选 | 处理状态：未处理 `0`、同意 `1`、拒绝 `-1`。 |
| `offset` | `int` | 可选 | 起始偏移量，默认 `0`。 |
| `count` | `int` | 可选 | 本次读取数量，默认 `40`。 |

```dart
final sentApplications = await OpenIM.iMManager.groupManager
    .getGroupApplicationListAsApplicant(
  req: GetGroupApplicationListAsApplicantReq(
    groupIDs: [groupID],
    handleResults: [0, 1, -1],
    offset: 0,
    count: 20,
  ),
);
```

Future 成功返回当前用户发出的 `List<GroupApplicationInfo>` 快照。两个列表继续加载时都按已经请求的记录数增加 `offset`；返回数量小于 `count` 时通常表示到达末页。

## 未处理数量

```dart
final count = await OpenIM.iMManager.groupManager
    .getGroupApplicationUnhandledCount(
  GetGroupApplicationUnhandledCountReq(time: 0),
);
```

`time` 是统计起点时间；传 `0` 读取当前用户作为群主或管理员需要处理的全部申请数量。查询列表和数量只建立快照，不会因为读取而触发申请事件。

固定 Flutter SDK 没有群申请 badge 读取、清除或删除申请记录 API，不应移植 WASM 对应调用。

## 同意或拒绝

申请记录中的 `groupID` 和 `userID` 都是 nullable。处理前必须确认两个标识有效；不要使用空字符串发起审核请求。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `String` | 必填 | 申请对应的群组 ID。 |
| `userID` | `String` | 必填 | 申请人的用户 ID，对应 `GroupApplicationInfo.userID`。 |
| `handleMsg` | `String?` | 可选 | 同意或拒绝时的处理说明。 |

```dart
final applicationGroupID = application.groupID;
final applicantUserID = application.userID;
if (applicationGroupID == null ||
    applicationGroupID.isEmpty ||
    applicantUserID == null ||
    applicantUserID.isEmpty) {
  throw StateError('Group application is missing groupID or userID');
}

await OpenIM.iMManager.groupManager.acceptGroupApplication(
  groupID: applicationGroupID,
  userID: applicantUserID,
  handleMsg: '同意加入',
);

await OpenIM.iMManager.groupManager.refuseGroupApplication(
  groupID: applicationGroupID,
  userID: applicantUserID,
  handleMsg: '暂不接受该申请',
);
```

Future 成功表示服务端完成处理请求，不应直接伪造最终成员记录。

## 保持申请列表同步

申请事件处理函数应纳入应用唯一的 `OnGroupListener`：

```dart
void handleApplicationChanged(GroupApplicationInfo application) {
  final groupID = application.groupID;
  final userID = application.userID;
  if (groupID == null || groupID.isEmpty || userID == null || userID.isEmpty) {
    return;
  }
  mergeGroupApplication(application);
}

void handleApplicationDeleted(GroupApplicationInfo application) {
  final groupID = application.groupID;
  final userID = application.userID;
  if (groupID == null || groupID.isEmpty || userID == null || userID.isEmpty) {
    return;
  }
  removeGroupApplication(groupID, userID);
}

```

`onGroupApplicationAdded`、`onGroupApplicationAccepted` 和 `onGroupApplicationRejected` 应调用合并函数，`onGroupApplicationDeleted` 调用删除函数。申请记录按 `groupID:userID` 幂等合并；事件也可能来自其他管理员或其他客户端，不是当前 Future 的专属完成回调。分页期间收到变化后应重置并从第一页重新加载，同时重新读取未处理数量。组合 listener 与唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。
