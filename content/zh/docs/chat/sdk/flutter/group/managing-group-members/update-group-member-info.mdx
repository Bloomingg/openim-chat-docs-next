使用 `setGroupMemberInfo()` 更新成员在指定群内的资料。它接收 `SetGroupMemberInfo`，`groupID` 和 `userID` 用于定位成员，其余可空字段只传本次需要修改的值。

## 参数说明

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `String` | 是 | 目标群组 ID。 |
| `userID` | `String` | 是 | 目标成员用户 ID。 |
| `nickname` | `String?` | 否 | 群内昵称。 |
| `faceURL` | `String?` | 否 | 群内头像地址。 |
| `roleLevel` | `int?` | 否 | 群成员角色值；群主转让使用专门接口。 |
| `ex` | `String?` | 否 | 群成员扩展字符串。 |

```dart
await OpenIM.iMManager.groupManager.setGroupMemberInfo(
  groupMembersInfo: SetGroupMemberInfo(
    groupID: groupID,
    userID: targetUserID,
    nickname: '项目负责人',
    roleLevel: GroupRoleLevel.admin,
  ),
);
```

`roleLevel` 使用 `GroupRoleLevel` 中的固定角色值：

| 枚举值 | 数值 | 说明 |
| --- | --- | --- |
| `GroupRoleLevel.member` | `20` | 普通群成员。 |
| `GroupRoleLevel.admin` | `60` | 群管理员。 |
| `GroupRoleLevel.owner` | `100` | 群主；不能通过资料更新直接完成群主转让。 |

## 更新昵称和角色

修改群内昵称时只传 `nickname`，修改成员角色时只传 `roleLevel`；两个字段需要同时变化时，可以放在一次 `setGroupMemberInfo()` 调用中。群主负责设置或移除管理员，前端角色按钮只是交互控制，最终权限由 OpenIMServer 校验。不能把角色设为 `GroupRoleLevel.owner` 来替代 `transferGroupOwner()`。

`ex` 是完整的群成员扩展字符串，不会按 JSON 字段自动合并。多个业务模块共用时，应先读取目标成员资料，合并已有命名空间后再写回完整值。

## 调用后的状态变化

Future 成功表示 OpenIMServer 已完成本次群成员资料更新。相关客户端随后可能通过 `onGroupMemberInfoChanged` 收到更新后的单个 `GroupMembersInfo`，应按 `groupID:userID` 覆盖本地成员记录。

该 listener 也可能收到其他管理员或其他客户端产生的修改，不能只把它当作当前调用的完成回调。完整监听和合并代码见[获取群成员](/sdk/flutter/group/retrieving-group-members/retrieve-group-members)；首次进入成员列表或用户主动刷新时，再查询目标成员获取当前资料。
