群组整体禁言与单个成员禁言是两个不同状态。

## 操作权限

- 群主可以禁言管理员和普通成员，也可以开启或关闭群组整体禁言。
- 管理员只能禁言普通成员，不能禁言群主或其他管理员。
- 普通成员不能修改群组或其他成员的禁言状态。

前端可以按 `GroupRoleLevel` 隐藏无权限入口，但最终权限仍由 OpenIMServer 校验。

## 群组禁言

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `String` | 必填 | 目标群组 ID。 |
| `mute` | `bool` | 必填 | `true` 开启群组禁言，`false` 关闭。 |

```dart
await OpenIM.iMManager.groupManager.changeGroupMute(
  groupID: groupID,
  mute: true,
);
```

`mute` 为 `true` 时开启，为 `false` 时关闭。固定 Flutter SDK 不支持 WASM 的 `muteBypassUserIDs` 参数。群禁言下群主和管理员仍可发送，因此不表示所有用户都无法发送。

## 成员禁言

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `String` | 必填 | 目标群组 ID。 |
| `userID` | `String` | 必填 | 要设置禁言状态的成员用户 ID。 |
| `seconds` | `int` | 可选 | 禁言时长，单位为秒；默认 `0`，表示解除禁言。 |

```dart
await OpenIM.iMManager.groupManager.changeGroupMemberMute(
  groupID: groupID,
  userID: targetUserID,
  seconds: 3600,
);
```

成员资料中的 `muteEndTime` 是服务端同步的禁言截止时间。固定 Flutter SDK 注释该值单位为秒，应和当前秒级时间戳比较；不要只按提交值推算最终状态。

## 调用后的状态变化

两个 Future 成功分别表示服务端完成群组或成员禁言设置。群资料和成员资料随后可能通过 `onGroupInfoChanged`、`onGroupMemberInfoChanged` 到达；事件可能来自其他管理员或其他客户端，不是当前 Future 的专属完成回调。

完整处理见[获取和搜索群组](/sdk/flutter/group/retrieving-groups/retrieve-and-search-groups)与[获取群成员](/sdk/flutter/group/retrieving-group-members/retrieve-group-members)。需要立即校准时，分别调用 `getGroupsInfo(groupIDList: [groupID])` 或 `getGroupMembersInfo(groupID: groupID, userIDList: [targetUserID])`；重新查询失败不应被误报为禁言设置失败。
