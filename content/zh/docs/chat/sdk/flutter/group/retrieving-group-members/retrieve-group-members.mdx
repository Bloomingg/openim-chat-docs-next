群成员资料由 `GroupMembersInfo` 表示。调用者必须已经加入目标群组。

## 分页获取成员

固定 Flutter package 的 barrel 没有导出内部 `GroupMemberFilter`。公开 `getGroupMemberList()` 实际接收 `int`：`0` 全部、`1` 群主、`2` 管理员、`3` 普通成员。固定 SDK 原始码把 `adminAndMember` 和 `superAndAdmin` 都定义为 `4`；文档必须忠实反映这个版本，不能按其他平台把 `superAndAdmin` 推测成 `5`。需要明确区分“管理员与普通成员”和“群主与管理员”时，直接调用 `getGroupOwnerAndAdmin()` 获取后者，不要依赖这个冲突的筛选值。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `String` | 必填 | 目标群组 ID。 |
| `filter` | `int` | 可选 | 成员角色筛选值，默认 `0`，表示全部成员。 |
| `offset` | `int` | 可选 | 起始偏移量，第一页传 `0`。 |
| `count` | `int` | 可选 | 本次读取数量；应显式传入固定页大小。 |

```dart
const allGroupMembers = 0;

final members = await OpenIM.iMManager.groupManager.getGroupMemberList(
  groupID: groupID,
  filter: allGroupMembers,
  offset: 0,
  count: 50,
);
```

Future 成功返回当前页的 `List<GroupMembersInfo>`。继续加载时按已经请求的条目数增加 `offset`，不要使用事件合并后的数组长度；返回数量小于 `count` 时通常表示到达末页。

Flutter SDK 没有独立的禁言成员列表 API。需要这类视图时应分页读取全部成员，再检查 `muteEndTime`；固定 SDK 注释其单位为秒：

```dart
bool isMemberMuted(GroupMembersInfo member) {
  final end = member.muteEndTime ?? 0;
  return end > DateTime.now().millisecondsSinceEpoch ~/ 1000;
}
```

## 获取指定成员

```dart
final selectedMembers =
    await OpenIM.iMManager.groupManager.getGroupMembersInfo(
  groupID: groupID,
  userIDList: userIDList,
);
```

同一用户在不同群中的昵称、角色和禁言状态可能不同，本地缓存应使用 `groupID:userID` 作为键。只判断用户是否仍在群内时，可调用 `getUsersInGroup(groupID, userIDs)`。

```dart
final presentUserIDs = await OpenIM.iMManager.groupManager.getUsersInGroup(
  groupID,
  userIDList,
);
```

该查询返回目标群组中实际存在的用户 ID；它只建立当前快照，不触发成员事件。

## 搜索成员和获取 @ 候选人

`searchGroupMembers()` 只搜索指定群组。先去除关键词首尾空白，并且不要用空关键词发起搜索。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `String` | 必填 | 目标群组 ID。 |
| `keywordList` | `List<String>` | 可选 | 搜索关键词；当前只使用第一个非空元素。 |
| `isSearchUserID` | `bool` | 可选 | 是否匹配成员 `userID`。 |
| `isSearchMemberNickname` | `bool` | 可选 | 是否匹配群内昵称。 |
| `offset` | `int` | 可选 | 起始偏移量，第一页传 `0`。 |
| `count` | `int` | 可选 | 本次读取数量。 |

```dart
final matchedMembers =
    await OpenIM.iMManager.groupManager.searchGroupMembers(
  groupID: groupID,
  keywordList: [keyword.trim()],
  isSearchUserID: true,
  isSearchMemberNickname: true,
  offset: 0,
  count: 50,
);
```

Future 成功返回当前页匹配的 `List<GroupMembersInfo>`。搜索分页独立于完整成员列表；关键词或筛选字段变化时重置 `offset`。只需要群主和管理员时，无需遍历全部成员：

```dart
final ownerAndAdmins =
    await OpenIM.iMManager.groupManager.getGroupOwnerAndAdmin(
  groupID: groupID,
);
```

## 查询结果与成员变化

查询结果是当前快照，不触发成员事件。成员事件的处理函数应纳入应用唯一的 `OnGroupListener`：

```dart
void handleGroupMemberAdded(GroupMembersInfo member) {
  upsertGroupMember(member);
}

void handleGroupMemberDeleted(GroupMembersInfo member) {
  final groupID = member.groupID;
  final userID = member.userID;
  if (groupID == null || groupID.isEmpty || userID == null || userID.isEmpty) {
    return;
  }
  removeGroupMember(groupID, userID);
}

void handleGroupMemberInfoChanged(GroupMembersInfo member) {
  upsertGroupMember(member);
}

```

三个事件都按 `groupID:userID` 幂等合并；它们可能来自其他管理员或其他客户端，不是查询 API 触发的回调。分页期间成员变化后应重置分页并从第一页重新查询；用户主动刷新时同样重建快照。组合 listener 与唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。
