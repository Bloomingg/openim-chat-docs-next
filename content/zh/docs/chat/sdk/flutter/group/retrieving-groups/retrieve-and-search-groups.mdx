## 获取指定群组

```dart
final groups = await OpenIM.iMManager.groupManager.getGroupsInfo(
  groupIDList: groupIDs,
);
```

`getGroupsInfo()` 返回匹配的 `List<GroupInfo>`。读取单个群组时仍传列表，并检查 `groups` 是否为空。

## 获取已加入群组

群组较少时调用 `getJoinedGroupList()`；数量较多时分页读取：

```dart
final firstPage =
    await OpenIM.iMManager.groupManager.getJoinedGroupListPage(
  offset: 0,
  count: 50,
);
```

两个方法只返回当前用户已加入的群组，不是公开群目录。继续加载时按请求条目数增加 `offset`。

## 检查成员状态

```dart
final isJoined = await OpenIM.iMManager.groupManager.isJoinedGroup(
  groupID: groupID,
);
```

返回值表示当前用户是否已经加入目标群组。

## 搜索已加入群组

`searchGroups()` 只搜索当前用户已加入且已同步到本地的群组，并且当前只使用第一个关键词。

```dart
final matchedGroups = await OpenIM.iMManager.groupManager.searchGroups(
  keywordList: [keyword.trim()],
  isSearchGroupID: true,
  isSearchGroupName: true,
);
```

公开群发现、复杂分类和权限过滤应由业务后端实现。

## 查询快照与群组事件

查询只建立快照，不触发事件。以下群组回调应纳入应用唯一的 `OnGroupListener`：

```dart
void handleJoinedGroupAdded(GroupInfo group) => mergeJoinedGroup(group);
void handleJoinedGroupDeleted(GroupInfo group) =>
    removeJoinedGroup(group.groupID);
void handleGroupInfoChanged(GroupInfo group) => mergeGroup(group);
void handleGroupDismissed(GroupInfo group) =>
    markGroupDismissed(group.groupID);
```

事件按 `groupID` 幂等合并；分页期间收到新增或删除事件时，应重置分页并从第一页重新加载。组合 listener 与唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。
