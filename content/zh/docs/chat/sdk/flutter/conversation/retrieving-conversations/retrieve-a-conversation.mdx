OpenIMSDK 支持按聊天目标读取一个会话，也支持按 `conversationID` 批量读取会话。调用前应完成 SDK 登录。

## 常见使用场景

| 入口 | 推荐处理方式 |
| --- | --- |
| 从通讯录点击某个用户 | 使用对方 `userID` 和 `ConversationType.single` 调用 `getOneConversation()`。 |
| 从群资料或群组列表进入聊天 | 使用 `groupID` 和 `ConversationType.superGroup` 调用 `getOneConversation()`。 |
| 从会话列表进入聊天 | 直接使用列表中的 `ConversationInfo`；需要刷新时再按 `conversationID` 查询。 |
| 从通知携带 `conversationID` 进入 | 使用 `getMultipleConversation()` 读取对应会话，并检查结果是否包含目标记录。 |

```dart
Future<void> openChatFromContact(String targetUserID) async {
  final conversation =
      await OpenIM.iMManager.conversationManager.getOneConversation(
    sourceID: targetUserID,
    sessionType: ConversationType.single,
  );

  openChatPage(targetUserID: targetUserID, conversation: conversation);
}
```

双方是否已经产生过消息不影响调用方式。首次聊天和已有聊天都应让 SDK 返回 `ConversationInfo`，不要自行拼接 `conversationID` 或构造会话对象。

## 按聊天目标获取会话

群聊把 `groupID` 作为 `sourceID`：

```dart
final conversation =
    await OpenIM.iMManager.conversationManager.getOneConversation(
  sourceID: groupID,
  sessionType: ConversationType.superGroup,
);

openConversation(conversation.conversationID);
```

Future 成功后返回 `ConversationInfo`。失败时重点检查登录状态、目标 ID、会话类型和连接状态。

## 获取 conversationID

只需要会话 ID 时，调用 `getConversationIDBySessionType()`：

```dart
final conversationID = await OpenIM.iMManager.conversationManager
    .getConversationIDBySessionType(
  sourceID: targetUserID,
  sessionType: ConversationType.single,
);
```

这个方法返回会话 ID，不返回完整 `ConversationInfo`。

## 按 conversationID 批量获取

```dart
final conversations = await OpenIM.iMManager.conversationManager
    .getMultipleConversation(conversationIDList: conversationIDs);

final byID = {
  for (final conversation in conversations)
    conversation.conversationID: conversation,
};
```

返回值是 `List<ConversationInfo>`。某个 ID 没有结果时，应按缺失记录处理，不要依赖数组位置与请求列表完全一一对应。

## 调用结果与后续变化

本页三个查询方法都应直接使用 Future 返回值完成当前导航或查询，不要等待会话事件后才打开聊天页，也不要把查询成功描述为触发事件。页面打开后的增量由[获取会话列表](/sdk/flutter/conversation/retrieving-conversations/retrieve-conversation-list)统一处理。
