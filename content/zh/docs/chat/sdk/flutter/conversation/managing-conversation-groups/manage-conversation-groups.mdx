会话分组用于整理当前登录用户的会话列表，例如“工作”“未读”或“稍后处理”。它管理 `conversationID` 与分组的关系，不会创建聊天群组、修改群成员，也不会改变消息发送目标。

## 分组类型

Flutter SDK 的 `ConversationGroupType` 公开三种值：

| 枚举值 | 数值 | 用途 |
| --- | ---: | --- |
| `ConversationGroupType.custom` | `0` | 普通自定义分组，可由用户创建和维护。 |
| `ConversationGroupType.preset` | `1` | 由 SDK 或服务端规则维护的预设分组。 |
| `ConversationGroupType.all` | `2` | 查询全部类型时使用。 |

普通用户分类使用 `custom`。预设分组的生成规则取决于当前 OpenIMServer 部署，不要只凭名称在客户端伪造系统分组。

## 创建和查询分组

### 创建分组

`CreateConversationGroupReq.groupName` 必填；其余字段可选：

```dart
final group = await OpenIM.iMManager.conversationGroupManager
    .createConversationGroup(
  req: CreateConversationGroupReq(
    groupName: '工作',
    order: 100,
    conversationID: conversationID,
    conversationGroupType: ConversationGroupType.custom,
  ),
);
```

| 字段 | 说明 |
| --- | --- |
| `groupName` | 分组名称，序列化给 SDK 时对应 `name`。 |
| `order` | 可选顺序值；排序方向应由产品统一约定。 |
| `ex` | 可选业务扩展字符串。 |
| `conversationID` | 可选；创建后立即加入该分组的会话。 |
| `conversationGroupType` | 默认 `ConversationGroupType.custom`。 |

返回的 `ConversationGroupInfo` 常用字段包括 `conversationGroupID`、`name`、`serial`、`version`、`ex`、`conversationGroupType`、`hidden`、`unreadCount` 和 `conversationIDs`；这些字段均为 nullable。

### 查询分组

```dart
final groups = await OpenIM.iMManager.conversationGroupManager
    .getConversationGroups(
  conversationGroupType: ConversationGroupType.custom,
);

final memberships = await OpenIM.iMManager.conversationGroupManager
    .getConversationGroupByConversationID(conversationID: conversationID);
```

`getConversationGroups()` 默认查询 `ConversationGroupType.all`，返回排序后的 `List<ConversationGroupInfo>`。`getConversationGroupByConversationID()` 返回该会话所属的完整分组对象列表，不是 WASM 同名能力返回的 ID 字符串列表。

### 分页查询分组内会话

```dart
final page = await OpenIM.iMManager.conversationGroupManager
    .getConversationGroupInfoWithConversations(
  req: GetConversationGroupInfoWithConversationsReq(
    groupID: group.conversationGroupID!,
    pagination: RequestPagination(pageNumber: 1, showNumber: 20),
  ),
);

renderConversationGroup(page.group, page.conversationElems ?? const []);
```

`groupID` 在此 request 中表示 `conversationGroupID`。`pageNumber` 从 `1` 开始，`showNumber` 是每页数量。响应提供 nullable 的 `group`、当前页 `conversationElems` 和 `isEnd`；Flutter model 没有 WASM 响应中的 `conversationTotal`，分页终止应依据 `isEnd`，不能臆造总数。

## 添加或移除会话

```dart
await OpenIM.iMManager.conversationGroupManager.addConversationsToGroups(
  conversationGroupIDs: [groupID],
  conversationIDs: [conversationID],
);

await OpenIM.iMManager.conversationGroupManager.removeConversationsFromGroups(
  conversationGroupIDs: [groupID],
  conversationIDs: [conversationID],
);
```

两个数组都必填：`conversationGroupIDs` 是目标分组，`conversationIDs` 是要关联或移除的会话。同一个会话可以属于多个分组；增删关联不会删除会话或清空消息。

## 更新、排序和删除

```dart
await OpenIM.iMManager.conversationGroupManager.updateConversationGroup(
  req: UpdateConversationGroupReq(
    conversationGroupID: groupID,
    name: '重点工作',
    hidden: false,
  ),
);

await OpenIM.iMManager.conversationGroupManager.setConversationGroupOrder(
  orders: [
    ConversationGroupOrder(groupID: firstGroupID, order: 100),
    ConversationGroupOrder(groupID: secondGroupID, order: 200),
  ],
);

await OpenIM.iMManager.conversationGroupManager
    .deleteConversationGroup(conversationGroupID: groupID);
```

`UpdateConversationGroupReq.conversationGroupID` 必填，只传本次要更新的 `name`、`ex`、`hidden` 或 `order`。`ConversationGroupOrder.groupID` 和删除方法的 `conversationGroupID` 都表示会话分组 ID，不是聊天群组 ID。删除只移除分组与关联，不删除其中会话或消息。

以上更新、排序、删除和成员关系 API 的 Flutter 声明返回未具体化的 `Future`，成功表示当前请求已完成，不应声称返回更新后的 `ConversationGroupInfo`。后续状态由 listener 增量和必要时重新查询校准。

## 监听分组变化

```dart
Future<void> registerConversationGroupListener() {
  return OpenIM.iMManager.conversationGroupManager
      .setConversationGroupListener(
    OnConversationGroupListener(
      onConversationGroupAdded: mergeConversationGroups,
      onConversationGroupChanged: mergeConversationGroups,
      onConversationGroupDeleted: removeConversationGroups,
      onConversationGroupMemberAdded: mergeConversationGroupMembers,
      onConversationGroupMemberDeleted: removeConversationGroupMembers,
    ),
  );
}
```

| Listener | 载荷 | 处理方式 |
| --- | --- | --- |
| `onConversationGroupAdded` | `List<ConversationGroupInfo>` | 合并新增分组。 |
| `onConversationGroupChanged` | `List<ConversationGroupInfo>` | 按 `conversationGroupID` 更新。 |
| `onConversationGroupDeleted` | `List<ConversationGroupInfo>` | 移除已删除分组。 |
| `onConversationGroupMemberAdded` | `List<ConversationGroupMemberChangedInfo>` | 合并各元素的 `group`、`conversationIDs` 与 `conversations`。 |
| `onConversationGroupMemberDeleted` | `List<ConversationGroupMemberChangedInfo>` | 移除对应关联。 |

本页是五个会话分组 listener 的完整归属页。每个 `ConversationGroupMemberChangedInfo` 的字段仍可能为空，应先检查 `group?.conversationGroupID` 和会话 ID。主动调用与 listener 可能覆盖同一分组，始终按 `conversationGroupID` 幂等合并，成员关系再结合 `conversationID`。

`ConversationGroupManager` 只保存一个 listener，固定 SDK 没有 remove 或 unset API。应用应集中设置一次；切换账号时停止向旧状态分发，并用新账号的完整 listener 覆盖。三个查询方法只建立快照，不应描述为触发分组 listener。
