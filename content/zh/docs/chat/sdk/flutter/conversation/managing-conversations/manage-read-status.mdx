普通会话未读数保存在 `ConversationInfo.unreadCount`，群聊 @ 状态保存在 `groupAtType`，两者是不同状态。

## 标记会话已读

```dart
await OpenIM.iMManager.conversationManager.markConversationMessageAsRead(
  conversationID: conversationID,
);
```

不要在只预览通知或后台收到消息时提前标记已读。Future 成功表示请求完成，会话未读数仍通过 `onConversationChanged` 合并。

## 获取总未读数

```dart
final value = await OpenIM.iMManager.conversationManager
    .getTotalUnreadMsgCount();
final totalUnread = int.tryParse(value.toString()) ?? 0;
updateApplicationBadge(totalUnread);
```

总未读数不是把当前已加载各页会话的未读数简单相加得到的值，应以 SDK 查询和回调为准。

本页回调必须并入应用唯一的 `OnConversationListener`：

```dart
void handleTotalUnreadChanged(int count) {
  updateApplicationBadge(count);
}
```

组合 listener 与唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。

## 清理群聊 @ 状态

```dart
await OpenIM.iMManager.conversationManager.setConversation(
  conversationID,
  ConversationReq(groupAtType: GroupAtType.atNormal),
);
```

标记消息已读和清理 @ 提醒可能发生在同一界面，但不是同一个 API 状态。`getTotalUnreadMsgCount()` 只返回快照，不触发 `onTotalUnreadMessageCountChanged`；后续变化由回调更新。
