会话列表中的置顶、消息接收选项、私聊、阅后即焚、消息销毁、群聊 @ 状态和扩展字段，可以通过 `setConversation()` 设置。调用时传入目标 `conversationID` 和 `ConversationReq`；未传入的字段保持原有状态。

## 参数说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `conversationID` | `String` | 必填，目标会话 ID，作为 `setConversation()` 第一个参数。 |
| `recvMsgOpt` | `int?` | 消息接收选项，使用 `ReceiveMessageOpt` 常量。 |
| `groupAtType` | `int?` | 群聊 @ 提醒状态，使用 `GroupAtType` 常量。 |
| `burnDuration` | `int?` | 阅后即焚时长，单位为秒。 |
| `msgDestructTime` | `int?` | 消息销毁时长，单位为秒。 |
| `isPinned` | `bool?` | 是否置顶会话。 |
| `isPrivateChat` | `bool?` | 是否启用私聊状态。 |
| `isMsgDestruct` | `bool?` | 是否启用消息销毁。 |
| `ex` | `String?` | 应用约定的会话扩展字符串。 |

```dart
await OpenIM.iMManager.conversationManager.setConversation(
  conversationID,
  ConversationReq(
    isPinned: true,
    recvMsgOpt: ReceiveMessageOpt.receive,
    ex: '{"category":"work"}',
  ),
);
```

`ex` 是完整字符串，不会自动合并 JSON。固定基线的 `ConversationReq` 没有 `isMarked` 和 `remark` 字段，不要把其他平台字段传入 Flutter API。

## 设置消息销毁

```dart
await OpenIM.iMManager.conversationManager.setConversation(
  conversationID,
  ConversationReq(isMsgDestruct: true, msgDestructTime: 3600),
);
```

`msgDestructTime` 的单位为秒。关闭时传 `isMsgDestruct: false`。

## 调用后的状态变化

Future 成功表示本次设置请求已完成。变化后的状态通过 `onConversationChanged` 同步，回调携带 `List<ConversationInfo>`，应按 `conversationID` 合并。完整 listener 见[获取会话列表](/sdk/flutter/conversation/retrieving-conversations/retrieve-conversation-list)。需要校准时再查询指定会话。
