群聊 @ 消息使用 `createTextAtMessage()`。提及逻辑必须基于 OpenIMSDK 用户 ID，而不是昵称、头像或会话 ID。`text` 应保存可稳定解析的 `@userID` 标记，`atUserIDList` 决定实际提醒对象，`atUserInfoList` 提供 ID 与群昵称的展示映射。

## 按用户 ID 提及成员

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `text` | `String` | 必填 | 完整正文；需要渲染成员节点时使用 `@userID` 作为稳定标记。 |
| `atUserIDList` | `List<String>` | 必填 | 被提醒的群成员 ID。 |
| `atUserInfoList` | `List<AtUserInfo>` | 可选 | 用户 ID 与群昵称映射，应与 ID 列表来自同一份成员数据。 |
| `quoteMessage` | `Message?` | 可选 | 同时引用的原消息。 |

```dart
final memberUserID = member.userID;
if (memberUserID == null || memberUserID.isEmpty) {
  throw StateError('Group member has no userID');
}

final message = await OpenIM.iMManager.messageManager.createTextAtMessage(
  text: '@$memberUserID 请确认发布内容',
  atUserIDList: [memberUserID],
  atUserInfoList: [
    AtUserInfo(
      atUserID: memberUserID,
      groupNickname: member.nickname,
    ),
  ],
);

await OpenIM.iMManager.messageManager.sendMessage(
  message: message,
  groupID: groupID,
  offlinePushInfo: OfflinePushInfo(title: groupName, desc: '@小明 请确认发布内容'),
);
```

发送前应确认正文不为空、目标用户仍在当前群内，并确保 `atUserIDList` 与 `atUserInfoList` 来自同一份成员数据。@ 消息面向群聊发送，因此 `sendMessage()` 传目标 `groupID`，不要同时把它当作单聊消息发送。

## 渲染可点击的 @ 成员

接收端同时读取 `message.atTextElem` 中的正文、用户 ID 列表和展示信息。只有与 `atUserID` 完整匹配的标记才渲染为可点击成员，点击时仍使用用户 ID 打开资料；不要用可能重复或变化的昵称反查 ID。

发送端编辑器可以显示 `@小明`，但提交给 `createTextAtMessage()` 时应序列化为稳定的 `@userID` 标记。接收端切分正文时，候选 ID 必须来自 `atUserInfoList`，并处理特殊字符和完整标记边界；没有匹配到成员信息的内容按普通文本显示。

## 提及全体成员

固定 Flutter SDK 通过 `conversationManager.atAllTag` 暴露全体成员标识，当前值为 `AtAllTag`。不要把全部群成员 ID 展开到列表，也不要把界面文本中的“@所有人”当作权限依据。

```dart
final atAllTag = OpenIM.iMManager.conversationManager.atAllTag;
final message = await OpenIM.iMManager.messageManager.createTextAtMessage(
  text: '@$atAllTag 请查看群公告',
  atUserIDList: [atAllTag],
  atUserInfoList: [
    AtUserInfo(atUserID: atAllTag, groupNickname: '全体成员'),
  ],
);
```

客户端应根据群组配置和当前角色决定是否展示入口，服务端仍会做最终权限校验。

## 发送带引用的 @ 消息

需要回复一条已有消息时，把当前会话列表、历史查询或消息事件提供的完整 `Message` 传给 `quoteMessage`，不要手工拼接不完整对象。

```dart
final reply = await OpenIM.iMManager.messageManager.createTextAtMessage(
  text: '@$memberUserID 我回复的是这条消息',
  atUserIDList: [memberUserID],
  atUserInfoList: [
    AtUserInfo(atUserID: memberUserID, groupNickname: member.nickname),
  ],
  quoteMessage: quotedMessage,
);
```

## 结果与提醒状态

`createTextAtMessage()` 的 Future 只返回待发送的 `Message`，不会触发新消息 listener。`sendMessage()` 成功后，发送端直接使用返回的 `Message` 按非空 `clientMsgID` 合并；接收端从新消息 listener 得到 @ 消息。完整接收、离线同步和去重流程见[接收消息](/sdk/flutter/message/receiving-messages/receive-messages)。

会话中的 `groupAtType` 只描述当前用户是否有未处理的 @ 提醒，不参与 @ 消息创建。用户进入会话并处理提醒后，应通过会话状态 API 清理对应状态；Future 成功、会话事件到达与重新查询校准是不同阶段。会话状态处理见[管理会话已读状态](/sdk/flutter/conversation/managing-conversations/manage-read-status)。

## 限制

单条消息的提及人数、全体提醒权限和目标范围最终以固定 SDK 底层及服务端校验为准。需要提醒大量成员时，优先使用经过权限控制的全体提醒或服务端通知方案，不要在客户端无限展开成员 ID。
