固定 Flutter SDK 的 `typingStatusUpdate()` 已废弃且会抛出 `UnimplementedError`，不要调用。输入框获得或失去焦点时使用 `ConversationManager.changeInputStates()`。

输入状态绑定到 `conversationID`，不会写入消息列表，也不用于保存草稿文本。当前用户开始输入时发送 `focus: true`；发送消息、输入框失焦、切换会话、页面离开或一段时间没有键盘输入时发送 `focus: false`。

```dart
await OpenIM.iMManager.conversationManager.changeInputStates(
  conversationID: conversationID,
  focus: true,
);

await OpenIM.iMManager.conversationManager.changeInputStates(
  conversationID: conversationID,
  focus: false,
);
```

`conversationID` 是当前会话，`focus` 表示当前用户是否处于输入焦点状态。Future 成功只代表状态请求完成，不等于对方已经收到回调。

实际界面中应对 `focus: true` 做节流或状态去重，例如只在“未输入”切换为“输入中”时调用一次。不要在每个键盘事件中重复上报；停止输入时则应确保最终发送一次 `focus: false`。

## 查询当前状态

需要主动读取指定用户的当前输入状态时：

```dart
final states = await OpenIM.iMManager.conversationManager.getInputStates(
  conversationID,
  otherUserID,
);
```

返回 `List<int>?`，内容是该用户当前仍处于输入状态的平台 ID 快照；列表非空时展示“正在输入”，空值或空列表按未输入降级处理。查询只建立快照，不触发 listener。

进入会话、连接恢复或需要补偿状态时，可以对重点成员调用 `getInputStates()` 校准。群聊需要展示多名成员输入状态时，应以 listener 维护当前会话的输入用户集合，避免频繁轮询所有成员。

## 同步输入状态变化

```dart
void handleInputStatusChanged(InputStatusChangedData data) {
  updateTypingState(data.conversationID, data.userID, data);
}
```

`onInputStatusChanged` 的 `platformIDs` 非空表示该用户仍在输入。该函数应纳入应用唯一的 `OnConversationListener`，按 `conversationID:userID` 覆盖平台快照，并设置业务超时避免网络中断后长期显示；不要按事件顺序累计次数。唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。Future 成功、对方 listener 到达与之后查询校准是独立阶段。
