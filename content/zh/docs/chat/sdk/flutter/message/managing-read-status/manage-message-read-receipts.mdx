OpenIMSDK 的单聊已读回执用于判断对方是否已经阅读某些消息。固定 Flutter SDK 没有群消息成员级回执能力，因此本页只记录单聊回执与会话未读数处理，不把会话已读错误描述为群成员阅读统计。

## 上报单聊已读回执

用户实际打开单聊并阅读完可见消息后，调用会话级已读 API。不要在后台收到通知、只显示通知预览或消息仍未展示时提前上报。

`conversationID` 是唯一必填参数，必须传单聊会话 ID；该方法不接收消息 ID。

```dart
await OpenIM.iMManager.conversationManager.markConversationMessageAsRead(
  conversationID: conversationID,
);
```

SDK 会根据该单聊会话中由对方发送的未读消息生成已读上报，同时清理当前用户的会话未读数。Future 成功表示本次已读处理完成，不代表消息发送方的界面已经更新；发送方通过 `onRecvC2CReadReceipt` 接收回执。

会话未读数的初始化、事件合并及全部标记已读见[管理会话已读状态](/sdk/flutter/conversation/managing-conversations/manage-read-status)。

## 接收单聊回执

```dart
Future<void> handleC2CReadReceipts(
  List<ReadReceiptInfo> receipts,
) async {
  for (final receipt in receipts) {
    final peerUserID = receipt.userID;
    if (peerUserID == null || peerUserID.isEmpty) continue;
    if (receipt.sessionType != null &&
        receipt.sessionType != ConversationType.single) continue;

    final value = await OpenIM.iMManager.conversationManager
        .getConversationIDBySessionType(
          sourceID: peerUserID,
          sessionType: ConversationType.single,
        );
    final conversationID = value?.toString();
    if (conversationID == null || conversationID.isEmpty) continue;

    for (final clientMsgID in receipt.msgIDList ?? const <String>[]) {
      if (clientMsgID.isEmpty) continue;
      markPeerRead(conversationID, clientMsgID, receipt.readTime);
    }
  }
}
```

`onRecvC2CReadReceipt` 携带 `List<ReadReceiptInfo>`。`userID`、`sessionType`、`msgIDList` 和 `readTime` 都来自 nullable model；先验证回执属于单聊，再用对端用户 ID 换取 `conversationID`，最后按每个非空 `clientMsgID` 更新对端已读状态。该处理函数应纳入唯一的 `OnAdvancedMsgListener`；设置入口见[事件概览](/sdk/flutter/events/overview-events)。

固定 SDK 的 `ReadReceiptInfo` 字段如下，所有字段都可能为 `null`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `userID` | `String?` | 单聊对端用户 ID，用于解析目标会话。 |
| `groupID` | `String?` | 群组 ID；本页单聊处理不使用该字段。 |
| `msgIDList` | `List<String>?` | 已读消息的 `clientMsgID` 列表。 |
| `readTime` | `int?` | 本次回执的阅读时间。 |
| `msgFrom` | `int?` | 消息来源值。 |
| `contentType` | `int?` | 消息内容类型值。 |
| `sessionType` | `int?` | 会话类型；单聊应为 `ConversationType.single`。 |

`ReadReceiptInfo` 是单聊回执结构，不包含群成员列表、已读人数或未读人数。不能把它当作 WASM 的群回执结构处理，也不能按数组位置关联消息。

## 群聊能力边界

固定 Flutter SDK 中 `markMessagesAsReadByMsgID()` 已废弃，不写入推荐流程；同时没有 WASM 的群消息读回执上报、已读／未读成员查询和群回执 listener。群聊的会话未读数仍可通过 `markConversationMessageAsRead()` 清理，但不能据此推断每条群消息的成员级阅读情况。已读 Future、发送方收到回执和重新进入会话后的快照校准应分别处理。

会话未读数与消息回执是两类状态：前者表示当前用户是否还有未读会话内容，后者表示消息发送方是否收到对端阅读增量。重新进入单聊时，以本地消息状态建立快照；在线期间再按 `conversationID + clientMsgID` 合并回执 callback。
