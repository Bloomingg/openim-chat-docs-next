发送前先创建 `Message`。单聊填写 `userID`，群聊填写 `groupID`；不要把 `conversationID` 当作发送目标。调用前应完成 SDK 初始化和登录。

## 消息类型

| 消息类型 | 创建方法 | 说明 |
| --- | --- | --- |
| 文本 | `createTextMessage()` | 纯文本内容。 |
| @ 文本 | `createTextAtMessage()` | 群聊提醒与可选引用。 |
| 图片、音频、视频、文件 | 对应 `create*Message()` 或 `create*MessageFromFullPath()` | 由 SDK 处理本地媒体上传。 |
| URL 媒体 | 对应 `create*MessageByURL()` | 业务已完成上传，随后使用 `sendMessageNotOss()`。 |
| 自定义、引用、合并、位置、名片和表情 | 对应消息工厂 | 承载结构化或富内容。 |

Android 与 iOS 都使用同一组 Dart 方法，但路径必须来自当前平台可读取的应用目录。不要把 Android 文件 URI、iOS Photos 临时标识或其他平台沙盒路径直接传给 SDK；跨平台选择文件后，应先取得当前平台可访问的本地路径。

## 发送单聊消息

```dart
final message = await OpenIM.iMManager.messageManager.createTextMessage(
  text: '你好，OpenIMSDK',
);

final sentMessage = await OpenIM.iMManager.messageManager.sendMessage(
  message: message,
  userID: receiverUserID,
  offlinePushInfo: OfflinePushInfo(title: '新消息', desc: '你收到一条消息'),
);

final clientMsgID = sentMessage.clientMsgID;
if (clientMsgID == null || clientMsgID.isEmpty) {
  throw StateError('Sent message is missing clientMsgID');
}
replacePendingMessage(clientMsgID, sentMessage);
```

Future 成功返回服务端确认后的 `Message`。发送端在当前会话上下文中按非空 `clientMsgID` 替换待发送项，从而更新状态、服务端 ID 和时间。

## 发送群聊消息

群聊发送时改传 `groupID`：

```dart
final message = await OpenIM.iMManager.messageManager
    .createTextMessage(text: '大家好');
final sentMessage = await OpenIM.iMManager.messageManager.sendMessage(
  message: message,
  groupID: groupID,
  offlinePushInfo: OfflinePushInfo(title: '群消息', desc: '大家好'),
);
```

## 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `message` | `Message` | 必填 | 消息工厂返回的待发送对象。 |
| `offlinePushInfo` | `OfflinePushInfo` | 必填 | 当前消息的离线推送标题、摘要和扩展数据。 |
| `userID` | `String?` | 条件必填 | 单聊接收方；群聊时不传。 |
| `groupID` | `String?` | 条件必填 | 群聊目标；单聊时不传。 |
| `isOnlineOnly` | `bool` | 可选 | 为 `true` 时只投递在线端，不写入历史，由 `onRecvOnlineOnlyMessage` 接收。 |

`userID` 与 `groupID` 二选一。`isOnlineOnly: true` 的消息只向在线客户端投递，不写入本地历史，因此不能依赖历史查询恢复。

## 文件和自定义消息

本地图片、音频、视频和文件遵循“先创建、再发送”的流程，创建参数与上传进度见[创建媒体与富消息](/sdk/flutter/message/sending-messages/create-media-and-rich-messages)和[上传文件并跟踪进度](/sdk/flutter/message/sending-messages/upload-files-and-track-progress)。自定义消息的数据结构与扩展字段见[自定义消息与扩展数据](/sdk/flutter/message/composing-messages/custom-message-and-extra-data)。

业务已经上传媒体并取得 URL 时，使用对应 `create*MessageByURL()`，再调用 `sendMessageNotOss()`；它是绕过 SDK OSS 上传的发送入口，不是新的消息类型。固定 Flutter SDK 没有 `createTargetedGroupMessage()`，不要模拟定向群消息。

## 调用结果与事件反馈

消息工厂 Future 只返回待发送对象，不触发新消息事件。`sendMessage()` 或 `sendMessageNotOss()` Future 成功表示服务端已确认发送，应使用返回 `Message` 更新发送端状态；它不等同于其他客户端已收到事件。

接收端根据消息路由字段解析目标会话，并按“目标 `conversationID` + `clientMsgID`”合并。普通消息与仅在线消息的完整处理见[接收消息](/sdk/flutter/message/receiving-messages/receive-messages)。调用失败时记录错误、当前用户和目标用户或群组 ID，不要记录 Token、完整正文或媒体 URL。
