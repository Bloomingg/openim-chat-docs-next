消息工厂只创建 `Message`，不会发送。使用移动端可访问的本地路径创建媒体对象，再交给 `sendMessage()`；完整路径场景可选用对应的 `*FromFullPath()` 方法。

先根据资源来源选择创建方式：资源仍在当前设备时，使用本地路径工厂并由普通发送流程上传；资源已经通过业务上传服务取得可访问 URL 时，使用 `create*MessageByURL()` 和 `sendMessageNotOss()`。两种流程都必须使用真实的文件大小、类型、尺寸和时长，不能用占位值满足参数要求。

## 支持的方法

| 方法 | 用途 |
| --- | --- |
| `createImageMessage()` / `createImageMessageFromFullPath()` | 从当前设备路径创建图片消息。 |
| `createSoundMessage()` / `createSoundMessageFromFullPath()` | 从当前设备路径创建音频消息。 |
| `createVideoMessage()` / `createVideoMessageFromFullPath()` | 从当前设备路径创建视频消息。 |
| `createFileMessage()` / `createFileMessageFromFullPath()` | 从当前设备路径创建文件消息。 |
| `createImageMessageByURL()` | 使用原图、大图和缩略图信息创建已上传图片消息。 |
| `createSoundMessageByURL()` | 使用 `SoundElem` 创建已上传音频消息。 |
| `createVideoMessageByURL()` | 使用 `VideoElem` 创建已上传视频消息。 |
| `createFileMessageByURL()` | 使用 `FileElem` 创建已上传文件消息。 |
| `createLocationMessage()` / `createCardMessage()` / `createFaceMessage()` | 创建位置、名片和表情消息。 |
| `createQuoteMessage()` / `createAdvancedQuoteMessage()` | 创建引用或富文本引用消息。 |
| `createAdvancedTextMessage()` | 创建带 `RichMessageInfo` 的富文本消息。 |

## 本地媒体文件

图片使用 `createImageMessage()`，只传当前平台可读的 `imagePath`。音频、视频和文件的参数如下；对应的 `*FromFullPath()` 接收相同字段，用于已经取得绝对路径的场景。

### 创建图片消息

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `imagePath` | `String` | 必填 | 当前平台可读的图片路径。 |

### 创建音频消息

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `soundPath` | `String` | 必填 | 当前平台可读的音频路径。 |
| `duration` | `int` | 必填 | 音频时长，单位为秒。 |

### 创建视频消息

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `videoPath` | `String` | 必填 | 当前平台可读的视频路径。 |
| `videoType` | `String` | 必填 | 视频 MIME 类型，例如 `video/mp4`。 |
| `duration` | `int` | 必填 | 视频时长，单位为秒。 |
| `snapshotPath` | `String` | 必填 | 当前平台可读的预览图路径。 |

### 创建文件消息

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `filePath` | `String` | 必填 | 当前平台可读的文件路径。 |
| `fileName` | `String` | 必填 | 接收端展示的文件名。 |

```dart
final image = await OpenIM.iMManager.messageManager
    .createImageMessage(imagePath: imagePath);
final sound = await OpenIM.iMManager.messageManager.createSoundMessage(
  soundPath: soundPath,
  duration: durationSeconds,
);
final video = await OpenIM.iMManager.messageManager.createVideoMessage(
  videoPath: videoPath,
  videoType: 'video/mp4',
  duration: durationSeconds,
  snapshotPath: snapshotPath,
);
final file = await OpenIM.iMManager.messageManager.createFileMessage(
  filePath: filePath,
  fileName: fileName,
);
```

音频和视频 `duration` 的单位是秒。视频同时需要 MIME 类型和预览图路径；文件名用于接收端展示。路径必须在当前平台可读，调用前应处理相册或文件权限。

## 富消息

### 创建位置消息

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `latitude` | `double` | 必填 | 纬度。 |
| `longitude` | `double` | 必填 | 经度。 |
| `description` | `String` | 必填 | 位置描述。 |

### 创建名片消息

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `userID` | `String` | 必填 | 名片对应的用户 ID。 |
| `nickname` | `String` | 必填 | 名片展示名称。 |
| `faceURL` | `String?` | 可选 | 名片头像地址。 |
| `ex` | `String?` | 可选 | 名片扩展字段的完整值。 |

`createFaceMessage()` 的 `index` 默认 `-1`，`data` 可保存网络表情地址。

```dart
final location = await OpenIM.iMManager.messageManager.createLocationMessage(
  latitude: 25.033,
  longitude: 121.5654,
  description: '台北 101',
);
final card = await OpenIM.iMManager.messageManager.createCardMessage(
  userID: userID,
  nickname: nickname,
  faceURL: faceURL,
);
final face = await OpenIM.iMManager.messageManager.createFaceMessage(
  index: 3,
  data: emojiURL,
);
```

还可以使用 `createQuoteMessage(text:, quoteMsg:)` 创建引用消息。`quoteMsg` 应直接使用当前列表或历史查询得到的原始 `Message`，不要手工拼接字段。使用 `createAdvancedTextMessage(text:, list:)` 和 `createAdvancedQuoteMessage(text:, quoteMsg:, list:)` 创建富文本时，`list` 中每个 `RichMessageInfo` 可描述 `type`、`offset`、`length`、`url` 和 `info`；偏移与长度必须和最终正文一致。它们都返回待发送 `Message`，需要再调用发送 API。

## 已上传媒体

自定义上传完成后，使用 URL 型工厂创建对象，再调用 `sendMessageNotOss()`。不要把本地路径冒充远端 URL。

### URL 图片

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `sourcePicture` | `PictureInfo` | 必填 | 原图信息。 |
| `bigPicture` | `PictureInfo` | 必填 | 大图信息。 |
| `snapshotPicture` | `PictureInfo` | 必填 | 缩略图信息。 |
| `sourcePath` | `String?` | 可选 | 需要保留在消息中的本地原图路径。 |

每个 `PictureInfo` 可填写 `uuid`、MIME `type`、`size`、`width`、`height` 和 `url`；三种规格都应来自真实上传或图片处理结果。

### URL 音频

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `soundElem` | `SoundElem` | 必填 | 已上传音频的完整元素。 |

用于远端音频时，`soundElem` 应提供 `uuid`、`sourceUrl`、`dataSize` 和秒级 `duration`；`soundPath` 只用于保留本地路径。

### URL 视频

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `videoElem` | `VideoElem` | 必填 | 已上传视频及预览图的完整元素。 |

`videoElem` 应提供视频的 `videoUUID`、`videoUrl`、`videoType`、`videoSize`、秒级 `duration`，以及预览图的 `snapshotUUID`、`snapshotSize`、`snapshotUrl`、`snapshotWidth`、`snapshotHeight`。缺少预览元数据会影响接收端稳定展示。

### URL 文件

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `fileElem` | `FileElem` | 必填 | 已上传文件的完整元素。 |

用于远端文件时，`fileElem` 应提供 `uuid`、`sourceUrl`、`fileName` 和 `fileSize`；`filePath` 只用于保留本地路径。

`sendMessageNotOss()` 与普通发送一样要求 `message` 和 `offlinePushInfo`，单聊传 `userID`，群聊传 `groupID`。URL 型消息中的地址必须在接收端可访问；不要把本地路径冒充远端 URL，也不要只传媒体 URL 而省略接收端展示所需的元数据。

## 调用结果与消息接收

各消息工厂的 Future 成功只代表待发送的 `Message` 创建完成，不代表消息已经发送，也不会触发新消息 listener。调用端可以在发送前继续检查消息内容或补充扩展字段。

`sendMessage()` 或 `sendMessageNotOss()` 的 Future 成功后，发送端使用返回的 `Message` 按非空 `clientMsgID` 合并到目标会话列表。其他客户端通过新消息 listener 接收消息；Future 成功、listener 到达和重新查询校准是三个不同阶段。完整的接收、会话路由和去重方式见[接收消息](/sdk/flutter/message/receiving-messages/receive-messages)。
