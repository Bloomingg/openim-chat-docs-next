转发不会修改原消息。先基于已发送的 `Message` 创建新的待发送对象，再发送到目标单聊或群聊；不要复用原消息的 `clientMsgID`。

## 转发单条消息

文本、引用、表情、图片、音视频和文件等消息使用相同流程。传给 `createForwardMessage()` 的原消息必须已经发送成功；发送中或发送失败的消息不能转发。来源可以是当前消息列表、历史查询结果或新消息 callback。

```dart
final forward = await OpenIM.iMManager.messageManager
    .createForwardMessage(message: sourceMessage);
final sent = await OpenIM.iMManager.messageManager.sendMessage(
  message: forward,
  userID: targetUserID,
  offlinePushInfo: OfflinePushInfo(title: '转发消息', desc: '你收到一条消息'),
);
```

转发到群聊时改为传入目标 `groupID`。无论原消息是什么类型，都要发送 `createForwardMessage()` 返回的新 `Message`，不要把原消息对象直接插入目标会话。重新上传本地文件属于创建并发送新的文件消息，不属于转发。

## 合并多条消息

多选转发或聊天记录分享使用 `createMergerMessage()`：

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `messageList` | `List<Message>` | 必填 | 需要合并展示的已发送消息。 |
| `title` | `String` | 必填 | 合并消息标题。 |
| `summaryList` | `List<String>` | 必填 | 用于消息气泡预览的安全摘要。 |

```dart
final merger = await OpenIM.iMManager.messageManager.createMergerMessage(
  messageList: selectedMessages,
  title: '聊天记录',
  summaryList: selectedMessages.map(buildMessageSummary).toList(),
);
await OpenIM.iMManager.messageManager.sendMessage(
  message: merger,
  groupID: targetGroupID,
  offlinePushInfo: OfflinePushInfo(title: '聊天记录', desc: '查看合并消息'),
);
```

`messageList` 应只包含可转发的已发送消息；`summaryList` 应按内容类型生成安全文本，不要直接展示序列化内部内容。`createForwardMessage()` 只接收一个完整 `message`；来源可以是消息列表、历史查询或新消息回调。

## 调用结果与消息接收

两个创建 Future 只返回新的待发送 `Message`，不会修改原消息，也不会触发新消息 listener。`sendMessage()` 成功后，发送端使用返回对象的新 `clientMsgID` 合并目标会话；其他客户端再通过通用新消息 listener 接收。Future 成功、远端 listener 到达与重新查询校准是三个独立阶段。

调用失败时，记录错误 `code`、`message` 和目标用户或群组 ID，并确认原消息发送状态以及当前用户在目标群聊的发言权限。不要因发送失败把待发送对象当成已成功消息插入列表。
