消息置顶用于固定展示会话中的重要消息。它与会话列表置顶不是同一个状态：消息置顶使用 `setConversationPinnedMsg()`，会话列表置顶使用 `setConversation()` 修改 `isPinned`。

## 获取置顶消息

`getConversationPinnedMsg()` 接收 `conversationID`，返回 `List<Message>`：

```dart
final pinnedMessages = await OpenIM.iMManager.messageManager
    .getConversationPinnedMsg(conversationID: conversationID);

replacePinnedMessages(conversationID, pinnedMessages);
```

Future 成功后直接使用返回列表建立该会话的置顶消息快照；查询本身不会触发 `onChangedPinnedMsg`。列表元素仍按非空 `clientMsgID` 去重。

Flutter 查询结果只有完整 `Message`，不包含置顶时间或操作者。需要展示 `pinnedTime` 与 `userID` 时，应使用后续事件携带的 `ConversationPinnedMsg`，不能从查询列表推测这些值。消息被删除、撤回或修改后，也应结合对应消息事件更新展示，不要永久缓存初次查询结果。

## 置顶或取消置顶

把 `pinned` 设为 `true` 表示置顶，设为 `false` 表示取消置顶：

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `conversationID` | `String` | 必填 | 消息所属的会话 ID。 |
| `clientMsgID` | `String` | 必填 | 要置顶或取消置顶的消息稳定 ID。 |
| `pinned` | `bool` | 必填 | `true` 表示置顶，`false` 表示取消置顶。 |

```dart
final clientMsgID = message.clientMsgID;
if (clientMsgID == null || clientMsgID.isEmpty) {
  throw StateError('Message has no clientMsgID');
}

await OpenIM.iMManager.messageManager.setConversationPinnedMsg(
  conversationID: conversationID,
  clientMsgID: clientMsgID,
  pinned: true,
);

await OpenIM.iMManager.messageManager.setConversationPinnedMsg(
  conversationID: conversationID,
  clientMsgID: clientMsgID,
  pinned: false,
);
```

可置顶的消息类型、数量上限和操作权限由当前 OpenIMServer 配置决定。客户端可以根据已知权限隐藏入口，但不能用本地状态绕过服务端校验。

`setConversationPinnedMsg()` 返回未具体化的 `Future`。成功表示服务端已完成本次设置请求，不表示置顶事件已经到达，也不提供变化后的置顶列表。

## 监听变化

`onChangedPinnedMsg` 携带 `ChangedPinnedMsg`，其中 `conversationID` 标识要替换的会话，`msgs` 是变化后的置顶详情列表。每个 `ConversationPinnedMsg` 都包含操作用户 `userID`、置顶时间 `pinnedTime` 和完整消息 `msg`：

```dart
void handlePinnedMessagesChanged(ChangedPinnedMsg change) {
  final detailsByMessageID = <String, ConversationPinnedMsg>{};

  for (final detail in change.msgs) {
    final clientMsgID = detail.msg.clientMsgID;
    if (clientMsgID == null || clientMsgID.isEmpty) continue;
    detailsByMessageID[clientMsgID] = detail;
  }

  replacePinnedMessageDetails(
    change.conversationID,
    detailsByMessageID.values.toList(),
  );
}
```

本页是 `onChangedPinnedMsg` 的完整事件归属页。按 `conversationID` 替换对应会话的置顶集合，再按 `detail.msg.clientMsgID` 去重；不要把不同会话的列表合并，也不要用置顶时间或数组位置作为消息标识。

该 handler 应纳入应用唯一的 `OnAdvancedMsgListener`，集中设置入口见[事件概览](/sdk/flutter/events/overview-events)。固定 SDK 没有 remove 或 unset API；退出登录或切换账号时停止向旧账号状态分发，并用新账号的完整 listener 覆盖。

事件也可能来自其他设备或其他有权限的成员。Future 成功、`onChangedPinnedMsg` 到达和调用 `getConversationPinnedMsg()` 重新建立快照是三个不同阶段；重新登录或需要校准当前会话时，应再次查询置顶消息。
