删除只影响消息是否仍对当前账号可见，不等同于撤回，也不会让会话成员看到撤回提示。需要展示撤回状态时，使用[撤回消息](/sdk/flutter/message/managing-messages/revoke-a-message)。

## 仅删除本地消息

`conversationID` 由当前页面或路由上下文提供，`clientMsgID` 来自目标消息并应先检查非空。

```dart
await OpenIM.iMManager.messageManager.deleteMessageFromLocalStorage(
  conversationID: conversationID,
  clientMsgID: clientMsgID,
);
removeMessage(conversationID, clientMsgID);
```

服务端仍保留该消息；重建本地数据库并重新同步后可能再次出现。

该操作只修改当前设备的本地消息库，不会向当前账号的其他设备或会话成员同步。发送失败或尚未取得服务端序号的消息只能执行本地删除。

删除当前会话最后一条消息时，会话预览、排序和未读数可能随会话 callback 更新；完整合并方式见[获取会话列表](/sdk/flutter/conversation/retrieving-conversations/retrieve-conversation-list)。

## 删除本地与服务端消息

```dart
await OpenIM.iMManager.messageManager.deleteMessageFromLocalAndSvr(
  conversationID: conversationID,
  clientMsgID: clientMsgID,
);
```

Future 成功表示当前账号的本地与服务端消息删除请求已经完成。它不是撤回，也不会删除其他会话成员已经收到的消息；其他设备是否已经持有本地副本以及后续同步范围，应以固定 Flutter SDK 与服务端行为为准，不能推断为跨端强制删除。

固定 Flutter SDK 没有 WASM 的批量 `deleteMessages()`、同步开关或按用户删除会话全部消息 API。应用需要批量操作时只能逐条调用并分别处理失败，不能宣称原子完成或跨端同步范围。

## 处理删除事件

```dart
Future<void> handleMessageDeleted(
  Message message,
  String currentUserID,
) async {
  final clientMsgID = message.clientMsgID;
  if (clientMsgID == null || clientMsgID.isEmpty) return;

  final conversationID = await resolveMessageConversationID(
    message,
    currentUserID,
  );
  if (conversationID == null) return;

  removeMessage(conversationID, clientMsgID);
}
```

Flutter `Message` 没有 `conversationID` 字段。示例中的 `resolveMessageConversationID()` 根据 `sessionType`、`groupID`、`sendID`、`recvID` 和当前用户解析来源，再调用 `getConversationIDBySessionType()` 校准；完整 helper 见[接收消息](/sdk/flutter/message/receiving-messages/receive-messages)。

`onMsgDeleted` 是本页归属事件，应纳入唯一的 `OnAdvancedMsgListener`，按 `conversationID:clientMsgID` 幂等移除。唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。删除 Future 成功、事件到达和重新查询历史是独立阶段；失败时不要先永久移除界面记录。

事件也可能来自当前账号的其他客户端或其他有权限的操作。调用端可以在 Future 成功后先更新当前列表，随后将 `onMsgDeleted` 当作幂等增量处理；不能假设每个 callback 都只对应当前页面刚完成的调用。

## 删除结果

| 用户动作 | API | 结果 |
| --- | --- | --- |
| 仅从当前设备删除 | `deleteMessageFromLocalStorage()` | 服务端仍保留消息，重新建立本地数据库后可能再次同步。 |
| 删除当前账号的本地与服务端消息 | `deleteMessageFromLocalAndSvr()` | 删除当前账号保存的消息，不等同于撤回，也不保证移除其他成员或其他设备已经保存的副本。 |
| 让会话成员看到撤回状态 | `revokeMessage()` | 属于撤回流程，不应由删除 API 模拟。 |

调用失败时保留原消息，并记录错误对象、`conversationID` 和 `clientMsgID`；不要记录完整正文或敏感附件 URL。需要确认当前快照时重新读取历史，重新查询失败不应被误报为删除请求失败。
