description: '清空单个会话、当前设备或当前账号保存的消息记录。'

清空历史不会产生逐条撤回提示，也不能删除其他会话成员已经保存的副本。所有操作都具有破坏性，执行前应二次确认。

先确定产品操作针对单个会话，还是当前账号的全部会话。单个会话清空适用于单聊和群聊；账号级清空会覆盖当前账号的所有会话，不应作为单个聊天窗口中的普通“清空”按钮操作。

## 清空单个会话

无论会话类型如何，都直接使用 `ConversationInfo.conversationID`。如果业务入口只有用户 ID 或群组 ID，应先取得对应会话，再传入其中的 `conversationID`。

```dart
await OpenIM.iMManager.conversationManager
    .clearConversationAndDeleteAllMsg(
      conversationID: conversationID,
    );
clearConversationMessages(conversationID);
```

Future 成功后可以按 `conversationID` 清空当前窗口。该操作不会删除会话记录、解散群组，也不会清除单聊对方或其他群成员保存的消息。会话摘要、未读数与最后一条消息继续通过会话 listener 合并，也可以重新查询该会话校准。该 API 位于 `ConversationManager`，不是 `MessageManager`。

如果产品动作是“删除此聊天”，应改用删除会话并删除消息的会话 API，具体边界见[删除或清空会话](/sdk/flutter/conversation/managing-conversations/delete-or-clear-conversation)。清空成功后，新到达的消息仍会通过消息 listener 进入该会话，并成为新的历史起点。

以下两个 API 则影响当前账号的全部消息，不是单个会话操作；不要放在普通单条消息菜单中。

## 只清空本地消息

```dart
await OpenIM.iMManager.messageManager.deleteAllMsgFromLocal();
clearLocalMessageState();
```

服务端记录仍保留，重建本地数据库或重新同步后可能再次出现。

Future 成功后应清空当前账号在内存中的全部消息快照，而不只是当前打开的会话。此操作适合重置当前设备数据，不适合表达永久删除。

## 清空本地与服务端消息

```dart
await OpenIM.iMManager.messageManager.deleteAllMsgFromLocalAndSvr();
clearLocalMessageState();
```

该操作删除当前账号本地及服务端保存的消息，但不等同于逐条撤回，也不能承诺删除其他会话成员已经收到的副本。Future 成功表示当前请求完成，应重置全部内存消息快照；会话预览、未读数和其他端状态仍应通过会话 listener 与重新查询校准。账号级 API 没有独立的逐条消息删除事件，不能把当前端 Future 成功当作其他设备已经清理完成。会话管理边界见[删除或清空会话](/sdk/flutter/conversation/managing-conversations/delete-or-clear-conversation)。
