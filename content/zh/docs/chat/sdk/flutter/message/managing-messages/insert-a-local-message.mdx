本地插入适合迁移记录、系统占位或业务导入。它不会向目标用户或群组发送消息，也不会触发远端新消息事件；不要用它模拟服务端消息。

## 单聊本地插入

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `receiverID` | `String?` | SDK 可选，业务必填 | 单聊接收方用户 ID。 |
| `senderID` | `String?` | SDK 可选，业务必填 | 单聊发送方用户 ID。 |
| `message` | `Message?` | SDK 可选，业务必填 | 要写入当前设备的完整消息对象。 |

```dart
final localMessage = await OpenIM.iMManager.messageManager
    .createTextMessage(text: '仅保存在本机的记录');

final inserted = await OpenIM.iMManager.messageManager
    .insertSingleMessageToLocalStorage(
      receiverID: receiverUserID,
      senderID: senderUserID,
      message: localMessage,
    );
```

## 群聊本地插入

群聊使用群组入口：

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `String?` | SDK 可选，业务必填 | 目标群组 ID。 |
| `senderID` | `String?` | SDK 可选，业务必填 | 消息发送方用户 ID。 |
| `message` | `Message?` | SDK 可选，业务必填 | 要写入当前设备的完整消息对象。 |

```dart
final inserted = await OpenIM.iMManager.messageManager
    .insertGroupMessageToLocalStorage(
      groupID: groupID,
      senderID: senderUserID,
      message: localMessage,
    );
```

两个 API 的参数在 Dart 声明中都是 nullable，但底层操作需要完整的目标、发送者和消息；业务调用前应全部校验。Future 成功返回写入后的 `Message`。Flutter `Message` 没有 `conversationID` 字段，应用应使用当前页面已有的会话 ID，并检查返回消息的 nullable `clientMsgID` 后合并列表，也可以重新读取对应会话历史校准。重新安装或清除本地数据后，此记录不会从服务端恢复。
