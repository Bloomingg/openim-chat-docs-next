OpenIMClientSDK 通过订阅机制同步用户在线状态。建议只订阅会话列表当前页、联系人当前页或资料卡中可见的用户。

在线状态只表示用户是否连接到 OpenIMServer。好友关系、群成员状态、禁言状态和应用自己的活跃时间，应使用各自的数据源。

## 订阅可见用户的状态

```dart
Future<List<UserStatusInfo>> subscribeVisibleUserStatus(
  Iterable<String> userIDs,
) async {
  final targets = userIDs.where((id) => id.isNotEmpty).toSet().toList();
  if (targets.isEmpty) return [];

  final states = await OpenIM.iMManager.userManager
      .subscribeUsersStatus(targets);
  mergePresence(states);
  return states;
}
```

Future 成功后返回 `List<UserStatusInfo>`，先用它建立快照；后续变化通过 `onUserStatusChanged` 到达。每项使用 `userID` 合并，`status` 表示在线状态，`platformIDs` 表示在线端平台列表。

`platformIDs` 可能为空；即使没有具体平台信息，界面也应根据 `status` 正常展示在线状态，不要把空平台列表直接解释成查询失败。

分页或搜索结果变化时，只订阅新进入可见范围的用户，并取消已经离开范围的用户。不要一次订阅整个用户目录。

## 监听状态变化

本页归属 `onUserStatusChanged` callback。它应与当前用户资料 callback 合并进一个应用级 `OnUserListener`。

```dart
void handleUserStatusChanged(UserStatusInfo state) {
  final userID = state.userID;
  if (userID == null) return;
  presenceByUserID[userID] = state;
  renderPresence(userID, state.status);
}

```

组合 listener 与唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。固定版本没有公开移除方法；退出时应隔离旧账号状态。

## 查询已订阅用户状态

```dart
final states = await OpenIM.iMManager.userManager
    .getSubscribeUsersStatus();
mergePresence(states);
```

`getSubscribeUsersStatus()` 返回调用时的订阅状态快照，不会触发 callback。若需要查询一组指定用户但不改变长期订阅集合，可使用 `getUserStatus(userIDs)`。

三种方法的用途不同：

| 场景 | 方法 | 后续增量 |
| --- | --- | --- |
| 指定一组可见用户并持续追踪 | `subscribeUsersStatus(userIDs)` | 通过 `onUserStatusChanged` 更新。 |
| 断线重连或重进页面后刷新既有订阅 | `getSubscribeUsersStatus()` | 不改变订阅集合。 |
| 临时查询指定用户 | `getUserStatus(userIDs)` | 不应假定已建立长期订阅。 |

这些 Future 成功只表示取得调用时的状态快照，不代表 callback 已经到达。快照与 callback 都按 `userID` 合并，不要按列表位置覆盖。

## 取消订阅

```dart
Future<void> unsubscribeUserStatus(List<String> userIDs) async {
  final targets = userIDs.where((id) => id.isNotEmpty).toSet().toList();
  if (targets.isEmpty) return;

  await OpenIM.iMManager.userManager.unsubscribeUsersStatus(targets);
  for (final userID in targets) {
    presenceByUserID.remove(userID);
  }
}
```

Future 成功后，从应用自己的订阅集合和状态缓存移除这些 `userID`。后续不再依赖它们的增量 callback 更新界面。

用户离开页面、列表翻页或搜索结果变化时，应取消已经离开可见范围的订阅。退出登录或切换账号时，还要清空旧账号的订阅集合和状态缓存；固定版本虽没有公开移除 `OnUserListener` 的方法，旧 callback 也必须由应用级分发器按当前账号隔离。
