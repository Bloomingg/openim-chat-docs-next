好友列表可用于通讯录、联系人选择器和好友关系缓存。所有场景都应使用 `getFriendListPage()` 分页读取 `FriendInfo`，不要使用非分页的 `getFriendList()`。

## 分页读取好友列表

### 参数说明

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `offset` | `int` | 起始偏移量，第一页传 `0`。 |
| `count` | `int` | 本次读取数量，默认 `40`。 |
| `filterBlack` | `bool` | 是否过滤黑名单中的用户，默认 `false`。 |

```dart
const pageSize = 40;

final friends = await OpenIM.iMManager.friendshipManager
    .getFriendListPage(
  offset: 0,
  count: pageSize,
  filterBlack: true,
);

final rows = friends.map((friend) => (
  userID: friend.userID,
  displayName: friend.remark?.isNotEmpty == true
      ? friend.remark
      : friend.nickname ?? friend.userID,
  avatar: friend.faceURL,
)).toList();
```

### 返回结果

Future 成功后返回当前页 `List<FriendInfo>`。常用字段包括 `userID`、`nickname`、`faceURL`、`remark`、`ex` 和 `createTime`。`FriendInfo` 在固定版本还没有公开 `isPinned` 字段；置顶设置虽可通过 `UpdateFriendsReq.isPinned` 写入，但此模型不能直接用该字段渲染列表。

加载下一页时，把 `offset` 设为当前已加载数量。返回数量小于 `count` 时，通常表示已经到末页。使用 `userID` 去重和合并，不要使用数组下标。

## 搜索好友

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `keywordList` | `List<String>` | 必填 | 搜索关键词；当前只传一个去除首尾空白后的非空关键词。 |
| `isSearchUserID` | `bool` | 必填 | 是否匹配好友 `userID`。 |
| `isSearchNickname` | `bool` | 必填 | 是否匹配好友账号级昵称。 |
| `isSearchRemark` | `bool` | 必填 | 是否匹配当前用户设置的好友备注。 |

```dart
Future<List<SearchFriendsInfo>> searchFriends(String keyword) async {
  final value = keyword.trim();
  if (value.isEmpty) return [];

  return OpenIM.iMManager.friendshipManager.searchFriends(
    keywordList: [value],
    isSearchUserID: true,
    isSearchNickname: true,
    isSearchRemark: true,
  );
}
```

当前只支持一个非空关键词。Future 返回 `List<SearchFriendsInfo>`；每项继承 `FriendInfo`，并增加 `relationship`。展示名称时优先使用非空 `remark`，再回退到 `nickname` 和 `userID`，与好友列表保持一致。

如果要搜索并添加新好友，通常先通过业务后端或精确的 `userID` 找到候选用户，再通过[获取指定用户资料](/sdk/flutter/user/retrieving-users/retrieve-users)展示公开资料，最后进入好友申请流程。

## 列表如何更新

第一页以返回结果建立快照，加载更多时按 `userID` 追加。好友新增、删除或资料变化的 listener 由[更新或删除好友](/sdk/flutter/user/managing-friends/update-or-delete-friends)归属。本页查询不会触发好友事件。

`searchFriends()` 返回的是当前关键词的搜索结果，只用于搜索界面，不要用它覆盖完整的好友列表快照。查询 Future、好友 listener 和重新分页是三个不同阶段。

列表仍在分页时，如果好友数量发生变化，应清空分页状态并从第一页重载，避免旧偏移跳过或重复记录。
