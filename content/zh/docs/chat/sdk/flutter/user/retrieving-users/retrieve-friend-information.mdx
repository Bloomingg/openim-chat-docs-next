已经知道一个或多个好友的 `userID`，只需刷新这些好友的关系资料时，使用 `getFriendsInfo()`。该接口适合好友资料卡、联系人详情和好友设置，无需重新获取整页列表。

## 获取指定好友资料

`userIDList` 是要查询的好友用户 ID 列表；`filterBlack` 决定是否过滤当前用户黑名单中的用户，普通好友资料页通常传 `true`。

```dart
final friends = await OpenIM.iMManager.friendshipManager.getFriendsInfo(
  userIDList: ['user_a', 'user_b'],
  filterBlack: true,
);

final friendsByID = {
  for (final friend in friends)
    if (friend.userID != null) friend.userID!: friend,
};
```

Future 成功后返回匹配到的 `List<FriendInfo>`。固定 SDK 中这些属性均为 nullable，消费端应先检查 `userID`，不要用昵称或数组位置作为合并标识。

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `userID` | `String?` | 好友用户 ID，也是查询结果的稳定合并标识。 |
| `ownerUserID` | `String?` | 当前好友关系所属的用户 ID。 |
| `friendUserID` | `String?` | 好友关系中的目标用户 ID。 |
| `nickname` | `String?` | 好友账号昵称。 |
| `faceURL` | `String?` | 好友头像地址。 |
| `remark` | `String?` | 当前用户为好友设置的备注。 |
| `createTime` | `int?` | 好友关系创建时间。 |
| `addSource` | `int?` | 建立好友关系的来源。 |
| `operatorUserID` | `String?` | 促成好友关系的操作用户 ID。 |
| `ex` | `String?` | 好友关系扩展字段。 |

返回为空通常表示目标不是好友，或已经被 `filterBlack` 过滤。

需要展示陌生人或好友候选人的账号级资料时，使用[获取指定用户资料](/sdk/flutter/user/retrieving-users/retrieve-users)中的 `getUsersInfo()`。

## 检查好友关系

```dart
final checks = await OpenIM.iMManager.friendshipManager.checkFriend(
  userIDList: ['user_a', 'user_b'],
);

final relationshipByID = {
  for (final item in checks)
    if (item.userID != null) item.userID!: item.result,
};
```

Future 返回 `List<FriendshipInfo>`。每项的 `userID` 是目标用户；`result` 为 `1` 表示目标是好友并且不在黑名单中，其他结果按非好友处理。不要把结果数字自行扩展为 SDK 未声明的额外状态；需要区分黑名单时另行查询黑名单。

## 调用结果与关系变化

两个方法都只读取当前快照，不会触发好友 listener。建立、删除关系或修改好友资料后的增量更新由[更新或删除好友](/sdk/flutter/user/managing-friends/update-or-delete-friends)处理。页面首次打开或用户主动刷新时，再调用查询校准目标用户状态。
