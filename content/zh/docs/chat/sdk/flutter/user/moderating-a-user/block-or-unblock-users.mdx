OpenIMSDK 黑名单是当前用户主动维护的单向关系。调用 `addBlacklist()` 将目标用户加入黑名单，调用 `removeBlacklist()` 将其移出黑名单。该能力可用于屏蔽用户、限制聊天入口，以及展示资料卡中的关系状态。

## 关系边界

黑名单只表示当前用户与目标用户之间的屏蔽关系。群管理员禁言或移除成员，以及平台封禁账号，属于不同的管理能力，应分别通过群成员 API 或可信后端实现。

黑名单对消息发送的限制是非对称的：拉黑后，对方不能向当前用户发送消息，但当前用户仍可向对方发送消息。如果产品要求双方都不能继续发送消息，还需要在界面和业务层限制当前用户的发送入口。

| 需求 | OpenIMSDK 对应能力 |
| --- | --- |
| 当前用户屏蔽另一个用户 | `addBlacklist(userID: ...)` |
| 当前用户解除屏蔽 | `removeBlacklist(userID: ...)` |
| 查看当前用户黑名单 | `getBlacklist()` |
| 禁言某个群成员 | 使用 `changeGroupMemberMute()`，见[禁言群组或群成员](/sdk/flutter/group/moderating-groups/mute-a-group-or-member)。 |
| 管理群角色或管理员 | 使用群成员 API，见[更新群成员资料和角色](/sdk/flutter/group/managing-group-members/update-group-member-info)。 |

全局封禁、风控处罚和管理后台策略应由可信后端维护，客户端只负责展示后端返回的结果。

## 拉黑用户

拉黑用户时传入目标 `userID`。`ex` 是可选扩展字符串，本次传入值会作为完整值保存；仅在业务已约定格式时使用。

```dart
Future<void> blockUser(String targetUserID) async {
  try {
    await OpenIM.iMManager.friendshipManager.addBlacklist(
      userID: targetUserID,
      ex: '{"source":"profile-menu"}',
    );
  } catch (error) {
    debugPrint(
      'addBlacklist failed: $error, targetUserID: $targetUserID',
    );
    rethrow;
  }
}
```

取消拉黑时，把目标用户 ID 传给 `removeBlacklist()`。

```dart
Future<void> unblockUser(String targetUserID) async {
  try {
    await OpenIM.iMManager.friendshipManager.removeBlacklist(
      userID: targetUserID,
    );
  } catch (error) {
    debugPrint(
      'removeBlacklist failed: $error, targetUserID: $targetUserID',
    );
    rethrow;
  }
}
```

## 调用后的状态变化

- `addBlacklist()` 的 Future 成功表示拉黑请求已经完成；`onBlackAdded` 随后携带对应的 `BlacklistInfo`，应按 `userID` 写入或覆盖本地黑名单记录。
- `removeBlacklist()` 的 Future 成功表示解除请求已经完成；`onBlackDeleted` 携带被移出的 `BlacklistInfo`，应按 `userID` 从本地集合移除。

Future 完成后可以结束当前按钮的提交状态。事件用于合并增量；如果事件未到达或需要校准完整列表，再调用 `getBlacklist()` 重新建立快照。不要把这三个阶段视为同一件事。

## 监听黑名单事件

Flutter SDK 只保存一个 `OnFriendshipListener`。应用应在统一的联系人状态层集中设置一次，并把黑名单回调与好友申请、好友关系回调合并；不要在页面或 Widget 中反复调用 `setFriendshipListener()`。

```dart
final friendshipListener = OnFriendshipListener(
  onBlackAdded: upsertBlockedUser,
  onBlackDeleted: (blockedUser) {
    final userID = blockedUser.userID;
    if (userID != null) {
      removeBlockedUser(userID);
    }
  },
  // 同一个监听器还应合并好友申请和好友关系回调。
);

```

组合 listener 与唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。完整快照查询见[获取黑名单列表](/sdk/flutter/user/moderating-a-user/retrieve-a-list-of-blocked-users)。
