好友申请用于建立需要对方确认的好友关系。当前用户既可以向其他用户发送申请，也可以查询收到和发出的申请，并处理自己收到的申请。

## 发送好友申请

调用 `addFriend()` 向目标用户发送好友申请。`userID` 是申请接收方的 OpenIMSDK 用户 ID；`reason` 是申请附言，可以不传。产品层还应限制附言长度并校验内容。

```dart
Future<void> sendFriendRequest(String userID, String reason) async {
  try {
    await OpenIM.iMManager.friendshipManager.addFriend(
      userID: userID,
      reason: reason,
    );
  } catch (error) {
    debugPrint('addFriend failed: $error, userID: $userID');
    rethrow;
  }
}

await sendFriendRequest('user_b', '你好，我是 Alex');
```

Future 成功表示服务端已收到申请，不表示对方已经同意。申请状态通过申请列表查询和好友申请事件同步。

## 查询申请列表

### 查询收到的申请

当前用户作为接收方时，调用 `getFriendApplicationListAsRecipient()` 获取其他用户发来的申请。

#### 参数说明

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `handleResults` | `List<int>` | 可选 | 要读取的处理状态。固定 Flutter SDK 未公开对应的 Dart 枚举，请只传入服务端契约明确的状态值；不筛选时传空列表。 |
| `offset` | `int` | 必填 | 起始偏移量，第一页传 `0`。 |
| `count` | `int` | 必填 | 本次读取数量；`offset` 大于 `0` 时必须大于 `0`。 |

```dart
Future<List<FriendApplicationInfo>> loadReceivedFriendRequests({
  int offset = 0,
  int count = 20,
}) async {
  try {
    return await OpenIM.iMManager.friendshipManager
        .getFriendApplicationListAsRecipient(
      req: GetFriendApplicationListAsRecipientReq(
        offset: offset,
        count: count,
      ),
    );
  } catch (error) {
    debugPrint(
      'getFriendApplicationListAsRecipient failed: $error, '
      'offset: $offset, count: $count',
    );
    rethrow;
  }
}
```

Future 成功后返回当前页的 `List<FriendApplicationInfo>`，可用于展示收到的待处理申请、历史记录和申请详情。

### 查询发出的申请

当前用户作为申请方时，调用 `getFriendApplicationListAsApplicant()` 获取自己发出的申请。第一页传 `offset: 0`，`count` 表示本次读取数量。

```dart
Future<List<FriendApplicationInfo>> loadSentFriendRequests({
  int offset = 0,
  int count = 20,
}) async {
  try {
    return await OpenIM.iMManager.friendshipManager
        .getFriendApplicationListAsApplicant(
      req: GetFriendApplicationListAsApplicantReq(
        offset: offset,
        count: count,
      ),
    );
  } catch (error) {
    debugPrint(
      'getFriendApplicationListAsApplicant failed: $error, '
      'offset: $offset, count: $count',
    );
    rethrow;
  }
}
```

Future 成功后返回当前页已发出的 `List<FriendApplicationInfo>`。

申请列表的分页方式与好友列表相同：后续请求把 `offset` 设为当前已加载的申请数量。返回数量小于 `count` 时，表示该列表已没有更多记录。收到申请新增、处理或删除事件后，如果列表仍在继续分页，应重置分页状态并从第一页重新加载，避免旧 `offset` 跳过或重复读取记录。

## 未处理申请数量

调用 `getFriendApplicationUnhandledCount()` 获取未处理申请数。`time` 通常传入上次查看申请列表的时间戳，用于统计该时间点之后的未处理申请。

```dart
Future<int> loadUnhandledFriendRequestCount(int lastViewedAt) async {
  try {
    return await OpenIM.iMManager.friendshipManager
        .getFriendApplicationUnhandledCount(
      GetFriendApplicationUnhandledCountReq(time: lastViewedAt),
    );
  } catch (error) {
    debugPrint(
      'getFriendApplicationUnhandledCount failed: $error, '
      'lastViewedAt: $lastViewedAt',
    );
    rethrow;
  }
}
```

Future 成功后返回符合统计条件的未处理申请数量。

## 处理好友申请

调用 `acceptFriendApplication()` 同意申请，调用 `refuseFriendApplication()` 拒绝申请。两个方法都接收申请方的 `userID` 和可选处理附言 `handleMsg`。

```dart
Future<void> acceptFriendRequest(String applicantUserID) async {
  try {
    await OpenIM.iMManager.friendshipManager.acceptFriendApplication(
      userID: applicantUserID,
      handleMsg: '已通过',
    );
  } catch (error) {
    debugPrint(
      'acceptFriendApplication failed: $error, '
      'applicantUserID: $applicantUserID',
    );
    rethrow;
  }
}

Future<void> refuseFriendRequest(String applicantUserID) async {
  try {
    await OpenIM.iMManager.friendshipManager.refuseFriendApplication(
      userID: applicantUserID,
      handleMsg: '暂不添加',
    );
  } catch (error) {
    debugPrint(
      'refuseFriendApplication failed: $error, '
      'applicantUserID: $applicantUserID',
    );
    rethrow;
  }
}
```

Future 成功表示对应的处理请求已经完成。同意申请后，新的好友关系还会通过好友事件同步。

本文档当前核对的 Flutter SDK 没有公开删除好友申请记录的方法。不要使用拒绝申请或删除好友关系来模拟删除记录；这三种操作的业务含义不同。SDK 仍可能通过 `onFriendApplicationDeleted` 通知其他端或服务端产生的删除变化。

## 调用后的状态变化

| API | Future 成功后 | 相关事件和处理方式 |
| --- | --- | --- |
| `addFriend()` | 服务端已接收好友申请，不代表双方已经成为好友。 | 通过 `onFriendApplicationAdded` 把申请写入收到或发出的申请列表。 |
| `acceptFriendApplication()` | 申请处理请求已成功。 | 通过 `onFriendApplicationAccepted` 更新申请状态；新好友关系由 `onFriendAdded` 同步。 |
| `refuseFriendApplication()` | 拒绝处理请求已成功。 | 通过 `onFriendApplicationRejected` 更新申请状态并移出待处理列表。 |

申请事件携带 `FriendApplicationInfo`。客户端应使用 `fromUserID:toUserID` 定位记录，不能只按展示名称或当前列表位置更新。查询申请列表和未处理数量的方法只返回调用时的快照，不会触发申请事件。

## 监听申请事件

Flutter SDK 在 `FriendshipManager` 中保存一个 `OnFriendshipListener`。应用应集中创建并只调用一次 `setFriendshipListener()`，把好友申请、好友关系和黑名单回调合并进同一个监听器；后续再次设置会覆盖此前保存的 Dart listener。固定版本没有公开的移除方法，退出登录或销毁业务层时应由应用停止分发回调并释放自身引用。

下面展示申请事件对应的回调。`currentUserID` 应取自当前登录会话；申请双方都会收到新增事件，因此只有 `toUserID == currentUserID` 时才计入收到的申请和未处理数量。

```dart
final friendshipListener = OnFriendshipListener(
  onFriendApplicationAdded: (application) {
    if (application.toUserID == currentUserID) {
      upsertReceivedFriendApplication(application);
      increaseUnhandledFriendApplicationCount(application);
      return;
    }

    upsertSentFriendApplication(application);
  },
  onFriendApplicationAccepted: (application) {
    markFriendApplicationAccepted(application);
    removePendingFriendApplication(application);
  },
  onFriendApplicationRejected: (application) {
    markFriendApplicationRejected(application);
    removePendingFriendApplication(application);
  },
  onFriendApplicationDeleted: removeFriendApplication,
  // 同一个监听器还应合并 onFriendAdded、onFriendInfoChanged、
  // onFriendDeleted、onBlackAdded 和 onBlackDeleted。
);

```

组合 listener 与唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。好友关系建立后的列表同步由 `onFriendAdded` 的主归属页[更新或删除好友](/sdk/flutter/user/managing-friends/update-or-delete-friends)处理。首次进入页面或用户主动刷新时，再重新查询申请列表建立完整快照。
