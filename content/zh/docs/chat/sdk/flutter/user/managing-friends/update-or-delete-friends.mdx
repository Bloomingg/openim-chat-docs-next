好友备注、置顶状态和好友关系扩展字段都属于当前用户维护的好友关系数据。调用 `updateFriends()` 更新这些字段，调用 `deleteFriend()` 删除当前用户与目标用户的好友关系。

## 更新好友关系资料

调用 `updateFriends()` 可以更新好友备注、置顶状态和关系扩展字段。一次调用会把相同的字段值应用到 `friendUserIDs` 指定的所有好友；如果要为不同好友设置不同备注，应分别调用。

### 参数说明

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `friendUserIDs` | `List<String>?` | 业务必填 | 要更新的好友用户 ID 列表。Dart 构造函数允许省略，但更新操作应提供非空列表。 |
| `remark` | `String?` | 可选 | 好友备注。 |
| `isPinned` | `bool?` | 可选 | 是否置顶该好友。 |
| `ex` | `String?` | 可选 | 好友关系扩展字段；本次传入值会整体覆盖原值。 |

```dart
Future<void> updateFriendRemark(
  String friendUserID,
  String remark,
) async {
  try {
    await OpenIM.iMManager.friendshipManager.updateFriends(
      UpdateFriendsReq(
        friendUserIDs: [friendUserID],
        remark: remark.trim(),
      ),
    );
  } catch (error) {
    debugPrint(
      'updateFriends failed: $error, friendUserID: $friendUserID',
    );
    rethrow;
  }
}
```

### 返回结果

Future 成功表示好友资料更新请求已经完成。好友列表和联系人详情再通过 `onFriendInfoChanged` 合并增量，必要时调用 `getFriendsInfo()` 或 `checkFriend()` 校准状态。

`ex` 是一个完整字符串，不会按 JSON 字段增量合并。多个业务模块共用该字段时，应先读取原值，合并各自命名空间后再写回完整结果。还要处理并发写入造成的覆盖；复杂场景建议由业务后端统一维护。

### 备注、置顶和扩展字段

`updateFriends()` 统一处理三类好友关系设置：

- 传入 `remark` 修改好友备注；不同好友需要不同备注时应分别调用。
- 传入 `isPinned` 设置好友是否置顶，可以一次更新多个好友。
- 传入 `ex` 更新好友关系扩展字符串；该字段按完整值覆盖，不会自动合并 JSON。

一次批量调用会把相同字段值应用到全部 `friendUserIDs`。提交前应确认目标用户范围，避免把同一备注或扩展数据误写给多个好友。固定 Flutter SDK 的 `FriendInfo` 没有公开 `isPinned` 字段；写入置顶状态后，不要假设好友查询一定能直接返回该字段，应以产品实际使用的数据源校准展示。

## 删除好友关系

调用 `deleteFriend()` 删除当前用户与目标用户的好友关系。删除后，应同步更新好友列表、好友资料卡和相关操作入口。

```dart
Future<void> removeFriend(String friendUserID) async {
  try {
    await OpenIM.iMManager.friendshipManager.deleteFriend(
      userID: friendUserID,
    );
  } catch (error) {
    debugPrint(
      'deleteFriend failed: $error, friendUserID: $friendUserID',
    );
    rethrow;
  }
}
```

Future 成功表示删除请求已经完成。可以结束按钮的提交状态；好友列表、资料卡和其他登录端的最终状态仍应通过事件合并，或重新查询进行校准。

## 调用后的状态变化

- `updateFriends()` 成功表示更新请求已经完成；最终好友资料通过 `onFriendInfoChanged` 同步，回调携带变化后的 `FriendInfo`，按 `userID` 覆盖本地记录。
- `deleteFriend()` 成功表示删除请求已经完成；通过 `onFriendDeleted` 从好友列表和关系缓存中移除对应 `userID`。
- `onFriendAdded` 用于同步新建立的好友关系，通常来自好友申请被同意等流程，不应写成由 `updateFriends()` 或 `deleteFriend()` 触发。

## 监听好友事件

Flutter SDK 只保存一个 `OnFriendshipListener`。应用应在统一的联系人状态层集中设置一次，并把本页回调与好友申请、黑名单回调合并；不要在多个页面或 Widget 中重复调用 `setFriendshipListener()`。

```dart
final friendshipListener = OnFriendshipListener(
  onFriendAdded: mergeFriend,
  onFriendInfoChanged: mergeFriend,
  onFriendDeleted: (friend) {
    final userID = friend.userID;
    if (userID == null) return;

    removeFriendFromStore(userID);
    resetContactDetailIfNeeded(userID);
  },
  // 同一个监听器还应合并好友申请和黑名单回调。
);

```

回调携带 `FriendInfo`，应使用 `userID` 合并本地状态。组合 listener 与唯一设置入口见[事件概览](/sdk/flutter/events/overview-events)。

如果更新或删除后没有收到预期事件，可以调用 `getFriendsInfo()` 或 `checkFriend()` 重新确认目标用户的关系状态。
