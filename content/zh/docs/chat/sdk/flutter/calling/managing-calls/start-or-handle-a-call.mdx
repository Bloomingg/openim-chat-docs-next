发起通话时先构造 `InvitationInfo`，再放入 `SignalingInfo`。单聊调用 `signalingInvite()`，群聊调用 `signalingInviteInGroup()`。OpenIM Flutter SDK 负责邀请、接受、拒绝和挂断等信令状态；应用仍需使用返回的凭据接入实时音视频媒体引擎。

## 发起单聊通话

```dart
final info = SignalingInfo(
  userID: currentUserID,
  invitation: InvitationInfo(
    inviterUserID: currentUserID,
    inviteeUserIDList: [peerUserID],
    groupID: '',
    roomID: roomID,
    timeout: 30,
    mediaType: 'video',
    sessionType: ConversationType.single,
    platformID: Platform.isIOS ? IMPlatform.ios : IMPlatform.android,
  ),
  offlinePushInfo: OfflinePushInfo(
    title: '视频通话',
    desc: '你收到一个视频通话邀请',
    iOSPushSound: 'default',
    iOSBadgeCount: true,
  ),
);

final credentials = await OpenIM.iMManager.signalingManager
    .signalingInvite(info: info);
```

### 参数说明

| 字段 | 说明 |
| --- | --- |
| `userID` | 执行当前信令操作的登录用户 ID。 |
| `inviterUserID` | 本次通话的发起人。 |
| `inviteeUserIDList` | 被邀请用户列表；单聊只放对方一个用户，不要包含当前用户。 |
| `groupID` | 单聊固定为空字符串。 |
| `roomID` | 本次通话的唯一房间标识；后续事件和状态按它合并。 |
| `timeout` | 邀请等待秒数。 |
| `mediaType` | 业务约定的 `audio` 或 `video`。 |
| `sessionType` | 单聊传 `ConversationType.single`。 |
| `platformID` | 当前 Flutter 运行平台，使用 `IMPlatform.ios` 或 `IMPlatform.android`。 |

`InvitationInfo` 的公开字段在 Dart 中均为 nullable，但发起端仍应完整填写本次通话所需字段。`initiateTime` 由信令流程返回或补充，不需要在新邀请中预填。

`offlinePushInfo` 不属于 `InvitationInfo`。通知能否送达仍取决于 OpenIMServer 和移动端推送配置。

Future 成功返回 `SignalingCertificate`，其中 `roomID`、`token`、`liveURL` 和 `busyLineUserIDList` 都可能为空。取得有效凭据后再连接媒体房间；成功不表示被邀请方已经接听。

## 发起群聊通话

群聊只邀请 `inviteeUserIDList` 明确列出的成员，不会因填写 `groupID` 自动邀请全群：

```dart
final groupInfo = SignalingInfo(
  userID: currentUserID,
  invitation: InvitationInfo(
    inviterUserID: currentUserID,
    inviteeUserIDList: selectedGroupMemberIDs,
    groupID: groupID,
    roomID: groupID,
    timeout: 30,
    mediaType: 'video',
    sessionType: ConversationType.group,
    platformID: Platform.isIOS ? IMPlatform.ios : IMPlatform.android,
  ),
);

final credentials = await OpenIM.iMManager.signalingManager
    .signalingInviteInGroup(info: groupInfo);
```

群聊的 `groupID` 不能为空，`sessionType` 使用 `ConversationType.group`。示例以 `groupID` 作为 `roomID`，方便按群组恢复房间；如果业务自行生成房间 ID，所有参与端必须使用同一值且避免与进行中的房间冲突。先排除空值、当前用户和重复的被邀请者。

`busyLineUserIDList` 只表示部分成员忙线，不应因此中止其他成员的邀请流程。群成员选择和发起权限由应用根据群资料处理，请求失败时直接处理 SDK 错误。

## 接受或拒绝邀请

被邀请方从 `onReceiveNewInvitation` 取得原始 `SignalingInfo`。接受、拒绝、取消和挂断都应沿用这个对象，不要重建只有 `roomID` 的资料：

```dart
final credentials = await OpenIM.iMManager.signalingManager
    .signalingAccept(info: receivedInfo);

// 用户拒绝时：
await OpenIM.iMManager.signalingManager
    .signalingReject(info: receivedInfo);
```

`signalingAccept()` 返回 `SignalingCertificate`；取得有效的 `token` 和 `roomID` 后再进入媒体房间。`signalingReject()` 的 Future 成功只表示拒绝请求完成，发起端界面仍由 `onInviteeRejected` 更新。

## 取消或挂断

邀请尚未接通时由发起方取消；通话已建立后由参与者挂断：

```dart
await OpenIM.iMManager.signalingManager
    .signalingCancel(info: activeInfo);

await OpenIM.iMManager.signalingManager
    .signalingHungUp(info: activeInfo);
```

这些调用只更新信令状态。应用还需要停止本地采集、断开媒体房间并释放摄像头和麦克风。Future 成功、远端 listener 到达和重新查询是不同阶段；取消、拒绝、超时和挂断应进入同一套按 `roomID` 幂等的清理流程。

## 监听通话状态

`OnSignalingListener` 同时承载邀请生命周期、参与者和流变化。应用初始化通话模块时集中设置一次：

```dart
Future<void> registerCallListener() {
  return OpenIM.iMManager.signalingManager.setSignalingListener(
    OnSignalingListener(
      onReceiveNewInvitation: handleIncomingCall,
      onInviteeAccepted: handleInviteeAccepted,
      onInviteeRejected: handleInviteeRejected,
      onInvitationCancelled: handleInvitationCancelled,
      onInvitationTimeout: handleInvitationTimeout,
      onInviteeAcceptedByOtherDevice: handleAcceptedElsewhere,
      onInviteeRejectedByOtherDevice: handleRejectedElsewhere,
      onHangup: handleHangup,
      onRoomParticipantConnected: handleParticipantConnected,
      onRoomParticipantDisconnected: handleParticipantDisconnected,
      onStreamChange: handleStreamChange,
      onReceiveCustomSignal: handleCustomSignal,
    ),
  );
}
```

| Listener | 载荷 | 应用处理 |
| --- | --- | --- |
| `onReceiveNewInvitation` | `SignalingInfo` | 展示来电并启动超时计时。 |
| `onInviteeAccepted`、`onInviteeRejected` | `SignalingInfo` | 合并被邀请方的处理结果。 |
| `onInvitationCancelled`、`onInvitationTimeout` | `SignalingInfo` | 关闭尚未接通的界面。 |
| `onInviteeAcceptedByOtherDevice`、`onInviteeRejectedByOtherDevice` | `SignalingInfo` | 当前设备停止响铃并同步其他设备结果。 |
| `onHangup` | `SignalingInfo` | 结束通话并释放媒体资源。 |
| `onRoomParticipantConnected`、`onRoomParticipantDisconnected` | `RoomCallingInfo` | 按 `roomID` 与用户 ID 合并参与者。 |
| `onStreamChange` | `RoomCallingInfo` | 更新参与者媒体流状态。 |

本页是以上 11 个非自定义信令 callback 的完整归属页。`SignalingManager` 只保存一个 listener，后一次设置会替换先前实例；不要在 Widget 的 `build()` 中重复设置。固定 SDK 没有 remove 或 unset API，退出或切换账号时应让应用级分发器停止写入旧状态，并以新账号的完整 listener 覆盖。

`SignalingInfo.invitation` 与 `RoomCallingInfo.roomID` 都可能为空，不能强制解包。先取得有效 `roomID` 再合并；参与者状态同时结合 `groupMemberInfo.userID` 或 `userInfo.userID`，不要使用数组下标或显示名称。
