应用重新启动、从离线状态恢复或通话界面重建后，可以查询尚未处理的邀请和房间信息。查询结果应与实时通话 listener 统一合并，不能仅凭本地界面状态判断当前通话。

## 恢复启动时未处理的邀请

`getSignalingInvitationInfoStartApp()` 不接收业务参数，返回启动时需要恢复的 `SignalingInfo`：

```dart
final info = await OpenIM.iMManager.signalingManager
    .getSignalingInvitationInfoStartApp();

final invitation = info.invitation;
if (invitation?.roomID?.isNotEmpty == true) {
  restoreIncomingCall(invitation!, info.offlinePushInfo);
}
```

`SignalingInfo.userID`、`invitation` 和 `offlinePushInfo` 均可能为空。恢复后仍需处理取消、超时、接受和挂断 listener，因为结果只代表查询时刻的状态。

## 按群组查询房间

群聊通话可以使用 `signalingGetRoomByGroupID()` 查询 `RoomCallingInfo`。参数是群组 ID（`groupID`），不是自定义 `roomID`。

```dart
final room = await OpenIM.iMManager.signalingManager
    .signalingGetRoomByGroupID(groupID: groupID);

if (room.roomID?.isNotEmpty == true) {
  renderParticipants(room.participant ?? const []);
}
```

返回快照可能包含 `invitation`、`participant`、`roomID`、`token`、`liveURL` 和 `groupID`。这些字段在 Dart model 中都是 nullable；没有进行中的群聊通话时，应把错误或空房间视为“当前无可恢复通话”，不要继续使用旧凭据。

## 获取房间凭据

已经知道 `roomID`、但需要重新取得入会凭据时，调用 `signalingGetTokenByRoomID()`：

```dart
final credentials = await OpenIM.iMManager.signalingManager
    .signalingGetTokenByRoomID(roomID: roomID);

final token = credentials.token;
final resolvedRoomID = credentials.roomID;
if (token == null || token.isEmpty || resolvedRoomID == null) return;

await mediaEngine.join(
  roomID: resolvedRoomID,
  token: token,
  liveURL: credentials.liveURL,
);
```

`SignalingCertificate` 的所有字段都是 nullable。只有取得有效 `token` 和 `roomID` 后才能连接媒体引擎。房间 token 属于短期敏感凭据，只在内存中使用，不要写入日志、URL、分析事件或持久化存储。

## 查询结果与事件增量

本页三个查询 API 的 Future 成功后，分别得到启动邀请、群组房间或房间凭据在当前时刻的快照。查询本身不等于通话 listener 已到达，也不需要等待 listener 后才能恢复界面。

邀请生命周期、参与者和媒体流的后续变化见[发起或处理通话](/sdk/flutter/calling/managing-calls/start-or-handle-a-call)。房间状态按 `roomID` 合并，参与者同时使用用户 ID；连接恢复或本地状态不一致时重新查询校准。
