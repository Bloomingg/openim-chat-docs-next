黑名单方法来自 `open_im_sdk/relation.go` 中的黑名单部分。它们用于维护当前用户的屏蔽关系；具体屏蔽效果仍以 SDK 和服务端策略为准。

## 方法列表

### AddBlack

| 项目       | 说明                                                                                        |
| ---------- | ------------------------------------------------------------------------------------------- |
| 方法签名   | `AddBlack(callback Base, operationID string, blackUserID string, ex string)`                |
| 作用       | 将指定用户加入黑名单，适合屏蔽对方消息或限制关系互动。具体业务影响由 SDK 和服务端策略决定。 |
| 关键参数   | `blackUserID` 是要加入黑名单的用户 ID；`ex` 是扩展信息。                                    |
| 结果或事件 | 成功返回处理结果，并触发黑名单新增事件。                                                    |

### GetBlackList

| 项目       | 说明                                                             |
| ---------- | ---------------------------------------------------------------- |
| 方法签名   | `GetBlackList(callback Base, operationID string)`                |
| 作用       | 获取当前用户黑名单列表，适合隐私设置页、拦截列表或关系状态恢复。 |
| 关键参数   | `operationID` 用于日志追踪；无其他业务参数。                     |
| 结果或事件 | 成功返回黑名单用户列表 JSON。                                    |

### RemoveBlack

| 项目       | 说明                                                                  |
| ---------- | --------------------------------------------------------------------- |
| 方法签名   | `RemoveBlack(callback Base, operationID string, removeUserID string)` |
| 作用       | 将指定用户移出黑名单，恢复与该用户的普通关系能力。                    |
| 关键参数   | `removeUserID` 是要移出黑名单的用户 ID。                              |
| 结果或事件 | 成功返回处理结果，并触发黑名单移除事件。                              |

## 回调与返回约定

Android core binding 的异步方法通过回调返回结果。

| 回调类型          | 方法                                    | 说明                                                   |
| ----------------- | --------------------------------------- | ------------------------------------------------------ |
| `Base`            | `OnSuccess(data string)`                | 成功时返回 JSON 字符串、空字符串或基础类型序列化结果。 |
| `Base`            | `OnError(errCode int32, errMsg string)` | 失败时返回 SDK 错误码和错误消息。                      |
| `SendMsgCallBack` | `OnProgress(progress int)`              | 仅发送消息时使用，表示发送或上传进度。                 |

除同步创建消息类方法外，带 `callback` 的方法不直接返回业务数据。参数中名称包含 `List`、`Info`、`Param`、`Req`、`Options` 的 `string` 通常是 JSON 字符串，SDK 内部会反序列化为对应结构。

## 事件联动

黑名单新增和移除通过 `SetFriendListener` 注册的 `OnFriendshipListener` 接收。
