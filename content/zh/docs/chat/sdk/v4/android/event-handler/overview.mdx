监听器方法来自 `open_im_sdk/listener.go`，回调接口来自 `open_im_sdk_callback/callback_client.go`。监听器用于接收连接、用户、好友、群组、会话、消息和自定义业务事件，是登录后实时同步 UI 和本地状态的入口。

建议在登录前完成主要监听器注册，避免登录后的同步事件先到达但业务层尚未接收。监听器是替换式注册；如果多个页面或模块都需要消费同一类事件，应在业务层封装统一分发。

## 使用顺序

| 阶段     | 建议                                                                 |
| -------- | -------------------------------------------------------------------- |
| 初始化后 | 先注册连接、会话、消息、群组、好友和用户监听器。                     |
| 登录后   | 监听服务端同步、新消息、关系变化、群变化和用户资料变化。             |
| 页面销毁 | 不建议由单个页面直接覆盖全局监听器；应从业务分发器中注销页面订阅。   |
| 账号切换 | 退出旧账号后，确认新账号需要的监听处理逻辑已经重新绑定或可复用。     |
