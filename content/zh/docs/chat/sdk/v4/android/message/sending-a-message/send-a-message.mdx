发送消息由两个阶段组成：先用 `Create*Message` 同步创建消息 JSON，再把该消息 JSON 传给 `SendMessage` 或 `SendMessageNotOss`。本页只列真实签名、参数关系和方法用途，不提供示例代码。

## 发送方法

### SendMessage

| 项目       | 说明                                                                                                                                                  |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| 方法签名   | `SendMessage(callback SendMsgCallBack, operationID string, message string, recvID string, groupID string, offlinePushInfo string, isOnlineOnly bool)` |
| 作用       | 发送由 Create 方法创建的消息，并使用 SDK 内置上传和发送链路。单聊与群聊只能选择一个目标字段。                                                         |
| 关键参数   | `message` 是消息 JSON；单聊传 `recvID`；群聊传 `groupID`；`offlinePushInfo` 是离线推送 JSON；`isOnlineOnly` 控制是否只在线投递。                      |
| 结果或事件 | 成功通过 `OnSuccess` 返回服务端确认后的消息 JSON，进度通过 `OnProgress` 返回，失败通过 `OnError` 返回。                                               |

### SendMessageNotOss

| 项目       | 说明                                                                                                                                                        |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 方法签名   | `SendMessageNotOss(callback SendMsgCallBack, operationID string, message string, recvID string, groupID string, offlinePushInfo string, isOnlineOnly bool)` |
| 作用       | 发送不走 SDK OSS 上传链路的消息。适合业务自定义上传、资源 URL 已经准备好或使用 `Create*ByURL` 方法创建的消息。                                              |
| 关键参数   | 参数语义与 `SendMessage` 相同，但调用方需要保证消息里的资源信息已经可被接收端访问。                                                                         |
| 结果或事件 | 成功、进度和失败回调与 `SendMessage` 一致。                                                                                                                 |

## 创建普通、媒体与文件消息方法

### GetAtAllTag

| 项目       | 说明                                                                                           |
| ---------- | ---------------------------------------------------------------------------------------------- |
| 方法签名   | `GetAtAllTag(operationID string) string`                                                       |
| 作用       | 获取群聊 @ 所有人使用的特殊标记。发送 @ 所有人消息时，应使用该标记而不是业务侧随意拼接字符串。 |
| 关键参数   | `operationID` 用于追踪。                                                                       |
| 结果或事件 | 同步返回 @ 所有人标记字符串。                                                                  |

### CreateTextMessage

| 项目       | 说明                                                                                 |
| ---------- | ------------------------------------------------------------------------------------ |
| 方法签名   | `CreateTextMessage(operationID string, text string) string`                          |
| 作用       | 创建普通文本消息。适合纯文本聊天内容，发送前可由业务层先完成长度、空白和敏感词处理。 |
| 关键参数   | `text` 是消息文本。                                                                  |
| 结果或事件 | 同步返回消息 JSON 字符串，随后传给发送方法。                                         |

| 方法签名   | `CreateAdvancedTextMessage(operationID string, text string, messageEntityList string) string` |

## 参数选择

| 目标 | recvID      | groupID  | 说明                         |
| ---- | ----------- | -------- | ---------------------------- |
| 单聊 | 接收用户 ID | 空字符串 | 向一个用户发送消息。         |
| 群聊 | 空字符串    | 群 ID    | 向一个 OpenIM 群组发送消息。 |

## 结果事件

| 事件     | 来源                                                                               | 说明                                     |
| -------- | ---------------------------------------------------------------------------------- | ---------------------------------------- |
| 发送成功 | `SendMsgCallBack.OnSuccess`                                                        | 返回服务端确认后的消息 JSON。            |
| 发送进度 | `SendMsgCallBack.OnProgress`                                                       | 文件、图片、视频等可能产生进度。         |
| 新消息   | `OnAdvancedMsgListener.OnRecvNewMessage` 或 `OnBatchMsgListener.OnRecvNewMessages` | 接收端通过监听器收到消息。               |
| 会话变化 | `OnConversationListener.OnNewConversation`、`OnConversationChanged`                | 发送或接收消息可能改变会话列表和未读数。 |

## 注意事项

- 单聊和群聊只选择一个目标字段：单聊传 `recvID`，群聊传 `groupID`。
- `Create*MessageByURL` 类方法适合资源已上传或由业务侧提供 URL 的场景。
- 发送前必须已登录，并已设置消息和会话监听器。
- 不要把旧文档里的跨平台示例直接复制到 Android core binding；应按本页签名映射。
