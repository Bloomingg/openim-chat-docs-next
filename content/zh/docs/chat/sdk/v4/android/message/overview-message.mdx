会话和消息方法来自 `open_im_sdk/conversation_msg.go`。创建消息类方法同步返回消息 JSON；发送、查询、删除、已读等方法通过回调返回。

## 会话方法

### GetAllConversationList

| 项目       | 说明                                                               |
| ---------- | ------------------------------------------------------------------ |
| 方法签名   | `GetAllConversationList(callback Base, operationID string)`        |
| 作用       | 获取当前用户的全部本地会话列表，适合聊天列表全量加载或登录后刷新。 |
| 关键参数   | 无业务参数。                                                       |
| 结果或事件 | 成功返回会话列表 JSON。                                            |

### GetConversationListSplit

| 项目       | 说明                                                                                 |
| ---------- | ------------------------------------------------------------------------------------ |
| 方法签名   | `GetConversationListSplit(callback Base, operationID string, offset int, count int)` |
| 作用       | 分页获取会话列表，适合会话数量较多时按页加载。                                       |
| 关键参数   | `offset` 和 `count` 控制分页。                                                       |
| 结果或事件 | 成功返回当前页会话列表 JSON。                                                        |

### GetOneConversation

| 项目       | 说明                                                                                           |
| ---------- | ---------------------------------------------------------------------------------------------- |
| 方法签名   | `GetOneConversation(callback Base, operationID string, sessionType int32, sourceID string)`    |
| 作用       | 按会话类型和目标 ID 获取单个会话。单聊 `sourceID` 通常是用户 ID，群聊 `sourceID` 通常是群 ID。 |
| 关键参数   | `sessionType` 指定会话类型；`sourceID` 指定目标 ID。                                           |
| 结果或事件 | 成功返回会话 JSON。                                                                            |

### GetMultipleConversation

| 方法签名   | `GetMultipleConversation(callback Base, operationID string, conversationIDList string)` |

## 消息方法

### GetAtAllTag

| 项目       | 说明                                                                                           |
| ---------- | ---------------------------------------------------------------------------------------------- |
| 方法签名   | `GetAtAllTag(operationID string) string`                                                       |
| 作用       | 获取群聊 @ 所有人使用的特殊标记。发送 @ 所有人消息时，应使用该标记而不是业务侧随意拼接字符串。 |
| 关键参数   | `operationID` 用于追踪。                                                                       |
| 结果或事件 | 同步返回 @ 所有人标记字符串。                                                                  |

### CreateTextMessage

| 项目       | 说明                                                                                 |
| ---------- | ------------------------------------------------------------------------------------ |
| 方法签名   | `CreateTextMessage(operationID string, text string) string`                          |
| 作用       | 创建普通文本消息。适合纯文本聊天内容，发送前可由业务层先完成长度、空白和敏感词处理。 |
| 关键参数   | `text` 是消息文本。                                                                  |
| 结果或事件 | 同步返回消息 JSON 字符串，随后传给发送方法。                                         |

### CreateAdvancedTextMessage

| 项目       | 说明                                                                                          |
| ---------- | --------------------------------------------------------------------------------------------- |
| 方法签名   | `CreateAdvancedTextMessage(operationID string, text string, messageEntityList string) string` |
| 作用       | 创建带实体信息的文本消息。适合文本中需要携带链接、富文本实体或业务范围标记的场景。            |

### SendMessage

| 项目       | 说明                                                                                                                                                  |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| 方法签名   | `SendMessage(callback SendMsgCallBack, operationID string, message string, recvID string, groupID string, offlinePushInfo string, isOnlineOnly bool)` |
| 作用       | 发送由 Create 方法创建的消息，并使用 SDK 内置上传和发送链路。单聊与群聊只能选择一个目标字段。                                                         |

| 结果或事件 | 成功通过 `OnSuccess` 返回服务端确认后的消息 JSON，进度通过 `OnProgress` 返回，失败通过 `OnError` 返回。                                               |

## 回调与返回约定

Android core binding 的异步方法通过回调返回结果。

| 回调类型          | 方法                                    | 说明                                                   |
| ----------------- | --------------------------------------- | ------------------------------------------------------ |
| `Base`            | `OnSuccess(data string)`                | 成功时返回 JSON 字符串、空字符串或基础类型序列化结果。 |
| `Base`            | `OnError(errCode int32, errMsg string)` | 失败时返回 SDK 错误码和错误消息。                      |
| `SendMsgCallBack` | `OnProgress(progress int)`              | 仅发送消息时使用，表示发送或上传进度。                 |

除同步创建消息类方法外，带 `callback` 的方法不直接返回业务数据。参数中名称包含 `List`、`Info`、`Param`、`Req`、`Options` 的 `string` 通常是 JSON 字符串，SDK 内部会反序列化为对应结构。
