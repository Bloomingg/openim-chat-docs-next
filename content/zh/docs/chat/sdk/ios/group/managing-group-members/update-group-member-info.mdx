固定版提供专用 selector 和字典形式的 `setGroupMemberInfo:onSuccess:onFailure:`。只修改昵称或角色时优先使用专用方法，使调用意图和字段边界更清楚。

## 更新群内昵称

`groupID` 和 `userID` 分别是目标群组与目标成员 ID，均不可为 `nil`；`groupNickname` 可以为 `nil`，用于提交空值语义。是否允许清空昵称以及最终写入结果由 OpenIMServer 校验。

```objc
[[OIMManager manager] setGroupMemberNickname:groupID
    userID:targetUserID
    groupNickname:@"项目负责人"
    onSuccess:^(NSString * _Nullable data) { [self showSaved]; }
    onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];
```

## 更新角色

`groupID` 和 `userID` 均不可为 `nil`。`roleLevel` 使用 `OIMGroupMemberRole`：

| 枚举值 | 数值 | 说明 |
| --- | --- | --- |
| `OIMGroupMemberRoleMember` | `20` | 普通群成员。 |
| `OIMGroupMemberRoleAdmin` | `60` | 群管理员。 |
| `OIMGroupMemberRoleOwner` | `100` | 群主；不能通过角色更新直接完成群主转让。 |

```objc
[[OIMManager manager] setGroupMemberRoleLevel:groupID
    userID:targetUserID
    roleLevel:OIMGroupMemberRoleAdmin
    onSuccess:^(NSString * _Nullable data) { [self showSaved]; }
    onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];
```

群主负责设置或移除管理员。不能把角色直接改为 `OIMGroupMemberRoleOwner` 来替代群主转让。

## 同时更新多个字段

需要在一次调用中更新多个字段时，传入 `NSDictionary`。字典只放本次要修改的可选字段；固定 SDK 的 `OIMSetGroupMemberInfo` 定义提供以下 key：

| key | Objective-C 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `NSString *` | 是 | 目标群组 ID。 |
| `userID` | `NSString *` | 是 | 目标成员用户 ID。 |
| `nickname` | `NSString * _Nullable` | 否 | 群内昵称。 |
| `faceURL` | `NSString * _Nullable` | 否 | 群内头像地址。 |
| `roleLevel` | `OIMGroupMemberRole` | 否 | 群成员角色值。字典传值时需包装为 `NSNumber`。 |
| `ex` | `NSString * _Nullable` | 否 | 群成员扩展字符串。 |

```objc
NSDictionary *changes = @{
    @"groupID": groupID,
    @"userID": targetUserID,
    @"nickname": @"项目负责人",
    @"faceURL": @"https://example.com/member.png",
    @"ex": @"{\"team\":\"mobile\"}"
};

[[OIMManager manager] setGroupMemberInfo:changes
    onSuccess:^(NSString * _Nullable data) { [self showSaved]; }
    onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];
```

只修改昵称或角色时优先使用专用 selector；使用字典形式时，不要把未修改的可空字段填成占位值。`ex` 是完整字符串，不会自动合并 JSON 字段。多个业务模块共用时，应先读取成员资料并合并已有命名空间，再写回完整值。

## 调用后的状态变化

成功 callback 表示 OpenIMServer 已完成本次更新。相关客户端随后可能通过 `onGroupMemberInfoChanged:` 收到更新后的单个 `OIMGroupMemberInfo`，应按 `groupID:userID` 覆盖本地成员记录。

该 delegate 也可能收到其他管理员或其他客户端产生的修改，不能只把它当作当前调用的完成回调。完整 listener 和快照查询见[获取群成员](/sdk/ios/group/retrieving-group-members/retrieve-group-members)；首次进入成员列表或用户主动刷新时，再查询目标成员校准当前资料。
