## 邀请群成员

`inviteUserToGroup:reason:usersID:onSuccess:onFailure:` 接收目标群组、邀请说明和一个或多个用户 ID。邀请权限、是否需要被邀请人确认以及用户最终能否入群，由群配置和 OpenIMServer 校验。

### 参数说明

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `groupID` | `NSString *` | 目标群组 ID。 |
| `reason` | `NSString *` | 邀请说明；无内容时传空字符串。 |
| `usersID` | `NSArray<NSString *> *` | 要邀请的用户 ID 数组。 |

```objc
[[OIMManager manager] inviteUserToGroup:groupID
    reason:@"邀请加入项目讨论组"
    usersID:@[@"user-002", @"user-003"]
    onSuccess:^(NSString * _Nullable data) { [self showInvitationSent]; }
    onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];
```

`groupID` 是目标群组，`usersID` 是用户 ID 数组，`reason` 没有内容时传空字符串。成功 callback 表示 OpenIMServer 已接受邀请请求，不等于所有用户已成为成员。

## 移除群成员

移除操作使用相同的 `groupID`、`reason` 和 `usersID` 字段，但传给 `kickGroupMember:reason:usersID:onSuccess:onFailure:`。群主或管理员能否移除每个目标成员，由 OpenIMServer 按双方角色校验。

```objc
[[OIMManager manager] kickGroupMember:groupID
    reason:@"已离开项目"
    usersID:@[targetUserID]
    onSuccess:^(NSString * _Nullable data) { [self showMemberRemoved]; }
    onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];
```

不能用该方法移除群主，必须先转让群主身份。批量操作成功 callback 只表示请求完成；实际成员快照以成员事件或重新分页查询校准。

用户实际加入后，`onGroupMemberAdded:` 提供单个 `OIMGroupMemberInfo`；移除成功后，成员列表可能收到 `onGroupMemberDeleted:`。两者都按 `groupID:userID` 合并，完整 listener 见[获取群成员](/sdk/ios/group/retrieving-group-members/retrieve-group-members)。需要审核的流程可能先产生申请事件，此时不要提前构造成员记录。
