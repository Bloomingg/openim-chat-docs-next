入群申请由群主或管理员处理。它和成员邀请不同：申请人主动提交请求后等待审核，邀请则由群内有权限的用户发起。

## 查询收到和发出的申请

固定版使用 `GetGroupApplicationListAsRecipientReq` 和 `GetGroupApplicationListAsApplicantReq` 分页查询。`groupIDs` 是群组 ID 数组，`handleResults` 使用 `OIMApplicationStatusNormal`（0）、`OIMApplicationStatusAccept`（1）或 `OIMApplicationStatusDecline`（-1）。

### 参数说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `groupIDs` | `NSArray<NSString *> *` | 要筛选的群组 ID。 |
| `handleResults` | `NSArray<NSNumber *> *` | 申请处理状态筛选。 |
| `offset` | `NSInteger` | 分页起始位置，第一页为 `0`。 |
| `count` | `NSInteger` | 本次读取数量。 |

```objc
GetGroupApplicationListAsRecipientReq *request =
    [[GetGroupApplicationListAsRecipientReq alloc]
        initWithGroupIDs:@[groupID]
        handleResults:@[@(OIMApplicationStatusNormal)]
        offset:0
        count:20];

[[OIMManager manager] getGroupApplicationListAsRecipientWithReq:request
    onSuccess:^(NSArray<OIMGroupApplicationInfo *> *applications) {
        [self replaceReceivedApplications:applications];
    }
    onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];
```

发出的申请改用 `getGroupApplicationListAsApplicantWithRep:onSuccess:onFailure:` 和对应 request 类型。查询成功后直接使用返回数组建立快照；读取列表不会触发申请事件。继续加载时按已请求条目数增加 `offset`，不要使用事件合并后的数组长度；返回少于 `count` 时可视为到达当前边界。

## 未处理数量

`GetGroupApplicationUnhandledCountReq.time` 是查询起点时间；首次建立角标快照时传 `0`。该查询只返回计数，不改变任何申请状态。

```objc
GetGroupApplicationUnhandledCountReq *request =
    [[GetGroupApplicationUnhandledCountReq alloc] initWithTime:0];

[[OIMManager manager] getGroupApplicationUnhandledCount:request
    onSuccess:^(NSInteger count) { [self updateBadge:count]; }
    onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];
```

固定 iOS SDK 通过上述 selector 读取未处理数量，但没有单独“清除角标”的高阶 selector。产品可在用户查看列表后更新本地角标基准，再以新的 `time` 重新查询；不要把本地清零描述为服务端已经修改申请状态。

## 同意或拒绝

两个操作都使用申请的 `groupID`、申请人 `userID` 和处理说明 `handleMsg`。只有群主或管理员可处理收到的申请，最终权限及申请当前状态由 OpenIMServer 校验。

```objc
[[OIMManager manager] acceptGroupApplication:application.groupID
    fromUserId:application.userID
    handleMsg:@"同意加入"
    onSuccess:^(NSString * _Nullable data) { [self showHandled]; }
    onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];

[[OIMManager manager] refuseGroupApplication:application.groupID
    fromUserId:application.userID
    handleMsg:@"暂不接受"
    onSuccess:^(NSString * _Nullable data) { [self showHandled]; }
    onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];
```

成功 callback 表示处理请求完成，不应直接伪造 `OIMGroupMemberInfo`。

## 删除入群申请记录

`deleteGroupRequests:onSuccess:onFailure:` 接收 `DeleteGroupRequest` 数组。每项使用 `groupID` 和申请方 `fromUserID` 定位记录；不要传审核人 ID，也不要用当前数组位置作为删除标识。

```objc
DeleteGroupRequest *deleteRequest =
    [[DeleteGroupRequest alloc] initWithGroupID:application.groupID
                                     fromUserID:application.userID];

[[OIMManager manager] deleteGroupRequests:@[deleteRequest]
                                onSuccess:^(NSString * _Nullable data) {
    [self removeApplicationForGroupID:application.groupID
                           fromUserID:application.userID];
}
                                onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showErrorCode:code message:message ?: @"删除入群申请失败"];
}];
```

成功表示删除请求完成，不等于成员关系发生变化。其他客户端通过 `onGroupApplicationDeleted:` 移除相同 `groupID:fromUserID` 的申请；若删除影响当前分页边界或未处理数量，应重新读取第一页和数量快照。

## 保持申请列表同步

在稳定的 `OIMGroupListener` 对象中添加和移除 listener，并以 `groupID:userID` 合并申请记录：

```objc
- (void)startApplicationEvents { [[OIMManager callbacker] addGroupListener:self]; }
- (void)stopApplicationEvents { [[OIMManager callbacker] removeGroupListener:self]; }

- (void)onGroupApplicationAdded:(OIMGroupApplicationInfo *)item { [self upsertApplication:item]; }
- (void)onGroupApplicationAccepted:(OIMGroupApplicationInfo *)item { [self upsertApplication:item]; }
- (void)onGroupApplicationRejected:(OIMGroupApplicationInfo *)item { [self upsertApplication:item]; }
- (void)onGroupApplicationDeleted:(OIMGroupApplicationInfo *)item { [self removeApplication:item]; }
```

事件可能来自其他管理员或其他客户端；分页列表收到变化后应重置并重新读取第一页。
