会话分组用于整理当前登录用户的会话列表，例如“工作”“未读”或“稍后处理”。它管理 `conversationID` 与分组的关系，不会创建聊天群组、修改群成员，也不会改变消息发送目标。

iOS enterprise SDK 当前通过随 `OpenIMSDK` 发布的 `OpenIMCore` Objective-C 函数提供这组能力，尚未封装成 `OIMManager` 高阶 model API。请求对象、返回结果和 listener 载荷都是 JSON `NSString`；应用应使用 `NSJSONSerialization` 解析，不要虚构 `OIMConversationGroup` 类。

```objc
#import <OpenIMSDK/CallbackProxy.h>
@import OpenIMCore;
```

## 分组类型

Core 声明使用整数表示分组类型：

| 数值 | 用途 |
| ---: | --- |
| `0` | 普通自定义分组，可由用户创建和维护。 |
| `1` | 由 SDK 或服务端筛选规则维护的分组。 |
| `2` | 仅用于 `Open_im_sdkGetConversationGroups` 查询全部类型，不能作为创建类型。 |

普通用户分类使用 `0`。筛选分组的生成规则取决于当前 OpenIMServer 部署，不要只凭名称在客户端伪造系统分组。

以下示例使用应用内的 JSON 序列化辅助函数；序列化失败时应停止调用并进入错误处理。示例聚焦应用需要构造的业务参数和 callback，不展开 Core 的底层调用上下文：

```objc
static NSString *OIMJSONText(id object) {
    NSError *error = nil;
    NSData *data = [NSJSONSerialization dataWithJSONObject:object options:0 error:&error];
    NSCAssert(data != nil, @"JSON serialization failed: %@", error);
    return [[NSString alloc] initWithData:data encoding:NSUTF8StringEncoding];
}
```

## 创建和查询分组

### 创建分组

`Open_im_sdkCreateConversationGroup` 的 `req` 是 JSON 对象：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| `name` | 必填 | 分组名称。 |
| `order` | 可选 | 顺序值；排序方向应由产品统一约定。 |
| `ex` | 可选 | 业务扩展字符串。 |
| `conversationID` | 可选 | 创建后立即加入该分组的会话。 |
| `conversationGroupType` | 可选 | `0` 为普通分组，`1` 为筛选分组；普通用户分组传 `0`。 |

```objc
NSDictionary *request = @{
    @"name": @"工作",
    @"order": @100,
    @"conversationID": conversationID,
    @"conversationGroupType": @0,
};

CallbackProxy *callback = [[CallbackProxy alloc]
    initWithOnSuccess:^(NSString * _Nullable data) {
        if (data == nil) {
            return;
        }
        NSDictionary *group = [NSJSONSerialization JSONObjectWithData:[data dataUsingEncoding:NSUTF8StringEncoding]
                                                               options:0
                                                                 error:nil];
        [self upsertConversationGroup:group];
    }
    onFailure:^(NSInteger code, NSString * _Nullable message) {
        [self showConversationGroupError:code message:message ?: @"创建会话分组失败"];
    }];

NSString *requestJSON = OIMJSONText(request);
```

将 `callback` 与 `requestJSON` 交给 `Open_im_sdkCreateConversationGroup`。成功数据是一个分组 JSON 对象。常用字段包括 `conversationGroupID`、`name`、`order`、`createTime`、`version`、`ex`、`conversationGroupType`、`hidden`、`unreadCount` 和 `conversationIDs`；iOS Core 没有对应的高阶 Objective-C model。

### 查询分组

```objc
int64_t groupType = 2; // 0：普通，1：筛选，2：全部
NSString *targetConversationID = conversationID;
```

将 `callback` 与 `groupType` 交给 `Open_im_sdkGetConversationGroups`，按类型建立完整分组快照；将 `callback` 与 `targetConversationID` 交给 `Open_im_sdkGetConversationGroupByConversationID`，查询指定会话所属的完整分组对象。两个查询都返回分组 JSON 数组，后者不是分组 ID 字符串数组。查询本身不会触发分组 listener。

### 分页查询分组内会话

```objc
NSDictionary *request = @{
    @"conversationGroupID": conversationGroupID,
    @"pagination": @{ @"pageNumber": @1, @"showNumber": @20 },
};
NSString *requestJSON = OIMJSONText(request);
```

将 `callback` 与 `requestJSON` 交给 `Open_im_sdkGetConversationGroupInfoWithConversations`。`pageNumber` 从 `1` 开始，`showNumber` 是每页数量。响应 JSON 提供 `group`、当前页 `conversationElems` 和 `isEnd`；没有 `conversationTotal`，分页终止应依据 `isEnd`，不能臆造总数。

## 添加或移除会话

`conversationIDs` 和 `conversationGroupIDs` 都以 JSON 数组字符串传入：

```objc
NSString *conversationIDsJSON = OIMJSONText(@[conversationID]);
NSString *groupIDsJSON = OIMJSONText(@[conversationGroupID]);
```

添加时把 `callback`、`conversationIDsJSON` 和 `groupIDsJSON` 交给 `Open_im_sdkAddConversationsToGroups`；移除时把相同业务参数交给 `Open_im_sdkRemoveConversationsFromGroups`。两个数组都必填且不能传空会话数组。一个会话可以属于多个分组；增删关联不会删除会话或清空消息。成功 callback 表示当前请求已完成，不返回更新后的分组对象，后续状态由 listener 增量和必要时重新查询校准。

## 更新、排序和删除

```objc
NSString *updateRequestJSON = OIMJSONText(@{
    @"conversationGroupID": conversationGroupID,
    @"name": @"重点工作",
    @"hidden": @NO,
});

NSString *orderRequestJSON = OIMJSONText(@[
    @{ @"conversationGroupID": firstGroupID, @"order": @100 },
    @{ @"conversationGroupID": secondGroupID, @"order": @200 },
]);

NSString *groupIDToDelete = conversationGroupID;
```

将 `callback` 与 `updateRequestJSON` 交给 `Open_im_sdkUpdateConversationGroup`；其中 `conversationGroupID` 必填，只传本次要更新的 `name`、`ex` 或 `hidden`。将 `callback` 与 `orderRequestJSON` 交给 `Open_im_sdkSetConversationGroupOrder`，按分组顺序对象的 JSON 数组更新排序。将 `callback` 与 `groupIDToDelete` 交给 `Open_im_sdkDeleteConversationGroup`；删除只移除分组及其会话关联，不删除其中的会话或消息。

## 监听分组变化

本页是五个会话分组 listener 的完整归属页。listener 方法直接收到 JSON 字符串：新增、变更和删除是分组数组；成员变更是包含 `group`、`conversationIDs` 和 `conversations` 的对象。

```objc
@interface ConversationGroupStore () <Open_im_sdk_callbackOnConversationGroupListener>
@end

@implementation ConversationGroupStore

- (void)startListening {
    Open_im_sdkSetConversationGroupListener(self);
}

- (void)onConversationGroupAdded:(NSString * _Nullable)json {
    [self mergeConversationGroupsJSON:json];
}

- (void)onConversationGroupChanged:(NSString * _Nullable)json {
    [self mergeConversationGroupsJSON:json];
}

- (void)onConversationGroupDeleted:(NSString * _Nullable)json {
    [self removeConversationGroupsJSON:json];
}

- (void)onConversationGroupMemberAdded:(NSString * _Nullable)json {
    [self mergeConversationGroupMembersJSON:json];
}

- (void)onConversationGroupMemberDeleted:(NSString * _Nullable)json {
    [self removeConversationGroupMembersJSON:json];
}

@end
```

Core 只保存一个会话分组 listener，且没有公开 remove 或 unset 函数。应用应由账号级状态层强引用并集中设置一次；切换账号时停止旧 listener 向界面分发，再用新账号的完整 listener 覆盖。分组按 `conversationGroupID` 幂等合并，成员关系再结合 `conversationID`；不要使用数组位置或分组名称作为标识。

调用成功、listener 增量到达和重新查询快照是三个独立阶段。
