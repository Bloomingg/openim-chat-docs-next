OpenIM iOS SDK 支持按聊天目标读取一个会话，也支持按 `conversationID` 批量读取会话。调用前应完成 SDK 登录并等待连接成功。

## 按聊天目标获取会话

单聊把对方 `userID` 作为 `sourceID` 并使用 `OIMConversationTypeC2C`；群聊把 `groupID` 作为 `sourceID` 并使用 `OIMConversationTypeGroup`。

```objc
[[OIMManager manager] getOneConversationWithSessionType:OIMConversationTypeC2C
                                              sourceID:targetUserID
                                              onSuccess:^(OIMConversationInfo *conversation) {
                                                  if (conversation) {
                                                      [chatRouter openConversation:conversation];
                                                  }
                                              }
                                              onFailure:nil];
```

返回的 `OIMConversationInfo` 可以直接交给聊天页。即使双方还没有产生过消息，也应通过这个方法获取会话，不要自行拼接 `conversationID` 或构造模型。群聊只需替换为 `OIMConversationTypeGroup` 和目标 `groupID`。

## 获取 conversationID

只需要会话 ID 时调用同步方法：

```objc
NSString *conversationID = [[OIMManager manager]
    getConversationIDBySessionType:OIMConversationTypeC2C
                          sourceID:targetUserID];

if (conversationID.length == 0) {
    return;
}
```

该方法返回字符串，不返回完整 `OIMConversationInfo`。会话设置、已读、隐藏和消息历史方法通常直接接收这个 ID。

## 按 conversationID 批量获取

```objc
[[OIMManager manager] getMultipleConversation:conversationIDs
                                    onSuccess:^(NSArray<OIMConversationInfo *> *conversations) {
                                        [conversationStore mergeByConversationID:conversations ?: @[]];
                                    }
                                    onFailure:nil];
```

成功 callback 的数组声明为 nullable；没有结果时按空数组处理。某个 ID 没有结果时按缺失记录处理，不要依赖数组位置与请求数组完全一一对应。

## 调用结果与后续变化

本页方法都直接使用返回值完成当前导航或查询，不要等待会话事件后才打开聊天页，也不要把查询成功描述为触发事件。页面打开后的会话新增和属性变化由[获取会话列表](/sdk/ios/conversation/retrieving-conversations/retrieve-conversation-list)中的 delegate 负责；需要校准指定会话时，再调用本页方法重新查询。
