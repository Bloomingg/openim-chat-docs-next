固定 iOS SDK 为常用会话属性提供独立 selector。调用时使用目标 `conversationID`，不要把 WASM `setConversation()` 的完整字段集合直接映射到 iOS。

## 设置置顶状态

```objc
[[OIMManager manager] pinConversation:conversationID
                             isPinned:YES
                            onSuccess:nil
                            onFailure:nil];
```

`YES` 表示置顶，`NO` 表示取消置顶。

## 设置消息接收选项

```objc
[[OIMManager manager] setConversationRecvMessageOpt:conversationID
                                             status:OIMReceiveMessageOptNotNotify
                                          onSuccess:nil
                                          onFailure:nil];
```

| 枚举 | 说明 |
| --- | --- |
| `OIMReceiveMessageOptReceive` | 在线接收消息，离线接收推送。 |
| `OIMReceiveMessageOptNotReceive` | 不接收消息，离线也不推送。 |
| `OIMReceiveMessageOptNotNotify` | 在线接收消息，离线不推送。 |

## 设置私聊与阅后即焚时长

```objc
[[OIMManager manager] setConversationPrivateChat:conversationID
                                        isPrivate:YES
                                        onSuccess:nil
                                        onFailure:nil];

[[OIMManager manager] setConversationBurnDuration:conversationID
                                          duration:30
                                         onSuccess:nil
                                         onFailure:nil];
```

`duration` 是阅后即焚时长，单位为秒。产品需要关闭私聊时传 `isPrivate:NO`；是否保留原时长由业务交互决定。

## 设置扩展字段

```objc
NSString *ex = @"{\"category\":\"work\"}";
[[OIMManager manager] setConversationEx:conversationID
                                     ex:ex
                              onSuccess:nil
                              onFailure:nil];
```

`ex` 是完整字符串值，SDK 不会替应用合并 JSON。更新前应解析并保留其他业务方拥有的字段。

## 设置定时删除

`isMsgDestruct` 控制定时删除是否启用，`msgDestructTime` 设置删除周期。两项分别调用，应用需要同时保存界面中的开关与周期状态；周期的具体计时与服务端策略应由产品配置和实际部署共同确认。

```objc
[[OIMManager manager] setConversationIsMsgDestruct:conversationID
                                     isMsgDestruct:YES
                                         onSuccess:nil
                                         onFailure:nil];

[[OIMManager manager] setConversationMsgDestructTime:conversationID
                                     msgDestructTime:destructTime
                                         onSuccess:nil
                                         onFailure:nil];
```

后续会话快照可从 `OIMConversationInfo.isMsgDestruct` 和 `OIMConversationInfo.msgDestructTime` 读取当前值。不要把定时删除与阅后即焚混为一项：`isPrivateChat`、`burnDuration` 描述私聊阅后即焚，`isMsgDestruct`、`msgDestructTime` 描述会话消息的周期删除设置。

固定版本还公开 `setConversation:req:onSuccess:onFailure:`，其 `req` 类型为 `NSDictionary`；优先使用上述类型明确的 selector。`isMarked` 和 `remark` 没有对应的高阶 Objective-C 专用 selector，不应从其他平台直接映射。

## 调用后的状态变化

成功 callback 表示本次设置请求完成，不等于 delegate 已到达。后续变化通过 `onConversationChanged:` 合并，完整注册和清理见[获取会话列表](/sdk/ios/conversation/retrieving-conversations/retrieve-conversation-list)；必要时重新查询校准当前会话。
