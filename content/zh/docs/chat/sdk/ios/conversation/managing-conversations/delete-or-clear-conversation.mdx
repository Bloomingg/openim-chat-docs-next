固定 iOS SDK 不公开 WASM `deleteConversation()` 对应的“只删除会话记录”selector。根据是否保留会话入口选择以下真实接口：

| Selector | 用途 |
| --- | --- |
| `deleteConversationAndDeleteAllMsg:onSuccess:onFailure:` | 删除指定会话记录及其消息。 |
| `clearConversationAndDeleteAllMsg:onSuccess:onFailure:` | 保留会话入口，清空该会话下的消息。 |
| `hideAllConversationsWithOnSuccess:onFailure:` | 隐藏当前账号的全部会话入口。 |

## 删除会话及其消息

```objc
[[OIMManager manager] deleteConversationAndDeleteAllMsg:conversationID
                                               onSuccess:nil
                                               onFailure:nil];
```

这不等于删除好友关系、退出群组或解散群组。聊天目标仍存在时，后续新消息或同步可能重新生成会话。

## 保留会话并清空消息

```objc
[[OIMManager manager] clearConversationAndDeleteAllMsg:conversationID
                                              onSuccess:nil
                                              onFailure:nil];
```

该操作适合“清空聊天记录”：成功后立即清理当前消息列表，再通过会话变化或重新查询校准最后一条消息、摘要和未读状态。

## 隐藏全部会话

```objc
[[OIMManager manager] hideAllConversationsWithOnSuccess:^(NSString * _Nullable data) {
    [self clearConversationList];
}
                                              onFailure:^(NSInteger code, NSString *message) {
        [self showErrorCode:code message:message];
    }];
```

这是账号级操作，执行前应清理当前账号的页面状态和分页引用。固定版本的 `deleteAllConversationFromLocalWithOnSuccess:onFailure:` 已废弃，公开文档不使用它。

成功 callback、`onConversationChanged:` 到达和重新查询校准是三个不同阶段。会话 delegate 的完整注册与清理见[获取会话列表](/sdk/ios/conversation/retrieving-conversations/retrieve-conversation-list)。
