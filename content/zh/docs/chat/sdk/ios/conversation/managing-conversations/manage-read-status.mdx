普通会话未读数保存在 `OIMConversationInfo.unreadCount`，群聊 @ 状态保存在 `groupAtType`，两者是不同状态。

## 标记会话已读

打开会话并阅读完当前可见消息后调用：

```objc
[[OIMManager manager] markConversationMessageAsRead:conversationID
                                           onSuccess:nil
                                           onFailure:nil];
```

不要在只预览通知或后台收到消息时提前标记。成功 callback 表示标记已读请求完成，不等于会话事件已经到达；随后通过 `onConversationChanged:` 合并最新未读数，需要立即确认时再重新查询指定会话校准。

## 获取并监听总未读数

```objc
[[OIMManager manager] getTotalUnreadMsgCountWithOnSuccess:^(NSInteger count) {
    [badgeStore setTotalUnread:count];
} onFailure:nil];
```

该查询只返回调用时的总未读快照。前台运行期间由本页唯一处理 `onTotalUnreadMessageCountChanged:`：

```objc
@interface TotalUnreadSync : NSObject <OIMConversationListener>
@end

@implementation TotalUnreadSync
- (void)onTotalUnreadMessageCountChanged:(NSInteger)count {
    [badgeStore setTotalUnread:count];
}
@end

[[OIMManager callbacker] addConversationListener:totalUnreadSync];
// 账号作用域结束时：
[[OIMManager callbacker] removeConversationListener:totalUnreadSync];
```

总未读数不能用当前已加载分页中的 `unreadCount` 简单相加，应以 SDK 查询和 delegate 为准。

## 清理群聊 @ 状态

```objc
[[OIMManager manager] resetConversationGroupAtType:conversationID
                                         onSuccess:nil
                                         onFailure:nil];
```

该 selector 把群聊 @ 状态重置为普通状态。成功 callback、`onConversationChanged:` 到达和重新查询校准是三个阶段；需要立即确认时，调用 `getMultipleConversation:onSuccess:onFailure:` 读取目标会话。消息已读和清理 @ 提醒可能在同一界面发生，但不要把它们当作同一个 API 状态。完整事件处理见[获取会话列表](/sdk/ios/conversation/retrieving-conversations/retrieve-conversation-list)。
