应用启动或从离线状态恢复后，可以查询尚未处理的邀请和房间信息，再重建通话界面。查询结果应与实时通话 delegate 统一合并处理，不能仅凭本地界面状态判断当前通话。

## 恢复启动时未处理的邀请

`getSignalingInvitationInfoStartAppWithOnSuccess:onFailure:` 返回应用启动时需要恢复的 `OIMSignalingInfo`，不接收业务参数。

```objc
[[OIMManager manager]
    getSignalingInvitationInfoStartAppWithOnSuccess:^(OIMSignalingInfo * _Nullable info) {
    NSString *roomID = info.invitation.roomID;
    if (roomID.length > 0) {
        [self restoreIncomingCall:info];
    }
}
    onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showCallError:code message:message ?: @"恢复通话邀请失败"];
}];
```

成功 callback 的参数声明允许为空，使用前应检查 `info` 和 `info.invitation.roomID`。恢复后仍需监听取消、超时、接受和挂断事件，因为读取结果只代表查询时刻的状态。

## 按群组查询房间

群聊通话可以使用 `signalingGetRoomByGroupID:onSuccess:onFailure:` 查询 `OIMParticipantConnectedInfo`。参数是群组 ID，不是自定义 `roomID`。

```objc
[[OIMManager manager] signalingGetRoomByGroupID:groupID
                                      onSuccess:^(OIMParticipantConnectedInfo * _Nullable room) {
    if (room.roomID.length > 0) {
        [self renderParticipants:room.participant roomID:room.roomID];
    }
}
                                      onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self handleMissingGroupCall:code message:message];
}];
```

返回快照可包含 `roomID`、`token`、`liveURL`、邀请、群组资料和参与者资料。没有进行中的群聊通话时，OpenIMServer 可能返回错误或空的房间数据；应用应将其视为“当前无可恢复通话”，不能继续使用旧的本地凭据。

通常可以在发起群聊通话时使用 `groupID` 作为 `roomID`，使发起、恢复和按群查询共用标识。如果业务使用自定义 `roomID`，仍应传 `groupID` 调用本方法，再用保存的 `roomID` 合并通话状态或查询令牌。

## 获取房间凭据

已经知道 `roomID`、但需要重新取得入会凭据时，调用 `signalingGetTokenByRoomID:onSuccess:onFailure:`。公开 header 的参数变量名写作 `groupID`，但 selector 与底层实现的真实语义都是房间 ID。

```objc
[[OIMManager manager] signalingGetTokenByRoomID:roomID
                                      onSuccess:^(OIMInvitationResultInfo * _Nullable credentials) {
    if (credentials == nil || credentials.roomID.length == 0) {
        [self showCallError:0 message:@"未取得有效房间凭据"];
        return;
    }
    [self joinMediaRoom:credentials];
}
                                      onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showCallError:code message:message ?: @"获取房间凭据失败"];
}];
```

成功 callback 返回 `OIMInvitationResultInfo`，包含 `token`、`liveURL`、`roomID` 和忙线用户列表。房间 token 属于短期敏感凭据，只在内存中交给媒体引擎，不要写入日志、URL、分析事件或持久化存储。

## 查询结果与事件增量

本页三个查询 API 的成功 callback 分别提供启动邀请、群组房间或房间凭据在当前时刻的快照。查询本身不会触发通话 delegate，也不需要等待 delegate 后才能恢复界面或连接媒体引擎。

邀请生命周期、参与者和媒体流的后续变化通过 delegate 增量更新，完整监听和清理代码见[发起或处理通话](/sdk/ios/calling/managing-calls/start-or-handle-a-call)。房间状态按 `roomID` 合并，参与者状态同时使用用户 ID。连接恢复或本地状态与事件不一致时，重新查询房间信息；不要把某个 delegate 当作此前查询 callback 的完成信号。
