发起通话时，先构造 `OIMInvitationInfo`。单聊通话调用 `signalingInvite:offlinePushInfo:onSuccess:onFailure:`，群聊通话调用 `signalingInviteInGroup:offlinePushInfo:onSuccess:onFailure:`。OpenIM iOS SDK 负责邀请、接受、拒绝和挂断等信令状态；应用仍需使用返回的房间凭据接入所选的实时音视频媒体引擎。

## 发起单聊通话

```objc
OIMInvitationInfo *invitation = [OIMInvitationInfo new];
invitation.inviteeUserIDList = @[@"user_b"];
invitation.roomID = NSUUID.UUID.UUIDString.lowercaseString;
invitation.timeout = 30;
invitation.mediaType = @"video";
invitation.groupID = @"";

OIMOfflinePushInfo *push = [OIMOfflinePushInfo new];
push.title = @"视频通话";
push.desc = @"你收到一个视频通话邀请";
push.iOSPushSound = @"default";
push.iOSBadgeCount = YES;

OIMSignalingInfo *signalingInfo =
    [[OIMManager manager] signalingInvite:invitation
                          offlinePushInfo:push
                                onSuccess:^(OIMInvitationResultInfo * _Nullable credentials) {
    if (credentials == nil) {
        [self showCallError:0 message:@"成功回调未返回房间凭据"];
        return;
    }
    [self joinMediaRoom:credentials];
}
                                 onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showCallError:code message:message ?: @"发起通话失败"];
}];

[self storePendingCall:signalingInfo];
```

### 参数说明

| 字段 | 说明 |
| --- | --- |
| `inviteeUserIDList` | 被邀请用户 ID 列表。单聊只填写对方一个用户 ID，不要包含当前用户。 |
| `roomID` | 本次通话的唯一房间标识。后续 callback、delegate 和媒体状态都按它合并；传空字符串时，iOS SDK 会生成小写 UUID。 |
| `timeout` | 邀请等待时长，单位为秒。超时后应关闭等待或来电界面。 |
| `mediaType` | 媒体类型字符串，例如 `audio` 或 `video`。传空字符串时，iOS SDK 默认补为 `video`。 |
| `groupID` | 单聊传空字符串。iOS SDK 会据此把 `sessionType` 设为 `OIMConversationTypeC2C`。 |

SDK 会使用当前登录用户补充 `inviterUserID`，并根据 `groupID` 补充 `sessionType`；调用方不需要伪造这两个字段。`platformID` 和 `initiateTime` 由底层信令流程使用，不应把服务端返回状态预先写入邀请。

`offlinePushInfo` 是可选的离线推送内容，不属于 `OIMInvitationInfo`。是否能够收到离线推送，仍取决于 OpenIMServer 和客户端的推送配置。

方法同步返回补齐后的 `OIMSignalingInfo`，应保存它供取消、挂断和状态合并使用。成功 callback 中的 `OIMInvitationResultInfo` 包含 `roomID`、`token`、`liveURL` 和 `busyLineUserIDList`；取得有效的房间凭据后再连接媒体房间。成功 callback 不表示被邀请方已经接听。

## 发起群聊通话

群聊邀请只发送给 `inviteeUserIDList` 中明确列出的群成员，不会因为填写了 `groupID` 就自动邀请全群成员：

```objc
OIMInvitationInfo *invitation = [OIMInvitationInfo new];
invitation.inviteeUserIDList = selectedGroupMemberIDs;
invitation.groupID = groupID;
invitation.roomID = groupID;
invitation.timeout = 30;
invitation.mediaType = @"video";

OIMSignalingInfo *signalingInfo =
    [[OIMManager manager] signalingInviteInGroup:invitation
                                 offlinePushInfo:nil
                                       onSuccess:^(OIMInvitationResultInfo * _Nullable credentials) {
    if (credentials != nil) {
        [self joinMediaRoom:credentials];
    }
}
                                        onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showCallError:code message:message ?: @"发起群聊通话失败"];
}];
```

### 群聊字段差异

| 字段 | 说明 |
| --- | --- |
| `inviteeUserIDList` | 本次实际邀请的群成员 ID。先排除当前用户、空值和重复 ID。 |
| `groupID` | 目标群组 ID，不能传空字符串。iOS SDK 会据此把 `sessionType` 设为 `OIMConversationTypeSuperGroup`。 |
| `roomID` | 群通话房间标识。示例使用 `groupID`；如果业务自行生成房间 ID，必须保证所有参与端使用同一个值，并避免与仍在进行的房间冲突。 |

群成员选择、能否发起群通话等权限，需要客户端根据群资料和群成员字段在业务层处理。成功 callback 同样返回 `OIMInvitationResultInfo`；`busyLineUserIDList` 只表示其中一部分成员忙线，不应因此中止其余成员的邀请和入会流程。

## 接受或拒绝邀请

被邀请方从 `onReceiveNewInvitation:` 取得完整的 `OIMSignalingInfo`。用户确认接听后调用 `signalingAccept:onSuccess:onFailure:`；拒绝时调用 `signalingReject:onSuccess:onFailure:`。

```objc
[[OIMManager manager] signalingAccept:signalingInfo
                            onSuccess:^(OIMInvitationResultInfo * _Nullable credentials) {
    if (credentials != nil) {
        [self joinMediaRoom:credentials];
    }
}
                            onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showCallError:code message:message ?: @"接受邀请失败"];
}];

// 用户拒绝时：
[[OIMManager manager] signalingReject:signalingInfo
                            onSuccess:^{
    [self closeIncomingCall];
}
                            onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showCallError:code message:message ?: @"拒绝邀请失败"];
}];
```

必须使用收到的原始 `OIMSignalingInfo`，尤其要保留同一个 `roomID`、邀请人、被邀请人列表和会话类型，不要在接受或拒绝时重新生成房间 ID。接受成功时取得房间凭据后再连接媒体引擎；拒绝成功只表示 OpenIMServer 已完成当前请求，发起端界面随后由 `onInviteeRejected:` 更新。

## 取消或挂断

邀请尚未接通时，发起方调用 `signalingCancel:onSuccess:onFailure:`；通话已经建立后，参与者调用 `signalingHungUp:onSuccess:onFailure:`。

```objc
[[OIMManager manager] signalingCancel:signalingInfo
                            onSuccess:^{ [self releaseCallResources]; }
                            onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showCallError:code message:message ?: @"取消邀请失败"];
}];

[[OIMManager manager] signalingHungUp:signalingInfo
                            onSuccess:^{ [self releaseCallResources]; }
                            onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showCallError:code message:message ?: @"挂断通话失败"];
}];
```

这些调用只更新 OpenIM iOS SDK 的信令状态。应用还需要停止本地采集、断开媒体房间，并释放摄像头、麦克风和 `AVAudioSession` 资源。取消、拒绝、超时和挂断应进入同一个按 `roomID` 幂等的清理流程。

## 监听通话状态

通话状态层实现 `OIMSignalingListener`，并使用同一实例注册和移除。以下 11 个邀请、成员和媒体流 delegate 的完整监听归本页所有：

```objc
@interface CallStore () <OIMSignalingListener>
@end

@implementation CallStore

- (void)startListening {
    [[OIMManager callbacker] addSignalingListener:self];
}

- (void)stopListening {
    [[OIMManager callbacker] removeSignalingListener:self];
}

- (void)onReceiveNewInvitation:(OIMSignalingInfo *)info {
    [self upsertCall:info state:@"ringing"];
}

- (void)onInviteeAccepted:(OIMSignalingInfo *)info { [self mergeCall:info state:@"accepted"]; }
- (void)onInviteeRejected:(OIMSignalingInfo *)info { [self mergeCall:info state:@"rejected"]; }
- (void)onInvitationCancelled:(OIMSignalingInfo *)info { [self finishCall:info.invitation.roomID]; }
- (void)onInvitationTimeout:(OIMSignalingInfo *)info { [self finishCall:info.invitation.roomID]; }
- (void)onInviteeAcceptedByOtherDevice:(OIMSignalingInfo *)info { [self stopRinging:info.invitation.roomID]; }
- (void)onInviteeRejectedByOtherDevice:(OIMSignalingInfo *)info { [self stopRinging:info.invitation.roomID]; }

// SDK 公开 protocol 的 selector 拼写为 onHunguUp:。
- (void)onHunguUp:(OIMSignalingInfo *)info {
    [self finishCall:info.invitation.roomID];
}

- (void)onRoomParticipantConnected:(OIMParticipantConnectedInfo *)info {
    [self mergeParticipants:info roomID:info.roomID];
}

- (void)onRoomParticipantDisconnected:(OIMParticipantConnectedInfo *)info {
    [self removeParticipants:info roomID:info.roomID];
}

- (void)onStreamChange:(OIMMeetingStreamEvent *)event {
    [self updateStream:event roomID:event.roomID];
}

@end
```

`OIMSignalingInfo` 事件按 `info.invitation.roomID` 合并。参与者连接或断开时，从 `OIMParticipantConnectedInfo` 的房间和参与者资料取得用户 ID；媒体流变化由 `OIMMeetingStreamEvent.roomID` 与对应参与者标识更新。不要按事件到达顺序、展示名称或数组下标合并状态。

在初始化成功后、登录前注册应用级 listener；退出登录、切换账号或销毁状态层时调用 `removeSignalingListener:`。需要显示当前房间快照时，再查询房间信息。
