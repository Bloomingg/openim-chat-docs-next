OpenIM iOS SDK 使用会话输入状态实现“正在输入”提示。当前用户开始输入时调用 `changeInputStates:focus:onSuccess:onFailure:`；发送消息、失焦、切换会话或停止输入时再传 `focus:NO`。输入状态绑定 `conversationID`，不会写入消息列表，也不用于保存草稿文本。

## 开始和停止输入

`conversationID` 是要更新输入状态的会话 ID，`focus` 表示当前用户是否仍在输入。

```objc
- (void)startTypingInConversation:(NSString *)conversationID {
    [[OIMManager manager] changeInputStates:conversationID
                                      focus:YES
                                  onSuccess:^(NSString * _Nullable data) {}
                                  onFailure:^(NSInteger code, NSString * _Nullable message) {
        [self logTypingError:code message:message ?: @"更新输入状态失败"];
    }];
}

- (void)stopTypingInConversation:(NSString *)conversationID {
    [[OIMManager manager] changeInputStates:conversationID
                                      focus:NO
                                  onSuccess:^(NSString * _Nullable data) {}
                                  onFailure:^(NSInteger code, NSString * _Nullable message) {
        [self logTypingError:code message:message ?: @"更新输入状态失败"];
    }];
}
```

实际界面中应对 `focus:YES` 做节流或状态去重，例如只在状态从“未输入”变为“输入中”时调用一次。`focus:NO` 可以由发送消息、输入框失焦、切换会话、页面退出或一段时间没有键盘输入触发。

固定版本还保留单聊用的 `typingStatusUpdate:msgTip:onSuccess:onFailure:`。新会话输入状态流程应优先使用以 `conversationID` 为标识的 `changeInputStates:focus:...`。

## 调用结果与状态同步

成功 callback 表示服务端已接受本次输入状态更新。正在编辑的本地输入框可以直接维护自身状态；其他成员是否看到提示，以其客户端收到的输入状态事件为准，不要把调用成功当作远端界面已经更新。

输入状态事件属于 `OIMConversationListener`。在会话列表状态层注册的同一个 listener 中实现：

```objc
- (void)onConversationUserInputStatusChanged:(OIMInputStatusChangedData *)data {
    if (![data.conversationID isEqualToString:self.activeConversationID]) {
        return;
    }

    if (data.platformIDs.count > 0) {
        [self showTypingIndicatorForUser:data.userID];
    } else {
        [self hideTypingIndicatorForUser:data.userID];
    }
}
```

输入提示按 `conversationID:userID` 更新，`platformIDs` 是该用户当前仍在输入的平台快照；不要按事件顺序累计输入次数。listener 的完整注册与清理见[获取会话列表](/sdk/ios/conversation/retrieving-conversations/retrieve-conversation-list)。

## 查询当前状态

进入会话页时，可以用 `getInputstates:userID:onSuccess:onFailure:` 查询指定成员当前仍处于输入状态的平台列表。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `conversationID` | `NSString *` | 必填 | 目标会话 ID。 |
| `userID` | `NSString *` | 必填 | 要查询的成员用户 ID。 |

```objc
[[OIMManager manager] getInputstates:conversationID
                              userID:memberUserID
                           onSuccess:^(NSArray<NSNumber *> *platformIDs) {
    if (platformIDs.count > 0) {
        [self showTypingIndicatorForUser:memberUserID];
    } else {
        [self hideTypingIndicatorForUser:memberUserID];
    }
}
                           onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self logTypingQueryError:code message:message ?: @"查询输入状态失败"];
}];
```

成功 callback 返回平台 ID 数组快照，查询本身不会触发输入状态事件。群聊中需要展示多名成员正在输入时，优先通过 `onConversationUserInputStatusChanged:` 维护当前会话的输入用户集合；只在进入页面、连接恢复或需要校准状态时，对重点成员执行查询。
