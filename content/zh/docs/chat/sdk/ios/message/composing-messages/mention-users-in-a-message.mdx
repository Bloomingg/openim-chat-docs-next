群聊提及消息使用 `createTextAtMessage:atUsersID:atUsersInfo:message:`。`atUsersID` 提供稳定用户 ID，`atUsersInfo` 提供对应的 `OIMAtInfo` 展示资料。

## 提及指定成员

提及逻辑必须基于 OpenIM 用户 ID，而不是昵称、头像或会话 ID。昵称只用于展示，真正触发提醒的是 `atUsersID`。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `text` | `NSString *` | 必填 | 实际发送的消息正文；需要可点击渲染时应包含稳定的用户 ID 标记。 |
| `atUsersID` | `NSArray<NSString *> *` | 必填 | 被提及成员的用户 ID；单条消息不超过底层限制的 10 人。 |
| `atUsersInfo` | `NSArray<OIMAtInfo *> *` | 必填 | 用户 ID 与群内展示名称映射，应与 `atUsersID` 来自同一份成员数据。 |
| `message` | `OIMMessageInfo *` | 可选 | 同时引用的原消息；不引用时传 `nil`。 |

```objc
OIMAtInfo *atInfo = [OIMAtInfo new];
atInfo.atUserID = member.userID;
atInfo.groupNickname = member.nickname;

OIMMessageInfo *message =
    [OIMMessageInfo createTextAtMessage:text
                              atUsersID:@[member.userID]
                            atUsersInfo:@[atInfo]
                                message:nil];
```

发送前应确认正文非空、成员仍属于当前群组，并验证每个 `OIMAtInfo.atUserID` 非空且存在于 `atUsersID`。`OIMAtInfo` 的用户 ID 和昵称在固定模型中可为空，不能直接用未校验值建立点击目标。

## 渲染可点击的提及成员

发送端编辑器可以显示 `@昵称`，但业务状态必须同时保存稳定用户 ID。接收端从 `OIMMessageInfo.atTextElem.text`、`atUserList` 和 `atUsersInfo` 读取内容，只将能完整匹配 `atUserID` 的标记转为可点击节点；没有匹配资料的部分按普通文本显示。

`atTextElem` 及其文本、用户列表与资料数组都可能为空。不要用昵称反查用户 ID，也不要把文本中所有 `@xxx` 都视为真实成员；群昵称可能重复、变化或包含特殊字符。

## 提及全体成员

```objc
OIMMessageInfo *message =
    [OIMMessageInfo createTextAtAllMessage:@"@所有人 请查看公告"
                               displayText:@"@所有人"
                                   message:nil];
```

需要在组合文本中自行插入全体成员标识时，可用 `createAtAllFlag:` 创建 `OIMAtInfo`，并用 `getAtAllTag` 取得 SDK 的全体成员标签。是否允许提及全体成员由群权限和 OpenIMServer 校验。

全体标签不是真实群成员 ID，也不是 `OIMGroupAtType` 枚举值。客户端应根据当前成员角色决定是否展示入口，服务端仍负责最终权限校验。

## 发送带引用的提及消息

`message` 可以传入当前会话列表、历史查询或新消息 listener 返回的完整 `OIMMessageInfo`。不要手工拼接缺少标识和元素的对象，否则接收端可能无法正确展示引用内容。

```objc
OIMMessageInfo *reply =
    [OIMMessageInfo createTextAtMessage:@"@user_a 我回复的是这条消息"
                              atUsersID:@[@"user_a"]
                            atUsersInfo:@[atInfo]
                                message:quotedMessage];
```

## 发送与状态变化

创建方法只返回待发送 `OIMMessageInfo`。将其传给 `sendMessage:...` 后，成功回调与远端新消息事件按 `clientMsgID` 合并。会话 @ 状态可能通过会话变更事件更新；完整处理见[管理会话已读状态](/sdk/ios/conversation/managing-conversations/manage-read-status)。

`OIMGroupAtTypeAtMe`、`OIMGroupAtTypeAtAll` 和 `OIMGroupAtTypeAtAllAtMe` 分别表示当前用户在会话中的提醒状态，不参与消息创建。创建对象不会触发 listener；其他客户端在消息实际发送后才通过普通新消息事件接收。
