OpenIMSDK 的消息扩展数据应先确定作用范围：卡片、邀请、订单等完整业务对象使用自定义消息；需要随消息同步的轻量字段可在发送前写入 `OIMMessageInfo.ex`；只影响当前设备界面的状态写入 `localEx`。SDK 不提供在消息发送后任意附加共享键值对的通用接口。

| 场景 | 推荐方式 | 是否同步至其他客户端 |
| --- | --- | --- |
| 订单卡片、任务卡片等结构化业务内容 | 创建自定义消息后发送 | 是 |
| 消息来源、显示版本等轻量字段 | 发送前设置 `OIMMessageInfo.ex` | 是 |
| 展开、选中、本地处理标记 | `setMessageLocalEx:...` | 否 |

## 创建自定义消息

`data`、`extension` 与 `description` 都是字符串；结构化内容应先使用 `NSJSONSerialization` 编码。建议在 `data` 保存带版本信息的 JSON，以 `extension` 标识业务协议，并将 `description` 作为解析失败时的降级文案。

### 参数说明

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `data` | `NSString *` | 序列化后的业务数据。 |
| `extension` | `NSString *` | 业务类型或协议标识；不使用时可按固定 selector 传 `nil`。 |
| `description` | `NSString *` | 无法解析业务数据时的降级显示文案。 |

```objc
NSDictionary *payload = @{
    @"type": @"order_card",
    @"version": @1,
    @"orderID": orderID,
};
NSError *jsonError = nil;
NSData *jsonData = [NSJSONSerialization dataWithJSONObject:payload
                                                   options:0
                                                     error:&jsonError];
if (jsonData == nil) {
    [self showCustomMessageError:jsonError.localizedDescription ?: @"业务数据序列化失败"];
    return;
}

NSString *json = [[NSString alloc] initWithData:jsonData
                                       encoding:NSUTF8StringEncoding];
if (json == nil) {
    [self showCustomMessageError:@"业务数据不是有效的 UTF-8 字符串"];
    return;
}

OIMMessageInfo *message =
    [OIMMessageInfo createCustomMessage:json
                              extension:@"com.example.order"
                            description:@"订单卡片"];
```

创建方法只返回本地待发送的 `OIMMessageInfo`，不会触发接收事件。之后按[发送消息](/sdk/ios/message/sending-messages/send-a-message)调用发送；发送端使用成功 callback 的消息合并列表，其他客户端再通过普通新消息 listener 接收。

接收端应先确认消息类型与 `customElem`，再解析 JSON 并验证版本和字段类型。未知版本、字段缺失或 JSON 无效时，降级显示 `description`，不要让消息列表中断。

不要把权限判断、支付状态或审批流程的最终状态只放在自定义消息中。这类权威状态应由业务服务验证与保存，消息只承载显示入口或状态快照。

## 写入共享扩展字段

`OIMMessageInfo.ex` 是随消息发送与保存的扩展字符串，适合存放消息来源、显示版本或其他轻量且不敏感的附加信息。创建消息后、调用发送前，写入完整字符串：

```objc
OIMMessageInfo *message = [OIMMessageInfo createTextMessage:@"请查看最新方案"];
message.ex = @"{\"source\":\"project-board\",\"schemaVersion\":1}";
```

`ex` 没有自动合并 JSON 对象的语义。多个业务模块共用此字段时，应约定字段命名与版本，并在应用层生成完整值后再覆盖。不要放入访问 Token、密钥、隐私数据或需要服务端持续验证的权威状态。

## 保存本地扩展数据

`setMessageLocalEx:clientMsgID:localEx:onSuccess:onFailure:` 只更新当前设备的 `localEx`，可保存选中状态、下载状态等本地 UI 信息。调用前应保留消息所属的 `conversationID`，并确认 `clientMsgID` 非空：

```objc
[[OIMManager manager] setMessageLocalEx:conversationID
                            clientMsgID:message.clientMsgID
                                localEx:@"{\"downloaded\":true}"
                              onSuccess:^(NSString * _Nullable data) {
    [self updateLocalMessageState:message.clientMsgID];
}
                              onFailure:^(NSInteger code, NSString * _Nullable errorMessage) {
    [self showLocalUpdateError:code message:errorMessage ?: @"更新本地扩展失败"];
}];
```

成功只表示本地数据库已更新，不会通知其他设备，也不会触发新消息事件。`localEx` 同样是完整值覆盖，不会自动合并旧 JSON；需要保留已有字段时，先在应用层解析、合并，再写回完整字符串。

## 自定义业务 listener

非聊天消息的 SDK 业务通知通过 `OIMCustomBusinessListener` 接收。应用需强引用 listener，并用同一实例移除：

```objc
- (void)startBusinessListening {
    [[OIMManager callbacker] addCustomBusinessListener:self];
}

- (void)stopBusinessListening {
    [[OIMManager callbacker] removeCustomBusinessListener:self];
}

- (void)onRecvCustomBusinessMessage:(NSDictionary<NSString *, id> * _Nullable)message {
    if (message == nil) {
        return;
    }
    [self handleBusinessPayload:message];
}
```

固定协议允许业务通知字典为 `nil`，应先忽略空载荷，再验证自有协议的事件 ID、类型、版本和字段。以业务 ID 或幂等键合并状态，不要依赖 callback 到达顺序或数组位置。

不要把未验证的自定义载荷直接用于文件路径、URL 跳转、权限判断或原生对象反序列化。listener 到达只代表收到通知；需要恢复权威状态时，仍应重新查询对应业务服务。

创建消息、发送 callback、本地扩展写入、业务 listener 与重新查询都是独立阶段。只有消息实际发送后，其他客户端才会通过消息 listener 收到自定义消息；`setMessageLocalEx:...` 永远不产生共享事件。
