在搜索结果、引用消息、消息列表或外部通知跳转场景中，可以使用 `findMessageList:onSuccess:onFailure:`，按会话与客户端消息 ID 定位一条或多条消息。

`clientMsgID` 是 OpenIMSDK 消息在客户端侧的稳定标识。发送、接收、搜索和读取历史时，应把它和消息所属的 `conversationID` 一起保存在应用状态中，后续才能准确重新定位。

## 查询消息

```objc
OIMFindMessageListParam *query = [OIMFindMessageListParam new];
query.conversationID = conversationID;
query.clientMsgIDList = @[clientMsgID];

[[OIMManager manager] findMessageList:@[query]
    onSuccess:^(OIMSearchResultInfo * _Nullable result) {
        if (result == nil) {
            [self showLookupError:0 message:@"消息定位回调未返回结果"];
            return;
        }
        OIMSearchResultItemInfo *item = result.findResultItems.firstObject;
        OIMMessageInfo *message = item.messageList.firstObject;
        if (message != nil) {
            [self openMessage:message conversationID:item.conversationID];
        }
    }
    onFailure:^(NSInteger code, NSString * _Nullable message) {
        [self showLookupError:code message:message ?: @"定位消息失败"];
    }];
```

### 参数说明

`findMessageList:onSuccess:onFailure:` 的第一个参数是查询条件数组，每一项包含：

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `conversationID` | `NSString *` | 是 | 目标消息所在的会话 ID。 |
| `clientMsgIDList` | `NSArray<NSString *> *` | 是 | 要查询的客户端消息 ID；获取单条消息时只传一个 `clientMsgID`。 |

一次调用可以传入多个会话查询对象。处理结果时不要假设返回项和输入条件始终具有相同的数组位置，应使用结果项的 `conversationID` 和每条消息的 `clientMsgID` 对应。

## 返回结果

成功 callback 参数按固定声明可以为 `nil`，应先处理空结果；非空时是 `OIMSearchResultInfo`。按 ID 查询的结果读取 `findResultItems`，每个 `OIMSearchResultItemInfo.messageList` 包含命中的 `OIMMessageInfo`。

本地缓存尚未同步、消息已删除、已撤回或 ID 不存在时，结果数组可能为空。这是“当前找不到消息”的正常结果之一，必须先检查再访问，不要把固定数组索引当成成功条件。

## 读取消息上下文

取得定位消息后，可以调用 `fetchSurroundingMessages:onSuccess:onFailure:` 一次读取它前后的消息，适用于从搜索结果或引用消息跳入上下文。

### 参数说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `startMessage` | `OIMMessageInfo *` | 已由搜索、定位或历史查询取得的锚点消息。 |
| `viewType` | `OIMFetchSurroundingHistoryViewType` | `Quote`（0）表示引用场景，`Search`（1）表示搜索场景。 |
| `before` | `NSInteger` | 锚点之前读取的消息数量。 |
| `after` | `NSInteger` | 锚点之后读取的消息数量。 |

```objc
OIMFetchSurroundingMessagesReq *request = [OIMFetchSurroundingMessagesReq new];
request.startMessage = locatedMessage;
request.viewType = OIMFetchSurroundingHistoryViewTypeSearch;
request.before = 20;
request.after = 20;

[[OIMManager manager] fetchSurroundingMessages:request
                                     onSuccess:^(NSArray<OIMMessageInfo *> * _Nullable messages) {
    [self replaceContext:messages ?: @[]
           anchorMessage:locatedMessage];
}
                                     onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showLookupError:code message:message ?: @"读取消息上下文失败"];
}];
```

成功 callback 返回按 SDK 结果排列的 `OIMMessageInfo` 数组。结果可能少于 `before + after + 1`，例如锚点靠近历史边界、部分消息已删除或本地同步尚未完成；界面应按每条消息的 `clientMsgID` 去重，并确认锚点是否包含在结果中，不要依赖固定数组位置。

需要继续向某一方向加载时，使用高级历史查询分页；分页与方向边界请参阅[获取消息历史](/sdk/ios/message/retrieving-messages/retrieve-message-history)。

定位 callback 成功后可直接使用结果建立快照，查询本身不会触发消息事件。页面打开期间收到新消息、删除或撤回事件时，仍按 `conversationID` 与 `clientMsgID` 合并；同步完成后需要当前结果时，重新执行原查询。

查询 callback、消息事件增量和重新查询是三条独立路径。本页只负责定位，不重复注册消息事件。
