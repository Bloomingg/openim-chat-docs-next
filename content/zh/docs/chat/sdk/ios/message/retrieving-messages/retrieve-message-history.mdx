OpenIM iOS SDK 使用 `conversationID` 和 `clientMsgID` 游标读取历史消息。进入会话时先请求第一页；用户滚动到列表边界时，再把当前边界消息的 `clientMsgID` 作为下一次请求的 `startClientMsgID`。

常规方向使用 `getAdvancedHistoryMessageList:onSuccess:onFailure:`；需要反向读取时使用 `getAdvancedHistoryMessageListReverse:onSuccess:onFailure:`。两个方法都接收 `OIMGetAdvancedHistoryMessageListParam`，单聊和群聊使用同一套分页模型。

## 读取会话历史

```objc
OIMGetAdvancedHistoryMessageListParam *param =
    [OIMGetAdvancedHistoryMessageListParam new];
param.conversationID = conversationID;
param.startClientMsgID = startClientMsgID; // 第一页可为 nil
param.count = 20;
param.viewType = OIMGetHistoryViewTypeHistory;

[[OIMManager manager] getAdvancedHistoryMessageList:param
    onSuccess:^(OIMGetAdvancedHistoryMessageListInfo * _Nullable result) {
        if (result == nil) {
            [self showHistoryError:0 message:@"历史消息回调未返回结果"];
            return;
        }
        NSArray<OIMMessageInfo *> *messages = result.messageList ?: @[];
        [self prependMessages:messages];
        self.historyEnded = result.isEnd || messages.count == 0;
    }
    onFailure:^(NSInteger code, NSString * _Nullable message) {
        [self showHistoryError:code message:message ?: @"获取历史消息失败"];
    }];
```

### 参数说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `conversationID` | `NSString *` | 优先使用的目标会话 ID；可从会话列表或会话查询结果取得。 |
| `userID` | `NSString *` | 未提供 `conversationID` 时，可指定单聊对象的用户 ID。 |
| `groupID` | `NSString *` | 未提供 `conversationID` 时，可指定群组 ID。 |
| `startClientMsgID` | `NSString *` | 分页游标；第一页可传 `nil`，后续传上一页边界消息的 `clientMsgID`。 |
| `count` | `NSInteger` | 本次读取数量；应使用适合页面渲染的分页大小。 |
| `viewType` | `OIMGetHistoryViewType` | 消息视图类型；普通历史列表使用 `OIMGetHistoryViewTypeHistory`。 |

不要同时依赖 `conversationID` 与用户／群组字段表达不同目标。聊天页已有会话时，优先使用稳定的 `conversationID`；若入口只有 `userID` 或 `groupID`，先查询对应会话或确保另一侧目标字段为空。

新页面应统一一种读取方向和游标规则，不要在同一列表任意混用正向与反向 API，否则容易造成消息顺序反转、重复插入或滚动位置跳动。

## 返回结果

成功 callback 参数按固定声明可以为 `nil`，应先处理空结果；非空时是 `OIMGetAdvancedHistoryMessageListInfo`。`messageList` 是本页的 `OIMMessageInfo` 数组，`isEnd` 表示是否到达历史边界，`lastMinSeq` 是 SDK 的分页上下文，不是应用的消息标识。

当 `messageList` 为空且 `isEnd` 为真时，停止继续请求。若列表为空但 SDK 仍在同步，可保留当前游标，待同步完成后重新查询当前会话。不要使用数组索引或当前列表长度作为游标。

历史查询成功后可以直接使用返回值建立快照，不会触发新消息事件。相同消息可能同时出现在历史分页与接收事件中，因此应限定目标 `conversationID`，并以 `clientMsgID` 幂等合并；页面打开期间的增量仍由[接收消息](/sdk/ios/message/receiving-messages/receive-messages)处理。

重新查询、事件增量与先前 callback 完成是三条不同路径，彼此不能取代。刷新当前列表时，可保留已有消息并重新读取边界页，再按相同标识合并。
