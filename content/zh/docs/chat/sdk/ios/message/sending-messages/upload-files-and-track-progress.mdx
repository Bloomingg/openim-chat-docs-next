一般媒体消息不需要预先上传：以 App 可读取的本地路径创建消息后调用 `sendMessage:...`，SDK 会在发送流程中上传文件。只有业务需要先获取远端 URL、大小或存储类型时，才使用独立上传 API。

上传百分比、失败提示和重试按钮都属于应用 UI 状态。SDK callback 提供进度与完成信号，应用仍需把它们关联至本地任务或消息气泡。

## 跟踪消息上传进度

```objc
[[OIMManager manager] sendMessage:mediaMessage
                           recvID:nil
                          groupID:groupID
                     isOnlineOnly:NO
                  offlinePushInfo:nil
                        onSuccess:^(OIMMessageInfo * _Nullable sentMessage) {
    if (sentMessage == nil) {
        [self markUploadFailed:mediaMessage.clientMsgID code:0];
        return;
    }
    [self upsertMessage:sentMessage];
}
                       onProgress:^(NSInteger progress) {
    [self updateProgress:progress clientMsgID:mediaMessage.clientMsgID];
}
                        onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self markUploadFailed:mediaMessage.clientMsgID code:code];
}];
```

`onProgress` 是当前发送调用的 block，不是共享 listener 事件。以创建消息时获取的 `clientMsgID` 绑定进度，避免多个并行上传、同名文件或重试流程互相覆盖。

进度到达 `100` 不代表消息已发送成功。最终状态仍以 `onSuccess` 或 `onFailure` 为准；成功时再用 callback 返回且已检查非空的 `OIMMessageInfo` 替换本地待发送项。单聊只传 `recvID`，群聊只传 `groupID`，不要同时指定两个不同目标。

## 独立上传文件

只有业务需要先获取存储结果时，才调用 `uploadFile:name:cause:onProgress:onCompletion:onSuccess:onFailure:`：

### 参数说明

| 参数           | 类型                          | 是否必填 | 说明                                          |
| -------------- | ----------------------------- | -------- | --------------------------------------------- |
| `fullPath`     | `NSString *`                  | 必填     | 当前 App 可读取的绝对文件路径。                |
| `name`         | `NSString *`                  | 选填     | 上传文件名；传 `nil` 时 SDK 使用路径末段。       |
| `cause`        | `NSString *`                  | 选填     | 业务约定的上传原因；传 `nil` 时使用空字符串。    |
| `onProgress`   | `OIMUploadProgressCallback`   | 必填     | 依次返回已保存、当前处理和总字节数。         |
| `onCompletion` | `OIMUploadCompletionCallback` | 必填     | 返回总字节数、远端 URL 与 `putType`。         |

```objc
[[OIMManager manager] uploadFile:fileURL.path
                            name:fileURL.lastPathComponent
                           cause:@"chat-file"
                      onProgress:^(NSInteger saveBytes, NSInteger currentBytes, NSInteger totalBytes) {
    [self updateUploadedBytes:currentBytes
                   savedBytes:saveBytes
                   totalBytes:totalBytes];
}
                    onCompletion:^(NSInteger totalBytes, NSString *url, NSInteger putType) {
    OIMMessageInfo *fileMessage = [OIMMessageInfo createFileMessageByURL:url
                                                               fileName:fileURL.lastPathComponent
                                                                   size:totalBytes];
    [self sendUploadedFileMessage:fileMessage putType:putType];
}
                       onSuccess:^(NSString * _Nullable data) {
    [self handleUploadCompleted];
}
                       onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self handleUploadFailure:code message:message ?: @"上传文件失败"];
}];
```

`onProgress` 依次提供已保存、当前处理和总字节数；`onCompletion` 提供总字节数、文件 URL 和上传类型 `putType`。`onCompletion` 表示已获取资源结果，但整个调用最终是否成功仍以 `onSuccess` 或 `onFailure` 为准。消息发送成功 callback 的消息对象及失败 callback 的错误字符串，按固定声明都可能为空，使用前必须防护。

独立上传不会自动创建或发送 `OIMMessageInfo`。若要把结果发送到会话，应使用 `onCompletion` 返回的 URL 和总字节数创建对应的 URL 媒体消息，再调用 `sendMessageNotOss:...`；不要从 `onSuccess` 的 `data` 猜测文件地址。

## 关联进度与重试

媒体消息发送进度按 `clientMsgID` 关联；若业务同时执行独立上传，应另外生成稳定且唯一的本地任务 ID，将 callback、文件和后续消息对象绑定在一起。不要以文件名或列表位置匹配，同名文件、跨会话并行和重试都可能造成状态串扰。

独立上传 callback、整体上传成功、消息发送 callback 与接收事件是不同阶段，应分别维护错误和重试状态。上传失败时可保留文件名、目标会话和错误码，但不要记录 Token、敏感文件内容或完整的私密资源 URL。

重试一般媒体消息时，重新创建消息以获取新的本地发送状态；若业务已确认独立上传成功，才重用资源 URL 创建 URL 型消息。不要把旧任务的进度写入新消息气泡，也不要把上传完成 callback 当成消息已经进入历史。

## 验证上传结果

- 确认 SDK 已登录且连接可用，再开始发送媒体消息。
- 使用较大的图片或文件，确认进度会从等待、上传中转为完成。
- 确认发送成功 callback 的消息已按 `clientMsgID` 合并至正确会话。
- 并行发送多个同名文件，确认进度不会互相覆盖。
- 独立上传后，确认只有后续创建并发送 URL 型消息，其他端才会收到聊天消息。
