OpenIM iOS SDK 发送消息分两步：先用 `OIMMessageInfo` 的工厂方法创建消息对象，再调用发送方法。本页说明如何创建本地媒体、已上传的 URL 型媒体，以及名片、位置、引用等富消息。

所有工厂方法都同步返回待发送的 `OIMMessageInfo`。创建成功不代表消息已经发送，也不会触发新消息事件。

## 支持的方法

| 方法 | 用途 |
| --- | --- |
| `createImageMessageFromFullPath:` | 使用图片绝对路径创建消息。 |
| `createSoundMessageFromFullPath:duration:` | 使用音频绝对路径和时长创建消息。 |
| `createVideoMessageFromFullPath:videoType:duration:snapshotPath:` | 使用视频和快照绝对路径创建消息。 |
| `createFileMessageFromFullPath:fileName:` | 使用文件绝对路径创建消息。 |
| `createImageMessageByURL:sourcePicture:bigPicture:snapshotPicture:` | 使用原图、大图和缩略图信息创建已上传图片消息。 |
| `createSoundMessageByURL:duration:size:` | 创建已上传音频消息。 |
| `createVideoMessageByURL:videoType:duration:size:snapshot:` | 创建已上传视频消息。 |
| `createFileMessageByURL:fileName:size:` | 创建已上传文件消息。 |
| `createFaceMessageWithIndex:data:` | 创建表情消息。 |
| `createLocationMessage:latitude:longitude:` | 创建地理位置消息。 |
| `createCardMessage:` | 创建名片消息。 |
| `createQuoteMessage:message:` | 从原消息对象创建引用消息。 |
| `createAdvancedTextMessage:messageEntityList:` | 创建带实体标记的高级文本消息。 |

enterprise SDK 随附的 OpenIMCore 还公开 `Open_im_sdkCreateMarkdownMessage`，创建后的消息 JSON 对应 `OIMMessageInfo.markdownTextElem`；`OIMMarkdownTextElem` 的真实字段只有 `content`。不过 `OIMMessageInfo` 高阶 Objective-C 类别目前没有对应的 factory selector，因此只使用 `[OIMMessageInfo ...]` 接口的业务不能把普通文本或自定义消息冒充为 Markdown 消息。

直接接入 OpenIMCore 的应用可以使用该 Core 工厂创建 Markdown 消息，并在自身 Core binding 中沿用同一发送链路。渲染接收消息时，先检查 Markdown 内容类型与 `markdownTextElem.content`；同时提供纯文本降级、链接安全策略和不受信任 Markdown 的过滤。Core 返回消息 JSON、转换为高阶 model 与发送分别是不同步骤，不要把创建成功写成已经发送。

## 创建本地媒体消息

使用绝对路径时，调用对应的 `FromFullPath` 方法：

```objc
OIMMessageInfo *image =
    [OIMMessageInfo createImageMessageFromFullPath:imageURL.path];

OIMMessageInfo *sound =
    [OIMMessageInfo createSoundMessageFromFullPath:audioURL.path
                                          duration:durationSeconds];

OIMMessageInfo *video =
    [OIMMessageInfo createVideoMessageFromFullPath:videoURL.path
                                         videoType:@"video/mp4"
                                          duration:durationSeconds
                                      snapshotPath:snapshotURL.path];

OIMMessageInfo *file =
    [OIMMessageInfo createFileMessageFromFullPath:fileURL.path
                                          fileName:fileURL.lastPathComponent];
```

`duration` 使用秒。相对路径版本 `createImageMessage:`、`createSoundMessage:duration:`、`createVideoMessage:videoType:duration:snapshotPath:` 和 `createFileMessage:fileName:` 要求文件位于初始化 SDK 时指定的数据目录下，并传入相对于该目录的路径。

本地路径媒体由 SDK 在发送过程中上传。需要展示进度时，使用发送方法的 `onProgress` callback；完整流程见[上传文件并跟踪进度](/sdk/ios/message/sending-messages/upload-files-and-track-progress)。

## 示例：创建名片消息

### 参数说明

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `userID` | `NSString *` | 必填 | 名片对应的稳定用户 ID。 |
| `nickname` | `NSString *` | 必填 | 名片展示名称。 |
| `faceURL` | `NSString *` | 必填 | 名片头像地址。 |
| `ex` | `NSString *` | 必填 | 名片扩展字段的完整值；没有扩展信息时传空字符串。 |

```objc
OIMCardElem *card = [OIMCardElem new];
card.userID = @"user_b";
card.nickname = @"Alex";
card.faceURL = @"https://example.com/avatar.png";
card.ex = @"";

OIMMessageInfo *cardMessage = [OIMMessageInfo createCardMessage:card];
```

## 示例：创建位置消息

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `description` | `NSString *` | 必填 | 位置描述。 |
| `latitude` | `double` | 必填 | 纬度。 |
| `longitude` | `double` | 必填 | 经度。 |

```objc
OIMMessageInfo *locationMessage =
    [OIMMessageInfo createLocationMessage:@"上海市浦东新区"
                                  latitude:31.2304
                                 longitude:121.4737];
```

## URL 型媒体消息

URL 型方法适用于文件已经通过业务上传服务完成上传，客户端只需把资源地址和必要元数据写入消息的场景。固定 iOS SDK 的参数少于 WASM：文件与音视频不接收 UUID，本方法也不能写入 WASM URL 型媒体对象中的全部本地来源和快照尺寸字段。

### 文件消息

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `fileURL` | `NSString *` | 必填 | 已上传文件的可访问 URL。 |
| `fileName` | `NSString * _Nullable` | 可选 | 文件名；建议从实际上传结果提供。 |
| `size` | `NSInteger` | 必填 | 文件大小，单位为字节。 |

```objc
OIMMessageInfo *fileMessage =
    [OIMMessageInfo createFileMessageByURL:uploadedFileURL.absoluteString
                                  fileName:@"report.pdf"
                                      size:uploadedFileSize];
```

### 图片消息

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `sourcePath` | `NSString *` | 必填 | 本地来源路径；只有远程地址时传空字符串。 |
| `sourcePicture` | `OIMPictureInfo *` | 必填 | 原图信息。 |
| `bigPicture` | `OIMPictureInfo *` | 必填 | 大图信息。 |
| `snapshotPicture` | `OIMPictureInfo *` | 必填 | 缩略图信息。 |

`OIMPictureInfo` 的 `url`、`size`、`width` 和 `height` 应来自实际上传结果；`uuID` 和 `type` 可以按业务上传结果设置。图片消息需要同时提供原图、大图和缩略图信息，即使三种规格暂时使用同一个地址，也应填写真实尺寸和大小。

```objc
OIMPictureInfo *picture = [OIMPictureInfo new];
picture.uuID = uploadedImageID;
picture.type = @"image/jpeg";
picture.size = uploadedImageSize;
picture.width = 1280;
picture.height = 720;
picture.url = uploadedImageURL.absoluteString;

OIMMessageInfo *imageMessage =
    [OIMMessageInfo createImageMessageByURL:@""
                               sourcePicture:picture
                                  bigPicture:picture
                             snapshotPicture:picture];
```

### 音频消息

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `fileURL` | `NSString *` | 必填 | 已上传音频的可访问 URL。 |
| `duration` | `NSInteger` | 必填 | 音频时长，单位为秒。 |
| `size` | `NSInteger` | 必填 | 音频大小，单位为字节。 |

```objc
OIMMessageInfo *soundMessage =
    [OIMMessageInfo createSoundMessageByURL:uploadedAudioURL.absoluteString
                                    duration:durationSeconds
                                        size:uploadedAudioSize];
```

### 视频消息

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `fileURL` | `NSString *` | 必填 | 已上传视频的可访问 URL。 |
| `videoType` | `NSString * _Nullable` | 可选 | 视频类型，例如 `video/mp4`。 |
| `duration` | `NSInteger` | 必填 | 视频时长，单位为秒。 |
| `size` | `NSInteger` | 必填 | 视频大小，单位为字节。 |
| `snapshotURL` | `NSString * _Nullable` | 可选 | 已上传视频快照的可访问 URL。 |

```objc
OIMMessageInfo *videoMessage =
    [OIMMessageInfo createVideoMessageByURL:uploadedVideoURL.absoluteString
                                  videoType:@"video/mp4"
                                   duration:durationSeconds
                                       size:uploadedVideoSize
                                   snapshot:uploadedSnapshotURL.absoluteString];
```

不要为了通过参数校验填写虚假的文件大小、图片尺寸或音视频时长，否则接收端可能无法正确展示、下载或播放媒体内容。

## 发送已经上传的媒体文件

用对应的 `create*MessageByURL` 方法创建完整消息后，调用 `sendMessageNotOss:...`，避免再次进入 SDK 内置的对象存储上传链路：

```objc
[[OIMManager manager] sendMessageNotOss:fileMessage
                                 recvID:receiverUserID
                                groupID:nil
                           isOnlineOnly:NO
                        offlinePushInfo:nil
                              onSuccess:^(OIMMessageInfo * _Nullable sentMessage) {
    if (sentMessage == nil) {
        return;
    }
    [self upsertMessage:sentMessage clientMsgID:sentMessage.clientMsgID];
}
                             onProgress:nil
                              onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showSendError:code message:message ?: @"发送失败"];
}];
```

发送到群聊时，把 `recvID` 传 `nil` 并提供 `groupID`。URL 型媒体应使用 `sendMessageNotOss:...`；本地路径媒体使用普通 `sendMessage:...` 让 SDK 负责上传。

## 引用和其他富消息

引用消息直接传入已经取得的原 `OIMMessageInfo`，不需要序列化或手工拼接字段：

```objc
OIMMessageInfo *quoteMessage =
    [OIMMessageInfo createQuoteMessage:@"同意这个方案"
                                message:originalMessage];

OIMMessageInfo *faceMessage =
    [OIMMessageInfo createFaceMessageWithIndex:1 data:@"smile"];
```

高级文本使用 `createAdvancedTextMessage:messageEntityList:`。每个 `OIMMessageEntity` 的 `offset` 和 `length` 必须对应文本中的实际范围，URL 和扩展信息也应在发送前完成校验。

## 调用结果与消息接收

本页的创建方法都只返回待发送对象。`sendMessage:...` 或 `sendMessageNotOss:...` 成功后，发送端使用 callback 中非空的 `OIMMessageInfo`，按 `clientMsgID` 合并到目标会话列表。其他客户端通过新消息 listener 接收消息；完整注册、清理、会话路由和去重方式见[接收消息](/sdk/ios/message/receiving-messages/receive-messages)。

创建、发送成功、事件到达和重新读取历史是不同阶段，不能把创建出 `OIMMessageInfo` 当成消息已经送达。
