在 OpenIM iOS SDK 中，删除影响消息是否仍对当前账号可见，不会让其他会话成员看到“消息已撤回”。需要展示撤回态时，使用[撤回消息](/sdk/ios/message/managing-messages/revoke-a-message)；需要更新消息内容时，使用[修改消息](/sdk/ios/message/managing-messages/modify-a-message)。

固定版本提供单条本地删除、单条账号侧删除、可选择同步的批量删除，以及按用户清除指定会话消息。先根据产品动作选择对应 API，不要用撤回替代删除。

## 只删除当前设备的本地消息

如果用户动作是“仅从这台设备删除”，调用 `deleteMessageFromLocalStorage:clientMsgID:onSuccess:onFailure:`。

### 参数说明

| 参数             | 类型         | 是否必填 | 说明                |
| ---------------- | ------------ | -------- | ------------------- |
| `conversationID` | `NSString *` | 必填     | 消息所属的会话 ID。 |
| `clientMsgID`    | `NSString *` | 必填     | 要删除的消息 ID。   |

```objc
[[OIMManager manager] deleteMessageFromLocalStorage:conversationID
                                        clientMsgID:clientMsgID
                                          onSuccess:^(NSString * _Nullable data) {
    [self removeMessage:clientMsgID conversationID:conversationID];
}
                                          onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showDeleteError:code message:message ?: @"删除本地消息失败"];
}];
```

该方法只修改当前设备的本地消息库，不删除服务端消息，也不会向当前账号的其他设备或会话中的其他用户同步删除动作。重新安装应用或重建本地数据库后，因为服务端仍保留这条消息，它可能再次同步。发送失败或尚未取得服务端序号的消息只能使用本地删除。

删除的是当前会话最后一条消息时，会话摘要随后可能通过 `onConversationChanged:` 更新；完整处理见[获取会话列表](/sdk/ios/conversation/retrieving-conversations/retrieve-conversation-list)。

## 删除本地和当前账号的服务端消息

`deleteMessage:clientMsgID:onSuccess:onFailure:` 删除本地消息，并删除当前账号保存在服务端的对应消息。

### 参数说明

| 参数             | 类型         | 是否必填 | 说明                |
| ---------------- | ------------ | -------- | ------------------- |
| `conversationID` | `NSString *` | 必填     | 消息所属的会话 ID。 |
| `clientMsgID`    | `NSString *` | 必填     | 要删除的消息 ID。   |

```objc
[[OIMManager manager] deleteMessage:conversationID
                        clientMsgID:clientMsgID
                          onSuccess:^(NSString * _Nullable data) {
    [self removeMessage:clientMsgID conversationID:conversationID];
}
                          onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showDeleteError:code message:message ?: @"删除消息失败"];
}];
```

成功表示当前删除请求完成。之后即使重建本地数据库，这条消息也不应再次同步到当前账号；但该操作不会删除其他会话成员保存的消息，不会显示“消息已撤回”，也没有 `isSync` 参数可要求把删除动作同步到其他客户端。

## 批量删除消息

`deleteMessages:clientMsgID:isSync:onSuccess:onFailure:` 一次接收同一会话内的多个 `clientMsgID`。`isSync` 为 `YES` 时，请求 SDK 将本次删除同步到当前账号的其他客户端；它仍不是面向其他会话成员的撤回。

```objc
[[OIMManager manager] deleteMessages:conversationID
                         clientMsgID:selectedClientMsgIDs
                              isSync:YES
                           onSuccess:^(NSString * _Nullable data) {
    [self removeMessages:selectedClientMsgIDs conversationID:conversationID];
}
                           onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showDeleteError:code message:message ?: @"批量删除消息失败"];
}];
```

数组中的消息 ID 必须属于同一个 `conversationID`。成功 callback 表示批量删除请求完成；`isSync` 只控制当前账号多端同步边界，不代表其他成员已经删除这些消息。

## 删除某用户在会话中的全部消息

群组管理或内容治理场景可调用 `deleteUserAllMessagesInConversation:userID:onSuccess:onFailure:`。`conversationID` 指定目标会话，`userID` 指定要清除消息的发送者；最终权限和可删除范围由 OpenIMServer 校验。

```objc
[[OIMManager manager] deleteUserAllMessagesInConversation:conversationID
                                                   userID:targetUserID
                                                onSuccess:^(NSString * _Nullable data) {
    [self removeMessagesFromUserID:targetUserID
                    conversationID:conversationID];
}
                                                onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showDeleteError:code message:message ?: @"删除该用户消息失败"];
}];
```

不要在成功 callback 中猜测具体被删除的 `clientMsgID`。可以先按 `userID` 从当前会话列表移除可见项，再通过事件或重新查询历史校准完整结果。

## 接收删除事件

本页是 `onMsgDeleted:` 的正文归属页。listener 由应用强引用，通过 `addAdvancedMsgListener:` 注册，并用同一实例调用 `removeAdvancedMsgListener:` 清理；完整代码见[接收消息](/sdk/ios/message/receiving-messages/receive-messages)。

```objc
- (void)onMsgDeleted:(OIMMessageInfo *)message {
    [self removeMessage:message.clientMsgID
         conversationID:[self conversationIDForMessage:message]];
}

- (void)onDeleteUserAllMsgsInConv:(OIMDeleteUserAllMsgsListenerMsg *)message {
    [self removeMessagesFromUserID:message.userID
                    conversationID:message.conversationID];
}
```

`onMsgDeleted:` 携带单个 `OIMMessageInfo`，按消息所属会话和 `clientMsgID` 幂等移除。`onDeleteUserAllMsgsInConv:` 携带 `OIMDeleteUserAllMsgsListenerMsg`，其中只有 `userID` 和 `conversationID`，按这两个字段清除当前缓存并重新查询可见历史；不要按数组位置处理。事件可能来自其他客户端或其他有权限的用户；没有证据表明每次本地删除都必然触发事件，因此调用端应先在成功 callback 中更新当前列表，再把事件作为增量合并。

## 删除结果

| 用户动作                   | Objective-C API                                 | 结果                                                               |
| -------------------------- | ----------------------------------------------- | ------------------------------------------------------------------ |
| 仅删除当前设备上的本地消息 | `deleteMessageFromLocalStorage:clientMsgID:...` | 服务端仍保留消息，重建本地数据库后可能再次同步。                   |
| 删除当前账号保存的消息     | `deleteMessage:clientMsgID:...`                 | 删除本地与当前账号服务端保存的单条消息，不影响其他成员保存的副本。 |
| 批量删除并选择是否同步     | `deleteMessages:clientMsgID:isSync:...`         | 删除同一会话的一组消息；`isSync` 控制当前账号其他客户端的同步。    |
| 删除某用户的全部会话消息   | `deleteUserAllMessagesInConversation:userID:...` | 按用户与会话执行管控删除，权限和范围由服务端校验。                 |

调用成功、事件到达和重新读取历史是不同阶段。失败时保留原列表，并记录错误码、`conversationID` 和 `clientMsgID` 以便排查。
