转发和合并都先同步创建新的 `OIMMessageInfo`，再调用发送 API。它们不会修改原消息，也不会复用原消息的 `clientMsgID`。

## 转发单条消息

```objc
OIMMessageInfo *forwardMessage =
    [OIMMessageInfo createForwardMessage:sourceMessage];

[[OIMManager manager] sendMessage:forwardMessage
                           recvID:targetUserID
                          groupID:nil
                     isOnlineOnly:NO
                  offlinePushInfo:nil
                        onSuccess:^(OIMMessageInfo * _Nullable sentMessage) {
    if (sentMessage == nil) {
        [self showForwardError:0 message:@"转发成功回调未返回消息对象"];
        return;
    }
    [self upsertMessage:sentMessage];
}
                       onProgress:nil
                        onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showForwardError:code message:message ?: @"转发消息失败"];
}];
```

原消息应为当前消息列表、历史查询结果或新消息 listener 返回的完整 `OIMMessageInfo`，并且已经成功发送。发送中或发送失败的消息不能直接转发。转发到群聊时使用 `recvID:nil`、`groupID:targetGroupID`。

无论原消息是什么类型，都应发送 `createForwardMessage:` 返回的新对象，不要把原消息直接插入目标会话。需要重新上传仅存在于本地的文件时，应重新创建并发送媒体消息；这属于一次新的文件发送，不是消息转发。

## 合并多条消息

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `messages` | `NSArray<OIMMessageInfo *> *` | 必填 | 要合并的已发送消息数组。 |
| `title` | `NSString *` | 必填 | 合并消息卡片标题。 |
| `summaryList` | `NSArray<NSString *> *` | 必填 | 与卡片展示顺序对应的摘要列表。 |

```objc
NSArray<NSString *> *summaries =
    [self summariesForMessages:selectedMessages];
OIMMessageInfo *merged =
    [OIMMessageInfo createMergeMessage:selectedMessages
                                  title:@"聊天记录"
                            summaryList:summaries];
```

`messages` 是待合并的原消息数组，`title` 是卡片标题，`summaryList` 是展示摘要。每条原消息都应已经成功发送，摘要顺序应与消息顺序一致。摘要应按消息类型生成，不要直接展示可能包含序列化内部数据的原始内容字段；文件、自定义消息等类型应提供安全的降级文案。

## 调用结果与消息接收

创建方法只产生待发送对象，不修改原消息，也不触发消息事件。发送成功 callback 的消息参数可能为空；取得非空新消息后，先根据目标用户或群组确定会话，再按新消息的 `clientMsgID` 合并。其他客户端通过普通新消息 listener 接收，完整注册、清理和离线补偿见[接收消息](/sdk/ios/message/receiving-messages/receive-messages)。

创建对象、发送成功和远端事件到达是不同阶段。发送失败时保留待发送项，并记录错误码、错误信息以及目标用户或群组 ID；群聊发送失败时还应确认当前用户是否仍有发言权限。
