清理消息历史时，先确定产品操作针对单个会话，还是当前账号的全部会话。单个会话清理适用于单聊和群聊；账号级清理覆盖当前账号的所有会话，不应作为单个聊天窗口的“清空”按钮操作。

## 清理单个会话的消息

对目标 `conversationID` 调用 `clearConversationAndDeleteAllMsg:onSuccess:onFailure:`。无论会话类型如何，都直接使用 `OIMConversationInfo.conversationID`，不需要分别维护单聊和群聊的清理逻辑。

```objc
[[OIMManager manager] clearConversationAndDeleteAllMsg:conversationID
                                             onSuccess:^(NSString * _Nullable data) {
    [self removeMessagesFromConversation:conversationID];
    [self resetConversationPreview:conversationID];
}
                                             onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showClearError:code message:message ?: @"清理消息失败"];
}];
```

成功后，可以按 `conversationID` 立即清空当前窗口的消息快照。此操作不会删除会话记录、解散群组，或清除单聊对方与其他群成员保存的消息。

如果业务入口只有目标用户 ID 或群组 ID，应先取得对应的 `OIMConversationInfo`，再传入其中的 `conversationID`。会话摘要、未读数和最后一条消息随后可能通过 `onConversationChanged:` 更新，并按 `conversationID` 合并；完整处理见[获取会话列表](/sdk/ios/conversation/retrieving-conversations/retrieve-conversation-list)。

该方法适合“保留会话入口，只清空当前聊天窗口历史”的操作。若产品动作是“删除此聊天”，应使用会话删除 API，具体流程见[删除或清空会话](/sdk/ios/conversation/managing-conversations/delete-or-clear-conversation)。清空后收到的新消息仍应通过新消息 listener 进入对应会话，并作为新的历史起点展示。

## 清理当前账号的全部消息

`deleteAllMsgFromLocalWithOnSuccess:onFailure:` 和 `deleteAllMsgFromLocalAndSvrWithOnSuccess:onFailure:` 都不接收业务参数，并且都会清理当前账号所有会话中的本地消息。两者的区别是是否同时删除当前账号保存在服务端的消息。

### 只清理本地全部消息

只需要重置当前设备的消息数据，同时保留服务端历史时调用：

```objc
[[OIMManager manager] deleteAllMsgFromLocalWithOnSuccess:^(NSString * _Nullable data) {
    [self resetAllMessageSnapshots];
}
                                               onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showClearError:code message:message ?: @"清理本地消息失败"];
}];
```

成功表示当前账号的本地消息已经清理。此时应清空内存中的全部消息快照，不要只重置当前打开的会话。服务端仍保留历史，因此重新安装应用或重建本地数据库后，旧消息可能再次同步。

### 同时清理本地和服务端全部消息

需要清理当前账号保存在服务端的全部历史消息时调用：

```objc
[[OIMManager manager] deleteAllMsgFromLocalAndSvrWithOnSuccess:^(NSString * _Nullable data) {
    [self resetAllMessageSnapshots];
}
                                                     onFailure:^(NSInteger code, NSString * _Nullable message) {
    [self showClearError:code message:message ?: @"清理全部消息失败"];
}];
```

成功后，这些旧消息不应在重建本地数据库时再次从服务端同步。这里的“服务端全部消息”只指当前账号保存的数据，不会删除单聊对方或其他群成员已经保存的消息。

固定 SDK 没有为两个账号级清理 selector 提供独立的消息删除事件。当前页面的消息快照在 callback 成功后直接重置；后续会话变化继续通过 `onConversationChanged:` 合并。成功 callback 不能证明其他已登录设备已经刷新本地副本，必要时应以实际同步结果或重新查询校准。
