本地插入用于迁移、系统提示或产品自定义的只读记录。它不会把消息发送给其他用户，也不能替代 `sendMessage:...`。

## 插入单聊消息

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `message` | `OIMMessageInfo *` | 必填 | 要写入本地单聊记录的完整消息对象。 |
| `recvID` | `NSString *` | 必填 | 单聊接收方用户 ID。 |
| `sendID` | `NSString *` | 必填 | 单聊发送方用户 ID。 |

```objc
OIMMessageInfo *message =
    [OIMMessageInfo createTextMessage:@"本地提示消息"];

[[OIMManager manager] insertSingleMessageToLocalStorage:message
                                                  recvID:@"user_b"
                                                  sendID:currentUserID
                                               onSuccess:^(OIMMessageInfo * _Nullable storedMessage) {
    if (storedMessage == nil) {
        [self showInsertError:0 message:@"本地插入回调未返回消息对象"];
        return;
    }
    [self upsertMessage:storedMessage];
}
                                               onFailure:^(NSInteger code, NSString * _Nullable errorMessage) {
    [self showInsertError:code message:errorMessage ?: @"插入单聊本地消息失败"];
}];
```

## 插入群聊消息

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `message` | `OIMMessageInfo *` | 必填 | 要写入本地群聊记录的完整消息对象。 |
| `groupID` | `NSString *` | 业务必填 | 目标群组 ID；固定 header 允许传 `nil`，但无法定位群组时不应调用。 |
| `sendID` | `NSString *` | 业务必填 | 消息发送方用户 ID；固定 header 允许传 `nil`，调用前应校验。 |

```objc
[[OIMManager manager] insertGroupMessageToLocalStorage:message
                                                groupID:groupID
                                                 sendID:currentUserID
                                              onSuccess:^(OIMMessageInfo * _Nullable storedMessage) {
    if (storedMessage == nil) {
        [self showInsertError:0 message:@"本地插入回调未返回消息对象"];
        return;
    }
    [self upsertMessage:storedMessage];
}
                                              onFailure:^(NSInteger code, NSString * _Nullable errorMessage) {
    [self showInsertError:code message:errorMessage ?: @"插入群聊本地消息失败"];
}];
```

成功 callback 的参数按固定声明可能为 `nil`；非空时才是写入后的 `OIMMessageInfo`。调用端按 `clientMsgID` 更新当前列表，或重新读取历史校准。

两个方法只修改当前设备，不触发 `onRecvNewMessage:`，也不通知其他设备。需要多端同步、离线投递或服务端审核的内容必须使用发送 API。
