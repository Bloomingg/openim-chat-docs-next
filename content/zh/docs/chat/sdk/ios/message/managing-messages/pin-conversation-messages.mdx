消息置顶用于固定展示会话中的重要消息。它与会话列表置顶不是同一个状态：消息置顶使用 `setConversationPinnedMsgWithConversationID:clientMsgID:pinned:onSuccess:onFailure:`，会话列表置顶使用 `pinConversation:isPinned:onSuccess:onFailure:`。

## 获取置顶消息

`getConversationPinnedMsgWithConversationID:onSuccess:onFailure:` 接收 `conversationID`，返回 `NSArray<OIMMessageInfo *> *`：

```objc
[[OIMManager manager]
    getConversationPinnedMsgWithConversationID:conversationID
    onSuccess:^(NSArray<OIMMessageInfo *> * _Nullable messages) {
        [self replacePinnedMessages:messages ?: @[]
                     conversationID:conversationID];
    }
    onFailure:^(NSInteger code, NSString * _Nullable message) {
        [self showPinnedMessageError:code message:message ?: @"获取置顶消息失败"];
    }];
```

成功后用返回数组建立该会话的置顶消息快照；查询本身不会触发置顶事件。数组只包含 `OIMMessageInfo`，不包含置顶时间和操作用户；需要这些详情时使用变化事件的 `OIMConversationPinnedMsg`。列表中的消息仍按 `clientMsgID` 去重。消息被删除、撤回或修改后，应结合对应事件更新展示，不要永久缓存初次查询结果。

## 置顶或取消置顶

把 `pinned` 设为 `YES` 表示置顶，设为 `NO` 表示取消置顶：

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `conversationID` | `NSString *` | 必填 | 消息所属的会话 ID。 |
| `clientMsgID` | `NSString *` | 必填 | 要置顶或取消置顶的消息 ID。 |
| `pinned` | `BOOL` | 必填 | `YES` 表示置顶，`NO` 表示取消置顶。 |

```objc
[[OIMManager manager]
    setConversationPinnedMsgWithConversationID:conversationID
    clientMsgID:message.clientMsgID
    pinned:YES
    onSuccess:^(NSString * _Nullable data) {
        [self setPinnedPending:NO clientMsgID:message.clientMsgID];
    }
    onFailure:^(NSInteger code, NSString * _Nullable errorMessage) {
        [self showPinnedMessageError:code message:errorMessage ?: @"置顶消息失败"];
    }];
```

取消置顶时只需把 `pinned` 改为 `NO`。可置顶的消息类型、数量上限和操作权限由当前 OpenIMServer 配置决定。客户端可以隐藏无权限入口，但不能用本地状态绕过服务端校验。

成功 callback 表示服务端已完成本次设置，不返回变化后的置顶列表，也不代表所有客户端已经收到事件。

## 监听变化

本页是 `onChangedPinnedMsg:` 的完整事件归属页。`OIMChangedPinnedMsg` 包含 `conversationID` 和 `msgs`；每个 `OIMConversationPinnedMsg` 提供操作用户 `userID`、`pinnedTime` 和完整消息 `msg`。

```objc
- (void)onChangedPinnedMsg:(OIMChangedPinnedMsg *)change {
    if (change.conversationID.length == 0) {
        return;
    }

    [self replacePinnedMessageDetails:change.msgs ?: @[]
                         conversationID:change.conversationID];
}
```

事件载荷是变化后的置顶消息详情快照。按 `conversationID` 替换对应会话的置顶集合，再按每个 `msg.clientMsgID` 渲染去重。事件也可能来自其他客户端或其他有权限的成员。listener 的注册和移除方式见[接收消息](/sdk/ios/message/receiving-messages/receive-messages)。

调用成功、`onChangedPinnedMsg:` 到达和重新调用查询 API 建立快照是三个独立阶段。
