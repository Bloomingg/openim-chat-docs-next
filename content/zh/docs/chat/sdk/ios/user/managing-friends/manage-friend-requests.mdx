## 发送好友申请

`userID` 是申请接收方，`reqMessage` 是申请附言；没有附言时传空字符串。产品层还应限制附言长度并校验内容。

```objc
[[OIMManager manager] addFriend:@"user_b"
                     reqMessage:@"你好，我是 Alex"
                      onSuccess:nil
                      onFailure:nil];
```

成功只表示服务端已接收申请，不表示双方已经成为好友。

## 查询申请列表

收到的申请使用 `GetFriendApplicationListAsRecipientReq`，可通过 `handleResults` 选择处理状态；发出的申请使用 `GetFriendApplicationListAsApplicantReq`。两者都用 `offset` 和 `count` 分页。

| 参数 | 适用对象 | 说明 |
| --- | --- | --- |
| `handleResults` | 收到的申请 | `OIMApplicationStatusNormal` 为待处理，`Accept` 为已同意，`Decline` 为已拒绝。 |
| `offset` | 两者 | 起始偏移量，第一页传 `0`。 |
| `count` | 两者 | 本次读取数量。 |

```objc
GetFriendApplicationListAsRecipientReq *receivedReq = [GetFriendApplicationListAsRecipientReq new];
receivedReq.handleResults = @[@(OIMApplicationStatusNormal)];
receivedReq.offset = 0;
receivedReq.count = 20;
[[OIMManager manager] getFriendApplicationListAsRecipientWithReq:receivedReq
                                                       onSuccess:^(NSArray<OIMFriendApplication *> *items) {
                                                           [requestStore replaceReceived:items];
                                                       }
                                                       onFailure:nil];

GetFriendApplicationListAsApplicantReq *sentReq = [GetFriendApplicationListAsApplicantReq new];
sentReq.offset = 0;
sentReq.count = 20;
[[OIMManager manager] getFriendApplicationListAsApplicantWithReq:sentReq
                                                       onSuccess:^(NSArray<OIMFriendApplication *> *items) {
                                                           [requestStore replaceSent:items];
                                                       }
                                                       onFailure:nil];
```

后续分页把 `offset` 设为已加载数量。关系变化后重置为第一页，避免旧偏移量跳过记录。

成功 callback 返回当前页的 `OIMFriendApplication` 数组。常用属性包括申请双方的 `fromUserID`、`toUserID`、昵称和头像，以及 `handleResult`、`reqMsg`、`createTime`、`handleMsg`、`handleTime` 和 `ex`。申请记录必须以 `fromUserID:toUserID` 定位，不能只按展示名称或列表位置更新。

## 未处理申请数量

`time` 通常传上次查看申请列表的时间，用于统计该时间点之后的未处理申请。成功 callback 返回符合条件的数量快照；查询本身不触发申请 delegate。

```objc
GetFriendApplicationUnhandledCountReq *req = [GetFriendApplicationUnhandledCountReq new];
req.time = lastViewedAt;
[[OIMManager manager] getFriendApplicationUnhandledCount:req
                                               onSuccess:^(NSInteger count) {
                                                   [requestStore setUnhandledCount:count];
                                               }
                                               onFailure:nil];
```

## 处理好友申请

同意和拒绝都传申请方 `userID` 与处理附言；没有附言时传空字符串。

```objc
[[OIMManager manager] acceptFriendApplication:applicantUserID
                                     handleMsg:@"已通过"
                                     onSuccess:nil
                                     onFailure:nil];

[[OIMManager manager] refuseFriendApplication:applicantUserID
                                     handleMsg:@"暂不添加"
                                     onSuccess:nil
                                     onFailure:nil];
```

固定版本没有公开删除好友申请记录的 selector，因此不提供 WASM `deleteFriendRequests()` 的替代调用。

## 调用后的状态变化

| 操作 | 成功 callback 表示 | 后续状态 |
| --- | --- | --- |
| `addFriend:reqMessage:...` | 服务端已接收申请。 | `onFriendApplicationAdded:` 分流到收到或发出的列表。 |
| `acceptFriendApplication:handleMsg:...` | 同意请求已完成。 | `onFriendApplicationAccepted:` 更新申请；好友关系由 `onFriendAdded:` 同步。 |
| `refuseFriendApplication:handleMsg:...` | 拒绝请求已完成。 | `onFriendApplicationRejected:` 更新状态并移出待处理列表。 |

callback 成功、申请 delegate、好友关系 delegate 和重新查询校准是不同阶段。

## 监听申请事件

新增事件会同时到达申请方和接收方，不能全部计入“收到的申请”或未处理数量。先固定当前登录用户 ID，再按 `toUserID` 分流：

```objc
@interface FriendRequestSync : NSObject <OIMFriendshipListener>
@property (nonatomic, copy) NSString *currentUserID;
- (instancetype)initWithCurrentUserID:(NSString *)currentUserID;
@end

@implementation FriendRequestSync
- (instancetype)initWithCurrentUserID:(NSString *)currentUserID {
    if (self = [super init]) { _currentUserID = [currentUserID copy]; }
    return self;
}
- (BOOL)isReceived:(OIMFriendApplication *)item {
    return [item.toUserID isEqualToString:self.currentUserID];
}
- (void)onFriendApplicationAdded:(OIMFriendApplication *)item {
    if ([self isReceived:item]) {
        [requestStore upsertReceived:item];
        if (item.handleResult == OIMApplicationStatusNormal) {
            [requestStore incrementUnhandledForKey:
                [NSString stringWithFormat:@"%@:%@", item.fromUserID, item.toUserID]];
        }
    } else {
        [requestStore upsertSent:item];
    }
}
- (void)onFriendApplicationAccepted:(OIMFriendApplication *)item {
    [requestStore upsert:item received:[self isReceived:item]];
    [requestStore removePendingAndDecrementUnhandledIfNeeded:item
                                                   received:[self isReceived:item]];
}
- (void)onFriendApplicationRejected:(OIMFriendApplication *)item {
    [requestStore upsert:item received:[self isReceived:item]];
    [requestStore removePendingAndDecrementUnhandledIfNeeded:item
                                                   received:[self isReceived:item]];
}
- (void)onFriendApplicationDeleted:(OIMFriendApplication *)item {
    [requestStore remove:item received:[self isReceived:item]];
    [requestStore decrementUnhandledIfNeeded:item received:[self isReceived:item]];
}
@end

NSString *currentUserID = [[OIMManager manager] getLoginUserID];
FriendRequestSync *friendRequestSync =
    [[FriendRequestSync alloc] initWithCurrentUserID:currentUserID];
[[OIMManager callbacker] addFriendListener:friendRequestSync];
// 账号作用域结束时：
[[OIMManager callbacker] removeFriendListener:friendRequestSync];
```

`incrementUnhandledForKey:` 应按 `fromUserID:toUserID` 去重，避免重复事件重复计数。处理或删除事件只在当前用户是接收方、且该记录原本计入待处理数时递减，最低保持为 `0`。首次进入或用户主动刷新时重新查询两份列表和未处理数量；分页期间收到变化则重置分页状态。

申请被同意后，好友关系由 `onFriendAdded:` 同步，其完整处理见[更新或删除好友](/sdk/ios/user/managing-friends/update-or-delete-friends)。
