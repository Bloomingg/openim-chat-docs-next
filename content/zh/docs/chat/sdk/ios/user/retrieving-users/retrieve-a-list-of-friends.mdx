好友列表可用于通讯录、联系人选择器和好友关系缓存。所有场景都必须使用分页接口读取 `OIMFriendInfo`，不公开一次加载完整列表的方法。即使好友数量暂时不多，也保留分页状态，避免增长后阻塞 UI 或占用过量内存。

## 分页读取好友列表

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `offset` | `NSInteger` | 必填 | 起始偏移量，第一页传 `0`。 |
| `count` | `NSInteger` | 必填 | 本次读取数量。 |
| `filterBlack` | `BOOL` | 必填 | 是否过滤当前黑名单用户；普通通讯录通常传 `YES`。 |

```objc
NSInteger offset = 0;
NSInteger count = 50;
[[OIMManager manager] getFriendListPageWithOffset:offset
                                             count:count
                                       filterBlack:YES
                                         onSuccess:^(NSArray<OIMFriendInfo *> *friends) {
                                             [friendStore appendPage:friends];
                                         }
                                         onFailure:^(NSInteger code, NSString *message) {
                                             NSLog(@"getFriendListPage failed: %ld %@", (long)code, message);
                                         }];
```

### 返回结果

成功 callback 返回当前页的 `OIMFriendInfo` 数组：

| 属性 | 说明 |
| --- | --- |
| `userID` | 好友用户 ID。 |
| `nickname` | 好友账号级公开昵称。 |
| `faceURL` | 好友账号级公开头像。 |
| `remark` | 当前用户为该好友设置的备注。 |
| `attachedInfo` | SDK 返回的附加信息。 |
| `ex` | SDK 返回的扩展字段。 |

使用 `userID` 合并记录，展示名优先使用 `remark`，再回退到 `nickname` 和 `userID`。

## 分页状态

加载下一页时，把 `offset` 设为当前已加载的好友数量。若本次返回数量小于 `count`，通常表示已经没有更多好友。用户主动刷新或分页期间发生好友关系变化时，应清空已加载数据并从第一页重建快照。

## 搜索好友

搜索参数的四个字段都应明确设置。`keywordList` 当前只传一个去除首尾空白后的非空关键词：

```objc
NSString *keyword = [searchText stringByTrimmingCharactersInSet:
    NSCharacterSet.whitespaceAndNewlineCharacterSet];
if (keyword.length == 0) {
    [searchStore replaceResults:@[]];
    return;
}

OIMSearchFriendsParam *param = [OIMSearchFriendsParam new];
param.keywordList = @[keyword];
param.isSearchUserID = YES;
param.isSearchNickname = YES;
param.isSearchRemark = YES;

[[OIMManager manager] searchFriends:param
                           onSuccess:^(NSArray<OIMSearchFriendsInfo *> *friends) {
                               [searchStore replaceResults:friends];
                           }
                           onFailure:nil];
```

成功 callback 返回 `OIMSearchFriendsInfo` 数组。每项包含好友资料字段，并通过 `relationship` 表示当前用户与结果用户的关系类型。展示名仍优先使用 `remark`，再回退到 `nickname` 和 `userID`。

## 列表如何更新

首次进入时从 `offset:0` 读取第一页，加载更多时按 `userID` 追加去重。搜索结果只属于当前搜索界面，不应覆盖完整好友列表。

好友新增、删除和资料变化由[更新或删除好友](/sdk/ios/user/managing-friends/update-or-delete-friends)页的 delegate 合并。查询成功只建立快照，不应描述为触发好友事件；重新登录后的变化由事件同步，首次进入页面时仍需分页查询。
