已经知道一个或多个好友的 `userID`，并且只需要刷新这些好友的关系资料时，使用 `getSpecifiedFriendsInfo:filterBlack:onSuccess:onFailure:`。该接口适合好友资料卡、联系人详情和好友设置等按需查询场景，无需为少量用户重新获取整页好友列表。

## 获取指定好友资料

第一个参数是要查询的好友用户 ID 数组，不可为 `nil`。`filterBlack` 决定是否过滤当前用户黑名单中的用户，普通好友资料页通常传 `YES`。

```objc
[[OIMManager manager] getSpecifiedFriendsInfo:@[@"user_a", @"user_b"]
                                  filterBlack:YES
                                    onSuccess:^(NSArray<OIMFriendInfo *> *friends) {
                                        [friendStore mergeFriends:friends];
                                    }
                                    onFailure:nil];
```

成功 callback 返回匹配的 `NSArray<OIMFriendInfo *> *`。固定 SDK 中的字符串属性均为 nullable，消费端应先检查 `userID`，不要用昵称或数组位置作为合并标识。

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `userID` | `NSString * _Nullable` | 好友用户 ID，也是查询结果的稳定合并标识。 |
| `ownerUserID` | `NSString * _Nullable` | 当前好友关系所属的用户 ID。 |
| `nickname` | `NSString * _Nullable` | 好友账号昵称。 |
| `faceURL` | `NSString * _Nullable` | 好友头像地址。 |
| `remark` | `NSString * _Nullable` | 当前用户为好友设置的备注。 |
| `createTime` | `NSInteger` | 好友关系创建时间。 |
| `addSource` | `NSInteger` | 建立好友关系的来源。 |
| `operatorUserID` | `NSString * _Nullable` | 促成好友关系的操作用户 ID。 |
| `attachedInfo` | `NSString * _Nullable` | SDK 保留的附加信息；只在业务已有约定时解析。 |
| `ex` | `NSString * _Nullable` | 好友关系扩展字段。 |

返回空数组通常表示目标不是当前用户的好友，或已经被 `filterBlack` 过滤。展示好友候选人、陌生人资料卡或业务搜索结果时，应使用[获取指定用户资料](/sdk/ios/user/retrieving-users/retrieve-users)中的 `getUsersInfo:onSuccess:onFailure:`。

## 检查好友关系

```objc
[[OIMManager manager] checkFriend:@[@"user_a", @"user_b"]
                        onSuccess:^(NSArray<OIMSimpleResultInfo *> *results) {
                            [relationshipStore mergeResults:results];
                        }
                        onFailure:nil];
```

`checkFriend:onSuccess:onFailure:` 适合更新资料卡中的关系标识、加好友按钮状态，以及发送申请前的校验；需要展示好友备注、头像或关系扩展字段时，仍应使用指定好友资料查询。

成功 callback 中每个 `OIMSimpleResultInfo` 包含以下属性：

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `userID` | `NSString * _Nullable` | 本项对应的目标用户 ID。 |
| `result` | `NSInteger` | `1` 表示目标是好友并且不在黑名单中；其他结果按非好友处理。 |

不要把 `result` 自行扩展为固定 SDK 未声明的额外状态。若需区分黑名单状态，应另行查询黑名单。

## 调用结果与关系变化

两个方法成功后都只更新本次查询的关系快照，不会触发好友 delegate。建立、删除好友关系或修改关系资料后的增量事件统一由[更新或删除好友](/sdk/ios/user/managing-friends/update-or-delete-friends)处理。页面首次打开或用户主动刷新时，再调用指定好友查询校准当前资料。
