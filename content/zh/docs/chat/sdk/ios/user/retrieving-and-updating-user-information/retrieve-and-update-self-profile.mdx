本页只处理当前登录用户。使用 `getSelfInfoWithOnSuccess:onFailure:` 获取完整 `OIMUserInfo`，使用 `setSelfInfo:onSuccess:onFailure:` 更新当前用户的昵称、头像或扩展字段。其他账号的公开资料使用 `getUsersInfo:onSuccess:onFailure:`；管理员维护用户目录时应由可信后端完成权限校验和审计。

## 读取当前用户资料

完成初始化、登录并等待连接成功后读取当前资料：

```objc
void (^loadCurrentUser)(void) = ^{
    [[OIMManager manager] getSelfInfoWithOnSuccess:^(OIMUserInfo *user) {
        [profileStore replaceCurrentUser:user];
    } onFailure:^(NSInteger code, NSString *message) {
        NSLog(@"getSelfInfo failed: %ld %@", (long)code, message);
    }];
};

loadCurrentUser();
```

成功 callback 返回完整 `OIMUserInfo`。常用属性包括 `userID`、`nickname`、`faceURL`、`createTime`、`ex`、`attachedInfo` 和 `globalRecvMsgOpt`。

## 更新当前用户资料

固定 Objective-C 接口接收完整 `OIMUserInfo`，不要直接 `[OIMUserInfo new]` 后只赋少数字段；未赋值的标量属性会带默认值，可能意外重设 `globalRecvMsgOpt`。应先读取当前对象，再只修改资料编辑器拥有的属性：

```objc
[[OIMManager manager] getSelfInfoWithOnSuccess:^(OIMUserInfo *current) {
    current.nickname = [nicknameInput stringByTrimmingCharactersInSet:
        NSCharacterSet.whitespaceAndNewlineCharacterSet];
    current.faceURL = avatarURL.absoluteString;

    [[OIMManager manager] setSelfInfo:current
                            onSuccess:^(NSString *data) {
                                // 资料写入成功；再独立重新读取最终值。
                                loadCurrentUser();
                            }
                            onFailure:^(NSInteger code, NSString *message) {
                                NSLog(@"setSelfInfo failed: %ld %@", (long)code, message);
                            }];
} onFailure:^(NSInteger code, NSString *message) {
    NSLog(@"getSelfInfo before update failed: %ld %@", (long)code, message);
}];
```

资料写入与重新读取是两个阶段。写入成功但刷新失败时，不要把刷新错误误报为写入失败；可以保留提交成功状态，并提示用户稍后刷新。

### 更新扩展字段

`OIMUserInfo.ex` 是完整字符串，写入会整体覆盖旧值，不会自动合并 JSON。多个模块共用时，先读取当前值、处理空值或解析失败，再合并各自命名空间并写回完整结果：

```objc
[[OIMManager manager] getSelfInfoWithOnSuccess:^(OIMUserInfo *current) {
    NSData *data = [current.ex dataUsingEncoding:NSUTF8StringEncoding];
    NSDictionary *decoded = data.length
        ? [NSJSONSerialization JSONObjectWithData:data options:0 error:nil]
        : nil;
    NSMutableDictionary *extra = [decoded isKindOfClass:NSDictionary.class]
        ? [decoded mutableCopy]
        : [NSMutableDictionary dictionary];
    extra[@"profile"] = @{ @"theme": @"dark" };

    NSData *encoded = [NSJSONSerialization dataWithJSONObject:extra options:0 error:nil];
    current.ex = [[NSString alloc] initWithData:encoded encoding:NSUTF8StringEncoding];

    [[OIMManager manager] setSelfInfo:current onSuccess:nil onFailure:nil];
} onFailure:nil];
```

`ex` 只适合轻量、低敏感度的聊天展示数据。组织、权限、风控、审计和搜索索引应由可信后端维护；多端并发写入需要业务后端提供版本或冲突控制。

### 设置全局消息接收选项

通知设置使用独立 selector，避免资料表单写回整个模型时意外改变接收选项：

```objc
[[OIMManager manager] setGlobalRecvMessageOpt:OIMReceiveMessageOptNotNotify
                                    onSuccess:nil
                                    onFailure:^(NSInteger code, NSString *message) {
                                        NSLog(@"setGlobalRecvMessageOpt failed: %ld %@", (long)code, message);
                                    }];
```

`OIMReceiveMessageOptReceive` 表示正常接收并允许离线推送，`OIMReceiveMessageOptNotReceive` 表示不接收消息，`OIMReceiveMessageOptNotNotify` 表示在线接收但离线不推送。

## 调用后的状态变化

- `getSelfInfoWithOnSuccess:onFailure:` 成功只建立当前账号资料快照，不触发资料变化 delegate。
- `setSelfInfo:onSuccess:onFailure:` 成功表示资料写入请求完成；`onSelfInfoUpdated:` 随后同步当前账号的最终资料。
- `setGlobalRecvMessageOpt:onSuccess:onFailure:` 成功表示全局接收选项设置请求完成；需要立即确认最终资料时再独立查询。
- callback 成功、delegate 到达和重新查询校准是三个不同阶段。

## 监听资料变化

```objc
@interface ProfileSync : NSObject <OIMUserListener>
@end

@implementation ProfileSync
- (void)onSelfInfoUpdated:(OIMUserInfo *)info {
    [profileStore replaceCurrentUser:info];
}
@end

[[OIMManager callbacker] addUserListener:profileSync];
// 页面或账号作用域结束时：
[[OIMManager callbacker] removeUserListener:profileSync];
```

delegate 实例必须保持稳定引用。切换账号前移除旧 listener，避免把旧账号资料写入新账号状态。

其他用户公开资料见[获取指定用户资料](/sdk/ios/user/retrieving-users/retrieve-users)，在线状态见[获取用户在线状态](/sdk/ios/user/retrieving-and-updating-user-information/retrieve-the-online-status-of-a-user)。
