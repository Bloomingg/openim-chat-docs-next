OpenIM iOS SDK 通过订阅机制同步用户在线状态。建议只订阅会话列表当前页、联系人当前页或资料卡中可见的用户。

在线状态只表示用户是否连接到 OpenIMServer。好友关系、群成员状态、禁言状态和应用自己的活跃时间，应使用各自的数据源。

## 订阅可见用户的状态

```objc
NSArray<NSString *> *userIDs = @[@"user_a", @"user_b"];
[[OIMManager manager] subscribeUsersStatus:userIDs
                                 onSuccess:^(NSArray<OIMUserStatusInfo *> *states) {
                                     [presenceStore mergeStates:states];
                                 }
                                 onFailure:^(NSInteger code, NSString *message) {
                                     NSLog(@"subscribeUsersStatus failed: %ld %@", (long)code, message);
                                 }];
```

调用前先移除空字符串和重复的 `userID`；没有目标用户时，不需要调用 SDK。成功 callback 返回 `OIMUserStatusInfo` 数组，应先用它建立当前快照；后续变化再通过 `onUserStatusChanged:` 到达。

每项结果都以 `userID` 合并，不能按数组位置覆盖。`status` 表示在线状态，`platformIDs` 表示在线端的平台列表；平台列表可能为空，即使没有具体平台信息，界面仍应按 `status` 正常显示状态，不要把空数组直接解释为查询失败。

分页、搜索结果或虚拟滚动范围变化时，只订阅新进入可见范围的用户，并取消已离开范围的订阅。不要一次订阅整个用户目录。

## 监听状态变化

```objc
@interface PresenceSync : NSObject <OIMUserListener>
@end

@implementation PresenceSync
- (void)onUserStatusChanged:(OIMUserStatusInfo *)info {
    if (info.userID.length == 0) {
        return;
    }
    [presenceStore mergeState:info userID:info.userID];
}
@end

[[OIMManager callbacker] addUserListener:presenceSync];
// 统一状态层销毁时：
[[OIMManager callbacker] removeUserListener:presenceSync];
```

本页是 `onUserStatusChanged:` 的完整 listener 归属页。若多个页面都显示在线状态，应在统一的用户状态层注册一个稳定的 `PresenceSync` 实例；各页面只订阅和读取自己需要的用户，避免重复处理同一事件。

事件只代表已订阅用户的增量变化。应以 `userID` 写入当前账号的状态表，再更新会话列表、成员列表或资料卡；不能把事件到达等同于先前的订阅调用刚完成。

## 查询已订阅用户状态

```objc
[[OIMManager manager] getSubscribeUsersStatusWithOnSuccess:^(NSArray<OIMUserStatusInfo *> *states) {
    [presenceStore mergeStates:states];
} onFailure:^(NSInteger code, NSString *message) {
    NSLog(@"getSubscribeUsersStatus failed: %ld %@", (long)code, message);
}];
```

需要局部刷新、断线重连或重新进入列表页时，可以使用此方法取得当前账号已订阅用户的状态快照。它不接收目标用户数组，也不改变订阅集合。

成功 callback、`onUserStatusChanged:` 到达及页面重新查询是不同阶段。此方法只返回调用当时的快照，不应描述为触发在线状态事件；快照和事件都按 `userID` 合并。

## 取消订阅

```objc
[[OIMManager manager] unsubscribeUsersStatus:userIDs
                                   onSuccess:^(NSString *data) {
                                       [presenceStore removeUserIDs:userIDs];
                                   }
                                   onFailure:^(NSInteger code, NSString *message) {
                                       NSLog(@"unsubscribeUsersStatus failed: %ld %@", (long)code, message);
                                   }];
```

用户离开页面、列表翻页、搜索结果变化或页面结束时，应取消已离开可见范围用户的订阅。成功 callback 后，再从应用自己的订阅集合和状态缓存移除这些 `userID`；调用失败时保留当前集合，以便重试或重新查询校准。

退出登录或切换账号时，还要清空旧账号的订阅集合和状态缓存，并使用注册时的同一个 delegate 实例调用 `removeUserListener:`。移除 listener 与取消用户订阅是两件事，两者都不能省略。
